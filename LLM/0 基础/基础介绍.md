1. 语言模型发展阶段
	- 统计语言模型（Statistical Language Model, `SLM`）
	- 神经语言模型（Neural Language Model, `NLM`）
	- 预训练语言模型（Pre-trained Language Model, `PLM`）
		- BERT：编码器架构被认为更适合去解决自然语言理解任务（如完形填空等）
		- GPT-1：解码器架构更适合解决自然语言生成任务（如文本摘要等）
		- 确立了“预训练-微调”这一任务求解范式
			- 预训练：大规模无标注文本建立模型的基础能力
			- 微调：有标注数据对于模型进行下游特定任务的适配
	- 大语言模型（Large Language Model, `LLM`）
		- 通过规模扩展 （如增加模型参数规模或数据规模）通常会带来下游任务的模型性能提升，这种现象通常被称为“扩展法则”（Scaling Law）
		- 模型参数、训练数据、计算算力，不再依靠下游任务数据的微调，进行通用任务的求解
2. 大语言模型的能力特点
	- 初步实现了通过统一形式来解决各种下游任务
	- 具有较为丰富的世界知识
	- 具有较强的通用任务解决能力
	- 具有较好的复杂任务推理能力
	- 具有较强的人类指令遵循能力
	- 具有较好的人类对齐能力
		- 通过基于人类反馈的强化学习技术，规避常见的使用风险
	- 具有可拓展的工具使用能力
3. 大语言模型关键技术
	- 规模扩展：参数、数据、算力
	- 数据工程：不是简单的扩大数据规模
		- 全面的采集，拓宽高质量的数据来源
		- 精细的清洗，尽量提升数据质量
		- 设计有效的数据配比与数据课程，加强模型对于数据语义信息的利用效率
	- 高效预训练
		- 需要联合使用各种并行策略以及效率优化方法，3D并行，ZeRO（内存冗余消除技术）
		- 代表性的分布式训练软件：DeepSpeed 和 Megatron-LM
	- 能力激发
		- 指令微调方面：训练大模型学会利用自身所掌握的知识与信息进行任务的求解
		- 提示学习方面：设计合适的提示策略去诱导大语言模型生成正确的问题答案，包括上下文学习、思维链提示
	- 人类对齐
		- 3H对齐标准：Helpfulness（有用性）、Honesty（诚实性）和 Harmlessness（无害性）
		- OpenAI 提出了基于人类反馈的强化学习算法（Reinforcement Learning from Human Feedback, RLHF）
		- RLHF优化过程较为复杂：通过监督微调的对齐方式简化，如DPO算法
	- 工具使用
		- 能力也受限于预训练数据时间范围以及覆盖内容
		- 通过“插件机制”（GPT系列）让大语言模型学会使用各种工具的调用方式，有效扩展能力范围
		- 在技术路径上，工具调用能力主要是通过指令微调以及提示学习两种途径实现
		- 随着应用范围的不断拓展，创建广泛的、可供大模型使用的工具资源变得愈为重要
4. 大语言模型
	- 是指在海量无标注文本数据上进行预训练得到的大型预训练语言模型
	- 是一种基于Transformer结构的神经网络模型
	- 是一种拥有大规模参数的函数，构建过程是使用训练数据对模型参数拟合的过程
	- 训练过程可以分为大规模预训练和指令微调与人类对齐两个阶段
	- 大规模预训练
		- 大规模预训练本质上是在做一个世界知识的压缩
		- “解码器架构+预测下一个词”成为现有大语言模型主要采纳的技术路径
		- 需要考虑各种实施细节：数据如何进行配比、如何进行学习率的调整、如何早期发现模型的异常行为
	- 指令微调与人类对齐
		- 指令微调（也叫做有监督微调， Supervised Fine-tuning, SFT），通过使用任务输入与输出的配对数据进行模型训练，主要起到了对于模型能力的激发作用，而不是知识注入作用
		- 基于人类反馈的强化学习对齐方法RLHF （Reinforcement Learning from Human Feedback），在指令微调后使用强化学习加强模型的对齐能力
5. 扩展法则
	- 扩展参数规模、数据规模和计算算力
	- KM扩展法则
		- 模型性能与三个主要因素：模型规模（$N$）、数据规模（$D$）和计算算力（$C$）之间的幂律关系
	- Chinchilla 扩展法则
		- DeepMind团队，旨在指导大语言模型充分利用给定的算力资源进行优化训练，能够获得算力资源固定情况下模型规模与数据规模的最优分配方案
		- 由于Transformer架构具有较好的数据扩展性，到目前为止（2024.03），还没有实验能够有效验证特定参数规模语言模型的饱和数据规模
	- 根据GPT-4的报告，通过扩展法则可以准确预测某些任务能力（例如编码能力），但是对于有些任务的性能预测是非常困难的。
	- 有些重要能力，例如上下文学习能力等涌现能力，根据扩展法则是不可预测的，只有当模型大小超过一定规模时才会出现
6. 涌现能力
	- 大语言模型的涌现能力被非形式化定义为“在小型模型中不存在但在大模型中出现的能力“
	- 具体指当模型扩展到一定规模时，模型的特定任务性能突然出现显著跃升的趋势，远超随机水平
	- 代表性的涌现能力
		- 上下文学习（In-context Learning, ICL）
			- 在提示中为语言模型提供自然语言指令和多个任务示例，无需显式的训练或梯度更新
			- 上下文学习能力还取决于具体的下游任务
		- 指令遵循（Instruction Following）
			- 大语言模型能够按照自然语言指令来执行对应的任务
			- 通常需要使用自然语言描述的多任务示例数据集进行微调，即指令微调
		- 逐步推理（Step-by-step Reasoning）
			- 可以利用思维链 （Chain-of-Thought, CoT）提示策略来加强推理性能
			- 具体指可以在提示中引入任务相关的中间推理步骤来加强复杂任务的求解
			- 思维链提示对不同任务的性能提升不完全相同
7. GPT系列模型
	- GPT是生成式预训练（Generative Pre-Training）的缩写。
	- 基本原理是训练模型学习恢复预训练文本数据，将广泛的世界知识压缩到仅包含解码器（Decoder-Only）的Transformer 模型中，从而使模型能够学习获得较为全面的能力。其中，有两个关键要素、
		- 训练能够准确预测下一个词的Transformer（只包含解码器）语言模型
		- 扩展语言模型的规模以及扩展预训练数据的规模
	- OpenAI对GPT的研发历程大致可分为四个阶段：早期探索阶段、路线确立阶段、能力增强阶段、能力跃升阶段
	- 早期探索阶段
		- 最开始使用循环神经网络，模型能力和并行训练能力还存在较大的局限性
		- 语言模型将每个（自然语言处理）任务都视为基于世界文本子集的下一个词预测问题。 因此，如果无监督语言建模经过训练后具有足够的能力复原全部世界文本，那么本质上它就能够解决各种任务
		- Transformer（2017）一经问世，OpenAI相继推出GPT-1（2018，100 M）和GPT-2（2019，1.5 B）
	- 规模扩展阶段
		- 在GPT-2基础上，GPT-3（2020，175B）针对（几乎相同的）模型参数规模进行了大幅扩展，在下游任务中初步展现出了一 定的通用性。
		- GPT-3的论文中，正式提出了“上下文学习”这一概念，上下文学习可以指导大语言模型学会“理解”自然语言文本形式描述的新任务，从而消除了针对新任务进行微调的需要
		- GPT-3是从预训练语言模型到大语言模型演进过程中的一个重要里程碑，建立了以提示学习方法为基础技术路线的任务求解范式
	- 能力增强阶段
		- OpenAI探索了两种主要途径来改进 GPT-3 模型，即代码数据训练和人类偏好对齐，即 GPT 3.5 模型
	- 性能跃升阶段
		- ChatGPT。2022 年11月，OpenAI发布了基于GPT模型的人工智能对话应用 服务ChatGPT。
		- GPT-4。OpenAI于2023年3月发布了GPT-4，首次将 GPT 系列模型的输入由单一文本模态扩展到了图文双模态
		- GPT-4V（视觉）、GPT-4 Turbo（升级版GPT-4）以及多模态支持模型。
