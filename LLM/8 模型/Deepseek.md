4. 独特创新
	- | 创新 | 说明 |
	  |------|------|
	  | **MLA** | 低秩 KV 联合压缩，KV cache 减少 90%+，推理效率远高于 GQA/MQA |
	  | **无辅助损失负载均衡** | 通过偏置项动态调整，而非辅助损失，避免性能损失 |
	  | **MTP** | 多 Token 预测（一次预测 2 个 token），提升训练效果和推理速度 |
	  | **FP8 训练** | 首次在超大规模模型上验证 FP8 训练可行性 |
	  | **专家专属学习率** | 共享专家和路由专家不同学习率（参考 Hunyuan）|

5. MLA vs GQA 区别
	- | | GQA | MLA（DeepSeek）|
	  |--|-----|---------------|
	  | **KV 缓存** | 减少头数（如 8 个 KV 头）| **低秩压缩**（压缩到 512 维 latent）|
	  | **缓存大小** | 与头数成正比 | **与头数无关，只与压缩维度有关** |
	  | **效果** | 性能略有损失 | **性能优于 MHA，缓存远小于 GQA** |
	  | **公式** | $K,V \in \mathbb{R}^{n_{kv} \times d}$ | $c^{KV} = W^{DKV}h \in \mathbb{R}^{d_c}$ |

8. DeepSeek 核心创新
	- （1）MLA（多头潜在注意力）
		- **核心思想**：用**低秩潜向量**替代传统 KV 缓存
		- 结构对比
			- | | 传统 GQA | **MLA** |
			  |--|---------|---------|
			  | **缓存内容** | 8 个 KV 头 × 128 维 | **潜向量 $c^{KV}$（512 维）** |
			  | **缓存公式** | $K, V \in \mathbb{R}^{8 \times 128}$ | $c^{KV} = W^{DKV} \cdot h \in \mathbb{R}^{512}$ |
			  | **解压方式** | 复制/插值到 128 头 | **矩阵乘法解压后 reshape** |
			  | **解压后头数** | 128（复制而来） | **128（独立恢复，与 Q 同）** |
			  | **显存节省** | 约 75% | **90%+** |
		- 关键公式
			- **压缩（缓存）**：$$c^{KV} = W^{DKV} \cdot h \in \mathbb{R}^{512}$$
			- **解压（计算时）**：$$k = W^{UK} \cdot c^{KV}, \quad v = W^{UV} \cdot c^{KV}$$
			- 然后 reshape 为 $[128 \text{ heads} \times 56 \text{ dim}]$ 
		- 注意
			- **所有头共享**一套压缩/解压矩阵（非每头独立）
			- 解压后恢复 **128 个完整头**（与 Q 头数相同），非 GQA 的 8 个分组
	- （2）DeepSeekMoE
		- **核心思想**：细粒度专家 + 无辅助损失均衡
		- 架构对比
			- | | Mixtral 8x22B | **DeepSeek-V3** |
			  |--|---------------|-----------------|
			  | **专家总数** | 8 个 | **256 路由 + 1 共享 = 257 个** |
			  | **每 token 激活** | 2 个专家 | **1 共享 + 8 路由 = 9 个专家** |
			  | **专家粒度** | 粗（22B/专家） | **细（约 2.6B/专家）** |
			  | **负载均衡** | 辅助损失 | **无辅助损失（偏置项动态调整）** |
		- 关键机制
			- **细粒度分割**：256 个小专家替代 8 个大专家，组合更灵活，表达能力更强
			- **共享专家**：1 个专家常驻（处理通用知识），避免路由抖动
			- **无辅助损失均衡**：
				- 传统：用额外 loss 强制均衡（损害性能）
				- **DeepSeek**：通过**偏置项**动态调整专家得分，无需辅助损失
			- **专家专属学习率**：共享专家和路由专家使用不同学习率（似 Hunyuan）
	- （3）总结
		- MLA 把 KV 缓存压成 512 维向量省显存
		- DeepSeekMoE 把专家切 256 份且不用辅助损失训得更稳
