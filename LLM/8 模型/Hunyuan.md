2. 核心架构
	- | 特性 | 配置 |
	  |------|------|
	  | **架构** | **Sparse MoE**（稀疏专家混合）|
	  | **总参数/激活参数** | 406B/32B（Hunyuan 2.0）；80B/13B（A13B）|
	  | **上下文长度** | **256K tokens** |
	  | **专家数量** | 64 专家（Image 3.0）；具体路由策略未完全公开 |
	  | **长上下文技术** | 基于 RoPE 的扩展（推测，未确认具体方法）|

3. 关键创新
	- | 特性 | 说明 |
	  |------|------|
	  | **双模式推理** | 快思考（直接回答）vs 慢思考（CoT 推理），可切换 |
	  | **Hybrid MoE-Mamba**（Turbo S）| 结合 Mamba 线性注意力与 MoE，降低长文本延迟 |
	  | **多模态原生** | Image/Video/3D 均为原生多模态，非外挂编码器 |
	  | **中文优化** | 训练数据中文占比高，汉字生成能力行业领先 |

	- MoE 架构 + 双模式推理（快/慢思考）+ 中文原生优化 + 256K 长上下文，底层组件细节（Norm/激活函数）未完全公开

4. Hunyuan-Large
	- 核心组件
		- | 组件 | 配置 |
		  |------|------|
		  | **位置编码** | **RoPE**（明确）|
		  | **激活函数** | **SwiGLU**（明确）|
		  | **注意力** | **GQA**（Grouped-Query Attention，明确）|
		  | **架构** | **MoE**（16 专家，Top-1 激活，明确）|
		  | **归一化** | **未明确说明**（论文未提 RMSNorm/LayerNorm）|
	- 关键参数
		- 总参数：389B
		- 激活参数：52B  
		- 专家数：16（ specialized）+ 1（shared）
		- 每 token 激活：1 shared + 1 specialized
		- 上下文：256K
		- 词表：128K
		- 层数：64
		- 注意力头：80
		- KV 头：8（GQA）
	- 独特机制
		- **Recycle Routing**：过载专家 token 随机分配给未过载专家（而非丢弃）
		- **专家特定学习率**：共享专家和专用专家不同学习率（0.31 倍）
		- **GQA + CLA**：KV 缓存压缩（跨层注意力共享）
