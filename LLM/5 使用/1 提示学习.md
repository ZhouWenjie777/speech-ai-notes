1. 任务提示（Prompting）的设计主要依靠人工设计和自动优化两种策略来实现
	- 上下文学习（In-context Learning, ICL）是一种典型的提示方法，将任务描述与示例以自然语言文本形式加入到提示中
	- 思维链提示（Chain-of-Thought, CoT）作为一种增强技术，将一系列中间推理步骤加入到提示中
2. 基础提示
	- （1）人工提示设计
		- 提示设计四要素
			- 任务描述：
				- 用清晰、具体动词说明目标，给出输入/输出格式关键词
			- 输入数据：
				- 自然语言即可；结构化数据先线性化或代码化，便于模型读取
			- 上下文信息：
				- 提供检索文档、示例样本（few-shot）或对话历史；示范输入-输出映射，降低歧义
			- 提示策略：
				- 前缀触发：“让我们一步一步地思考”→激活 CoT；“你是××专家”→提升领域表现
				- 对话模型宜拆多轮，逐子任务输入，利用其对话先验
			- 实用 checklist
				- 一次只给一项核心任务，避免多目标冲突  
				- 用分隔符`（###、```）`清晰标注不同部分  
				- 先 few-shot 再 zero-shot，逐步减少示例以测试鲁棒性
		- 提示设计十原则
			- 清晰具体：任务目标、输入格式、输出长度、风格全写明
			- 分解复杂任务：按编号拆子任务，逐步求解
			- 少量样本示例：高质量、多样化，用“###”或 XML 分隔
			- 先给角色/前缀：如“你是××专家”“Let’s think step by step” 
			- 用特殊符号高亮关键部分：`*、"""、<tag>`
			- 多选/比较题先限制答案空间，再让模型选
			- 长输入把问题放末尾，减少位置偏差
			- 非英语任务先机翻成英文，再喂模型
			- 若模型搞不定，把工具封装成可调 API，让模型调用
			- 示例不足时，先让模型自生成答案，再回送当 few-shot
	- （2）自动提示优化
		- 离散提示
			- 由一系列自然语言词元组成的完整句子表达（如“请根据提供的检索信息回答下列问题”）
			- 梯度近似：
				- 把词元替换成软嵌入→梯度下降→最近邻映射回词表；早期方法，需模型开放梯度
			- RL 搜索：
				- 把大模型当策略网络，以任务奖励（准确率/匹配度）为回报，用策略梯度生成词元序列；适合黑盒 API
			- 编辑迭代：
				- 预设“改任务描述/增删示例/换标签词”等编辑操作，小批量推理评估→保留涨分修改；零梯度、纯黑盒
			- 大模型自优化：
				- 用 LLM 生成/改写提示→评估→高分提示留种再变异；多轮蒙特卡洛避免局部最优
		- 连续提示
			- 由一组连续空间中的嵌入向量组成，可根据下游任务的损失直接通过梯度更新进行优化
			- 监督学习方法：
				- Prefix/Prompt-tuning 把嵌入当参数，冻结大模型，仅反向更新连续向量；节省显存但输入无关  
			- 迁移学习方法：
				- 先为多个源任务学各自连续提示，再在目标实例上用注意力加权组合，实现“实例级”动态提示，缓解单一提示泛化不足
3. 上下文学习
	- （1）上下文学习的形式化定义
		- 提示构成：任务描述 I + k 个示例 $D_k = \{f(x_i,y_i)\}_{i=1}^k$ + 新输入 $x_{k+1}$，整体喂给 LLM，直接生成 $y_{k+1}$：$$\text{LLM}\Bigl(I,\ f(x_1,y_1),\ \dots,\ f(x_k,y_k),\ f(x_{k+1},\underline{\phantom{y}})\Bigr)\to y_{k+1}$$  
		- 与指令微调区别：ICL **无需梯度更新**，靠提示即做任务；指令微调需训练且能提升零样本（即仅依赖任务描述而无需额外的示例） ICL 能力  
		- 示例作用：模型通过自然语言上下文 **“学习”** 输入-输出映射，本质是条件生成而非参数更新
	- （2）示例设计
		- 示例选择
			- 相关度排序：
				- 用 BERT 等编码器把候选与测试样本 embedding 算余弦，k-NN 选 top-k；简单高效，默认 baseline
			- 集合多样性：
				- MMR 或 DPP，在“与测试样相关”与“与已选样不相似”间折中，覆盖更多任务信息，减少冗余  
			- 大模型自评分：
				- 直接用 LLM 试跑加当前示例后的验证集，算性能增益当 reward；
				- 成本高，可先用正负例训轻量分类器代理筛选
			- 经验：先相关度粗筛，再多样性去冗余，最后 LLM 快速验证，兼顾质量与效率
		- 示例格式
			- 人工标注格式：
				- 从“输入-输出”→加任务描述→加思维链，格式清晰、推理透明；
				- 缺点：规模受限、多样性低  
			- 自动生成格式：
				- 先给少量种子示例，让 LLM 按模板续写新样本→人工/规则过滤→扩增至千条；
				- 快速覆盖长尾，但需去噪、去重复
			- 自动生成技巧
				- 种子多样化（≥8 条）、模板随机化（顺序、措辞）
				- 生成后立即用正则或模型自检过滤格式错误、逻辑矛盾
				- 对高难度任务可“自洽投票”：同一输入让 LLM 生成 3 次，保留多数答案一致的样本，降低噪声
		- 示例顺序
			- 位置偏置：模型更关注提示末端信息  
			- 候选生成：按“与测试样本语义相似度”排列，把最相关示例放末尾更靠近测试样本的位置，强化利用  
			- 质量评估：
				- 有测试集：直接跑任务，取最高准确率顺序
				- 无测试集：用模型预测分布 **熵值** 当指标，熵越低→置信越高→顺序越优  
			- 实践：先相似度粗排，再熵值/验证集精选，兼顾相关性与置信度
	- （3）底层机制
		- 预训练如何塑造 ICL 能力
			- 预训练任务：
				- MetaICL 在数十个领域任务上玩“示例→预测”元游戏，无需显式任务描述；元训练任务越多且越多样，小模型也能超越大模型，证明 **任务形式本身** 比参数规模更关键  
			- 预训练数据：
				- 领域混合 → 提升 ICL 广度；长程拼接 → 增强示例-目标关联感知
				- 高梯度相似度子集（低频长尾词多）对 ICL 增益最大；说明“学习难度高”的语料迫使模型学会从上下文快速抽象规则，恰好对齐 ICL 目标
		- 推理阶段两种 ICL 范式
			- 任务识别：
				- 依赖预训练已编码的隐变量，示例仅作“触发器”；
				- 预存隐变量已涵盖多数任务，LLM 仅需从示例中提取任务表征 → 触发对应内部回路；替换标签为随机符号仍可用，说明依赖“任务语义”而非标签字面
			- 任务学习（隐式梯度下降）：
				- 示例被当作**元训练集**，前向生成损失信号 → 模型内部隐式地执行梯度步，更新其“元参数”
				- 换无意义随机标签时，大模型性能下降更小，说明大模型从示例中习得新映射规则的能力随规模显著增强，即“任务学习”成分随规模增大而增强
			- 规模结论：
				- 大模型参数量大，可近似更复杂优化器，小模型只能做简单任务匹配
				- 小模型 ≈ 任务识别器  
				- 大模型（≥66 B）≈ 在线学习器，可在上下文里**真正学会**未见过的新规则
4. 思维链提示
	- （1）思维链提示的基本形式
		- 形式：
			- 把 ICL 的 $⟨输入, 输出⟩$ 升级为 $⟨输入, 思维链, 输出⟩$，用自然语言中间步骤桥接 
		- 零样本触发：
			- 仅加一句 “Let’s think step by step.” 即可让模型先输出推理链再给出答案，无需人工写链
		- 效果：
			- 在算术、常识、符号推理任务上，相比直接答，准确率得到提升；模型越大，增益越显著  
		- 代价：
			- 链长与推理时间线性增加；链质量（逻辑、数值）直接影响最终答案，需后续校验或投票机制
	- （2）思维链提示的优化策略
		- 思维链示例设计
			- 复杂化：
				- 选推理步骤多或问题长度长的样本做示例；步骤越多 → 子问题越丰富，模型学会拆解复杂链，准确率↑  
			- 多样化：
				- k-means 先聚类问题空间，每簇挑近质心样本→生成 CoT；簇间差异大 → 提供多种推理路径，容错率↑（即使个别链有错，整体仍提升）
			- 经验：复杂 + 多样组合使用，效果 > 单一路线；k 一般 4~8，簇内再按置信度过滤，可降低链错误对最终答案的冲击
		- 思维链生成增强方法
			- 基于采样（Self-Consistency）
				- 温度>0下1个思维链提示生成 M 条推理路径 → 投票选最高频答案；M₁ 条路径 × M₂ 个不同提示可进一步放大多样性
				- 适用：数学、逻辑等有多解路径的任务；简单分类反而可能过思考降分  
			- 基于验证（DIVERSE 为代表）
				- 训练二元验证器：整条路径正例=答案正确；中间步骤正例=与“正确路径”该步一致
				- 推理时每一步先“前瞻”打分，若后续无望则“回溯”换兄弟节点
				- 效果：减少错误累积，24点、谜题类任务准确率↑15–20 %；
				- 代价：需额外验证器与数据标注
		- 拓展推理结构（针对整个思维链结构进行优化）
			- 思维树（Tree of Thought, ToT）
				- 每个节点=一个推理步骤；父节点可扩展多子节点 → 链式变树式
				- 关键：“前瞻打分”预估当前分支成功概率；搜索算法（BFS/DFS + 剪枝）优先走高分路径，提前放弃低分节点 
				- 效果：24 点、创意写作等需探索任务准确率↑，推理步数↓
			- 思维图（Graph of Thought, GoT）
				- 每个节点=一个推理步骤，节点任意连接，支持“合并”操作：不同子树结果可汇总成新中间步骤 → 打破树只能父子回溯的限制
				- 例：长数组排序 → 分 4 段并行排序 → 图边连接各段结果 → 合并节点一次性归并；减少重复排序调用
				- 适用：多路径可汇聚、需反复比对的复杂任务；相比 ToT，搜索空间更灵活，但图搜索与打分策略实现更复杂
	- （3）关于思维链的进一步讨论（思维链有效性来源与扰动实验）
		- 来源：
			- 训练语料存在大量“重叠中间变量”的链式因果；CoT 借这些隐变量完成多步推理，比直接预测两条远变量更准确（贝叶斯网络实验验证）  
		- 组合学习视角：
			- CoT 把复杂推理拆成一种“信息聚焦 + 上下文学习单步”的组合函数的两阶段，降低上下文学习复杂度，标准提示难以学到这种组合函数  
		- 符号 vs 模式：
			- 扰动实验表明：数字/实体具体内容不重要，**算式结构 & 子步骤逻辑**才是关键；甚至示例算式算错，也不显著影响最终准确率  
		- 无 CoT 也能受益：
			- 只要采样路径中**显式出现中间推理文字**，正确答案概率就显著高于无推理路径，说明 CoT **本质**是通过“提高生成含推理路径的文本序列比例”来提高模型生成正确答案的概率
