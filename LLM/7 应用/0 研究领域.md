1. NLP任务中的大语言模型
	- | 任务 | 小模型时代 | 大模型做法 | 短板 |
	  |---|---|---|---|
	  | 序列标注（NER/POS） | BiLSTM-CRF/BERT+CRF | 零样本提示：“请提取实体…”无需 BIO 标记 | 低频/嵌套实体召回低，需小模型初筛+大模型重排 |
	  | 关系抽取 | 管道：实体→分类头 | 上下文学习生成三元组，复杂推理占优 | 关系规模大时上下文塞不下，用小型检索器先过滤候选 |
	  | 文本生成（MT/摘要） | 微调 mBART/PEGASUS | 少样本提示接近商业系统，交互式多轮润色 | 低资源语言（如高棉→英）语料不足，BLEU 仍落后 8-10 分 |
	- 融合建议：
		- 标注充足→小模型微调性价比更高；标注稀缺→大模型零/少样本立即上线
		- 大小模型级联：小模型高速打草稿，大模型做二次校验与长程推理，兼顾效率与效果
2. 信息检索中的大语言模型
	- | 方向 | 做法 | 典型工作 | 效果 |
	  |---|---|---|---|
	  | LLM 做检索/重排 | 检索端：冻结 LLM 输出 CLS 向量；重排端：提示“相关度 1-5”打分 | REP★, RankGPT | MS MARCO MRR@10 +2.3 % |
	  | LLM 生成标注数据 | 让 LLM 为无标注文档写“伪查询”→训练稠密检索器 | InPars, Promptagator | 零标注即达 BM25 95 % 性能 |
	  | 查询改写 | 提示 LLM 把用户原句扩展为 3-5 条同义查询，再并行检索 | HyDE, Q2D | 长尾 query Recall+15 % |
	  | RAG 嵌入优化 | 相关度过滤 + 摘要压缩 + 实体保留，再送入 LLM | FILCO, LongLLMLingua | 减少 30 % token，答案 F1 +4.2 % |
	  | 主动触发检索 | 当 LLM 内部概率 < τ 时自动调用搜索引擎 | Self-RAG, Flamingo | 幻觉率 –18 %，吞吐量几乎不降 |
	- 未来路线：
		- 效率端：检索-生成联合蒸馏，10B 模型替代 175B
		- 决策端：可学习 τ 的“何时查”网络，实现自适应 RAG
		- 长文本端：滑动窗口 + 块级重要性评分，支持 128 k token 级多文档融合
3. 推荐系统中的大语言模型
	- | 场景 | 做法 | 代表 | 效果 | 代价 |
	  |---|---|---|---|---|
	  | 直接推荐器 | 提示：交互历史→文本→“请猜下一物品” | P5, UniCRS | 冷启动↑30 % | 推理延迟 200 ms，难上线 |
	  | 微调排序器 | 指令数据：userID+itemID→偏好标签 | TALLRec, InstructRec | 召回+4 %，DCG+7 % | 需 GPU 周级微调+蒸馏 |
	  | 特征/数据增强 | LLM 写“隐含兴趣”文本→喂入传统 DSSM | KAR, GENRE | AUC+1.8 % | 离线批处理，0 线上成本 |
	  | 用户-物品仿真 | RecAgent 生成百万“伪交互”→训练 MF | RecAgent, AgentCF | 真实场景 A/B 提升 3.5 % | 仅仿真阶段用 7B 模型，部署用 100 M 小模型 |
	- 未来路线
		- 对话式推荐：LLM 即对话引擎，实时问→检索→解释→推荐，一次推理完成；
		- 可解释生成：推荐理由用自然语言即时给出，提升用户信任与 CTR
		- 长序列建模：用 Recency-Weighted Summarization 把 1 k+ 行为压成 512 token，兼顾效率与记忆
4. 多模态大语言模型
	- | 阶段 | 关键操作 | 经验规则 | 代表模型 |
	  |---|---|---|---|
	  | 视觉-语言对齐 | 图文对→只训连接层 | <10 M 图文对别动 LLM | MiniGPT-4、LLaVA |
	  | 视觉指令微调 | 图+指令→生成文本 | 自写 3-5 k 高质量图对话 > 50 k 低质对 | LLaVA-1.5、InstructBLIP |
	  | 端到端预训练 | 文本+图从头混合 | 需 1 B+ 图文对，算力×10 | Gemini、GPT-4V |

	- 评测：
		- MMMU（大学考试）、MMBench（多选）、POPE（物体幻觉）
		- 闭源平均领先开源 15 %，幻觉率仍是硬伤（GPT-4V 约 18 %）
	- 未来路线：
		- 统一预训练而非“对齐”：减少模态间隙
		- 工具调用：图片→分割/检测→LLM 汇总，提升细粒度理解
		- 安全对齐：收集“视觉红队”攻击样本，做指令微调降低幻觉
5. 知识图谱增强的大语言模型
	- | 策略 | 做法 | 代表系统 | 关键技巧 | 效果 |
	  |---|---|---|---|---|
	  | 子图检索 | PageRank 选节点 → 三元组序列化喂 LLM | KAPING, Keqing | 保留 top-k 实体+1-hop 关系，F1 +4.2 % |
	  | 查询交互 | 多轮 SPARQL → 结果拼接 → LLM 推理 | KB-Agent, ToG | 每步用 LLM 规划查询，直至收集足够事实，多跳 QA Acc +7 % |
	  | 冲突消解 | 外部三元组 vs 参数知识投票 | CRES, KDTC | 训练“置信度门”决定相信谁，幻觉率 –12 % |
	  | 统一接口 | 把不同 KG 转成“头-关系-尾”扁平文本 + 特殊 token | UnifiedKG | 一个提示兼容 Freebase/Wikidata/医学 KG |
	- 未来方向
		- 统一 schema：用本体对齐把多源 KG 映射到公共语义层，减少 prompt 长度
		- 知识编辑：参数+图谱双通道更新，实现“实时修正”而不重训
		- 安全对齐：用图谱事实做“硬约束”，RLHF 阶段把违背三元组的生成直接惩罚
