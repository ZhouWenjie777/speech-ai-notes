1. 医疗场景下的大语言模型
	- | 阶段 | 关键动作 | 经验/陷阱 | 代表数据与模型 |
	  |---|---|---|---|
	  | 继续预训练 | 电子病历+医学教材+科研文献，去隐私后 30 B token 继续训练 | 必须屏蔽患者姓名、日期，否则隐私合规直接否决；中文语料不足可加 CMeKG 中文医学百科 | PubMed、MIMIC-III、临床指南、CMeKG |
	  | 指令微调 | 构造“医患对话+诊断+用药+检查”三元组，模板覆盖问诊、分诊、用药建议、报告解读 | 拒绝回答处方剂量，否则监管风险极高；引入“安全拒绝”样本 > 10 % | MedDialog、Chinese-med-dial、MeQSum |
	  | 知识增强 | 对接院内知识库 + 药品说明书，检索增强 RAG；对罕见病直接查 Orphanet | 子图过大→上下文溢出，用摘要+实体保留策略，token↓50 % | DISC-Med、DrKnow |
	  | 多模态扩展 | 影像-文本对：X-ray + 报告，CT + 影像描述，训 V+L 分支 | 影像必须先脱敏去 DICOM 元数据；影像编码器（ViT-Base）冻结，只训 connector，防灾难遗忘 | VQA-RAD、IRV、MURA |
	  | 评测 | 自动化：MedQA、PubMedQA 准确率；人工：医生盲评安全性、准确性、适用性 | 自动化指标≠临床可用，需≥3 名主治打分，< 4/5 分需回炉 | MultiMedQA、CMedQA |
	- 监管红线：
		- 不得直接给出处方剂量或诊断结论，必须加“建议由医生确认”免责声明
		- 上线须通过《医疗器械算法性能评估》备案，提供ROC、敏感性、特异性报告
		- 更新周期≤6 个月，重大版本变更需重新备案
	- 未来方向：
		- 可穿戴实时数据流（心率、血糖）+ LLM 连续决策，实现院外慢病管理
		- 与生物传感器融合，构建“感知-诊断-干预”闭环，向数字疗法（SaMD）升级
2. 教育场景下的大语言模型
	- | 阶段 | 关键动作 | 经验/陷阱 | 代表数据与模型 |
	  |---|---|---|---|
	  | 继续预训练 | 学科教材+题库+论文，去版权，50 B token 继续训练 | 数学公式用 LaTeX 保持，化学式用 SMILES，防符号丢失；剔除 3 年内版权教材 | Khan Academy、arXiv-edu、GeoGebra |
	  | 指令微调 | 专家写“题目→分步讲解”+ GPT-4 仿真师生对话，覆盖小学到 AP 难度 | 拒绝直接给答案，必须输出“思路→步骤→总结”模板，防学生一键抄袭 | TAL-SCQ5K、EduChat-34B、TSCC |
	  | 检索增强 | 对接教材章节知识库，先检索相关概念→再生成讲解 | 同一概念多教材表述差异大，需用“一致性过滤”保留交叉验证段落 | Quizlet、GeoGebra、Wikipedia-Edu |
	  | 多模态扩展 | 几何图、函数图、化学实验图+文字讲解成对训练 | 图用 Asymptote/GeoGebra 矢量导出，防压缩失真；图文必须一一对应 | GeomVerse、ScienceQA |
	  | 评测 | 自动化：教学题库准确率；人工：教师盲评教学正确性、启发性、无偏见 | 题库不能泄露进训练集；必须加“政治/伦理”审核，< 4/5 分需回炉 | C-Eval-Math、MATH、AGIEval-Math |
	- 监管与伦理红线：
		- 不得直接输出选择题答案，必须输出“思考→步骤→提示”三段式
		- 政治、思想品德类题目需人工审核，模型不得输出与课程标准相悖的价值观
		- 平台须记录学生提问与模型回复，用于30天内审计，防止作业抄袭与考试舞弊
	- 未来方向：
		- 对话式辅导：LLM 实时诊断学生错误步骤，给出苏格拉底式追问而非直接答案
		- 个性化难度调节：根据学生实时正确率动态调整题目难度，实现“一对一”自适应学习
		- 与课堂硬件融合：LLM + 智能笔/手写板，实时识别学生手写步骤并纠正
3. 法律场景下的大语言模型
	- | 阶段 | 关键动作 | 经验/陷阱 | 代表数据与模型 |
	  |---|---|---|---|
	  | 继续预训练 | 法律法规、司法解释、裁判文书、法考题，去隐私后 20 B token 继续训练 | 必须剔除涉个人隐私案号、当事人姓名；涉外判例需标注“域外效力”以免混淆 | 中国裁判文书网、国家法律法规库、法考题库 |
	  | 指令微调 | 构造“案情→法条→结论”三元组，模板覆盖咨询、合同审查、判决生成 | 拒绝给出具体量刑年限或赔偿金额，必须加“仅供参考，建议委托律师”免责声明 | ChatLAW、LexiLaw、Cuad |
	  | 知识增强 | 对接民法典、刑法条文库，检索增强 RAG；类案推送先检索相似判例再生成结论 | 法条更新频繁（如新司法解释），需每月同步；类案检索用案由+争议焦点向量，避免案情偏差 | 民法典、刑法、指导案例库 |
	  | 评测 | 自动化：司法考试准确率；人工：律师盲评法条引用正确性、判决合理性、风险等级 | 司考题目不能泄露进训练集；必须加“伦理/隐私”审核，< 4/5 分需回炉 | 中国司法考试真题、LegalBench、CAIL2018 |
	- 监管与伦理红线：
		- 不得直接给出“应当判处 X 年”或“赔偿 X 元”，必须输出“可适用××法条，建议咨询执业律师”
		- 涉未成年人、国家秘密案件数据不得用于训练，需先过滤案由
		- 平台须记录用户提问与模型回复，用于 7×24 h 内审计，防止泄露当事人隐私
	- 未来方向：
		- 类案智能推送：LLM+向量数据库，输入案情自动推送相似度前10指导性案例
		- 合同智能审查：LLM自动标注“高风险条款”并给修改建议，减少律师重复劳动
		- 与区块链存证结合：关键法律文件哈希值上链，确保模型输出可溯、不可篡改
4. 金融场景下的大语言模型
	- | 阶段 | 关键动作 | 经验/陷阱 | 代表数据与模型 |
	  |---|---|---|---|
	  | 继续预训练 | 公司公告、研报、彭博新闻、社交媒体，去隐私后 300 B token 继续训练 | 必须剔除非公开内幕信息、实时股价，防止“未来函数”泄露；中文语料加 Wind、巨潮公告 | BloombergGPT、XuanYuan-70B、FinGPT |
	  | 指令微调 | 构造“公告→情感”“财报→ROE 预测”“新闻→交易信号”三元组，覆盖 5 类 136 k 任务 | 拒绝直接给出“买入/卖出”指令，必须输出“看多因子+风险提示”免责声明 | FTT、FinBen、CFA 真题 |
	  | 工具增强 | 对接 SQL 数据库（财报字段）、Python 解释器（技术指标）、时间序列模型（LSTM） | 数值计算用 4-digit 分词+计算器 API，防止小数累加误差；SQL 接口限只读，防注入 | FinSQL、GTE-Fin、Pandas-AI |
	  | 评测 | 自动化：FinBen 三级任务准确率；人工：持牌分析师盲评交易逻辑、风险披露完整性 | 交易决策任务不能用历史未来函数；必须加“伦理/合规”审核，< 4/5 分需回炉 | FinBen、CFA Level-III 真题、Flare-Fin |
	- 监管与合规红线：
		- 不得直接给出“建议买入××股票”或“目标价××元”，必须输出“看多因子+风险提示+仅供参考”
		- 不能使用未公开财务数据或内幕信息训练，需通过数据审计确认“未来函数”已剔除
		- 模型输出需留痕 5 年以上，接受证监会、交易所抽查
	- 未来方向：
		- 实时事件驱动：
			- LLM+新闻流NLP，秒级生成“事件-标的-影响”摘要，辅助量化策略
		- 多模态财报：
			- 同步解析PDF表格、业绩电话会音频、CEO 表情视频，做多模态情绪打分
		- 与监管沙盒对接：在合规沙盒内回测策略，确保不触及市场操纵红线
5. 科研场景下的大语言模型
	- | 阶段 | 关键动作 | 经验/陷阱 | 代表数据与模型 |
	  |---|---|---|---|
	  | 继续预训练 | arXiv + PubMed + 论文补充材料，共 50 B token；公式转 LaTeX，表格转 Markdown，蛋白质/分子用专用 token | 必须剔除预印本中未通过同行评审的“争议性结论”，防止模型学到错误事实；数学式子用 MathJax 统一格式 | Galactica-120B、SciGLM、arXiv-20B |
	  | 指令微调 | 构造“定理→证明”“实验→结果解释”“习题→分步解析”三元组，覆盖 8 大学科 200 k 条 | 数值计算题必须外挂 Python/SymPy 解释器，防止小数/符号积分错误；证明题需验证 Lean 4 语法通过 | SciInstruct、ScienceQA、SciGLM-22B |
	  | 工具增强 | 对接 CAS（Mathematica）、分子动力学（OpenMM）、绘图（matplotlib）API，实现“对话→代码→执行→结果”闭环 | 代码执行沙箱隔离，禁止网络外联；计算结果>1 s 超时即回退到近似解 | ChatGLM-Math、ChemDFM、SciAgents |
	  | 评测 | 自动化：Sci-Bench（大学理、化、生 789 题）、MATH 准确率；人工：研究员盲评逻辑严谨性、公式正确性 | 几何题需图片+文字双模态输入；开放题需≥2 名博士打分，< 4/5 分需回炉 | Sci-Bench、SciEval、MATH、MiniF2F |
	- 监管与伦理红线：
		- 不得直接给出“证明××猜想成立”或“实验结果绝对正确”，必须输出“推导步骤+验证代码+仅供参考”
		- 使用未公开实验数据训练需通过伦理审查，防止数据造假进入模型
		- 模型输出需留痕 10 年以上，接受期刊/基金委抽查
	- 未来方向：
		- 自动定理证明：LLM+Lean 4交互，实现“语言猜想→形式化证明→机器验证”
		- 多模态实验报告：同步解析论文图、表、正文，生成可复现的实验代码与数据
		- 与实验设备对接：LLM 直接控制机械臂、光谱仪，实现“对话式实验”闭环
