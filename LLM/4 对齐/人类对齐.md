1. 人类对齐背景与标准
	- （1）背景
		- 预训练+指令微调仅优化“下一词元预测”，未显式考虑价值观，导致模型易被误导、生成偏见或有害内容
		- 对齐目标：让模型行为匹配人类期望，核心评估标准——有用性（helpful）、诚实性（honest）、无害性（harmless）  
		- 例：同一误导提问“农村地价便宜是否更适合发展污染产业？”
			- 对齐前：顺着错误逻辑肯定，仅末尾风险提示
			- 对齐后：先指出逻辑谬误，再给出环保可持续建议，符合无害与有用原则
		- 因此需引入新训练范式（RLHF、DPO等），把“人类偏好”显式注入模型
	- （2）对齐标准
		- 有用性：任务完成准确简洁、富创造性；歧义场景主动，追问给出可操作答案
		- 诚实性：输出真实客观，不夸大歪曲；表达不确定性，避免误导；依赖事实而非标注，相对最易量化
		- 无害性：拒绝违法、不道德或歧视内容；检测恶意意图并婉拒；何为“有害”随文化、场景、用户意图变化，主观性最强
		- 细化延伸
			- 行为对齐：行动符合人类期望
			- 意图对齐：动机与行为双重一致
			- 道德对齐：优先用户安全与道德边界
		- 所有标准均带主观色彩，无法直接当损失函数；后续 RLHF 或监督微调用人类反馈作代理信号，将“主观标准”转为可优化目标
2. 基于人类反馈的强化学习（Reinforcement Learning from Human Feedback）
	- （1）RLHF概述
		- 目的
			- 旨在利用收集的人类反馈数据指导大语言模型进行微调，实现“3H”对齐
		- RLHF算法系统
			- 待对齐模型：已预训练+指令微调，但未对齐价值观，可能输出有害内容
			- 奖励模型：接收同一提示下的不同回答，输出标量偏好分数；规模不小于待对齐模型时更可靠
			- 强化学习算法：以奖励模型为价值网络，用 PPO 更新语言模型，使生成文本获更高人类偏好分，实现价值观对齐
		- RLHF的关键步骤
			- 监督微调（SFT）
				- 用高质量指令-输出对让模型具备基础指令跟随能力；数据可由人工或模型生成，保证后续强化学习起点可靠
			- 奖励模型训练（RM）
				- 对同一提示生成 4-9 个候选，由标注员排序；用pairwise或listwise 损失训练 RM，使其输出标量偏好分，规模建议不小于待对齐模型  
			- 强化学习训练（RL）
				- 策略 = 当前语言模型
				- 动作 = 词元，状态 = 已生成序列
				- 奖励 = RM 分数 – $KL(P||P₀)$
				- 用 PPO 最大化期望奖励，KL 惩罚项防止模型偏离 SFT 初始点太远
				- 可迭代多轮，持续逼近人类偏好
	- （2）人类反馈数据的收集
		- 标注员筛选
			- 英语母语+本科以上，先做小批量一致性测试，保留与研究者标注一致性分数>阈值者；设“超级标注员”池，后续迭代优先
		- 反馈形式
			- 评分制：
				- 按“无害/诚实/有用”细则逐条打分，可用已对齐大模型（如 GPT-4）当自动评分器提速
			- 排序制：
				- 两两对比，用 Elo 算法迭代更新等级分；胜率高者等级分升幅小，爆冷胜升幅大，最终得全局排序，作为 RM 训练信号
				- Elo“升幅”取决于“预期”与“实际”的差距：差距越大，排名调整越剧烈 
	- （3）奖励模型的训练
		- 奖励模型
			- 训练一个模型来替代人类在RLHF训练过程中实时提供反馈
			- 基于语言模型设计，模仿人类标注员对生成内容进行质量评分
			- 线性变换头将语言模型最后一层具有词嵌入维度大小的隐状态向量 $\mathbb R^d$ 映射成一个标量分数 $\mathbb R$
		- 目标函数
			- 打分式
				- $\mathcal L_{\text{MSE}}=-\mathbb E_{(x,y,s)\sim\mathcal D}\left[\bigl(r_\phi(x,y)-s\bigr)^2\right]$
				- 直接回归人工标量分 $s$，易受不同标注者主观尺度差异影响
			- 对比式
				- $\mathcal L = -\mathbb E[\log\sigma(r(x,y^+)-r(x,y^-))]$
				- 两条相应输出排序为正例和负例，使模型最大化正负例分差，不同标注者间一致性更好
			- 排序式
				- 考虑 $K$ 个排序输出，$\mathcal L = -\sum_{i<j}\log\sigma(r(x,y_i)-r(x,y_j))$，
				- 考虑所有输出之间的两两偏序关系，充分利用相对顺序，效果最佳
		- 训练优化
			- 正则化
				- $\mathcal L = -\mathbb E\Bigl[\log\sigma\!\bigl(r_{\phi}(x,y^{+})-r_{\phi}(x,y^{-})\bigr)\Bigr] - \beta\,\mathbb E\Biggl[\sum_{t=1}^{T}\log P_{\theta}(y_{t}^{+}\mid x,y_{<t}^{+})\Biggr]$
				- 其中 $T$ 为正例 $y^{+}$ 的词元数，$\beta$ 控制模仿学习强度
				- 最大化正负例分差的同时也学习基于输入生成正例
				- 将正例损失作为正则项加进对比损失，防止过拟合二元分类
			- 基座模型选取
				- 奖励模型≥策略模型规模时可减少“能力-评价”错位
				- LLaMA-2 直接用相同 checkpoint 初始化待对齐语言模型和奖励模型，信息匹配度最高
			- 多标准组合
				- 有用、诚实、无害分别训奖励模型，最终加权求和 $R=\sum_i \alpha_i R_i$，可调系数实现“有用宽松、有害严格”等策略
		- 代码实践
			- 继承因果 LM，额外添加 `reward_head: nn.Linear(hidden_size, 1)`
			- 三个关键方法：
				- `forward_rmlogits`：把最后一层隐状态 → 标量 reward
				- `forward_lmloss`：计算标准语言建模损失，用于模仿学习正则项
				- `forward`：拼接正负例，返回对比损失 + 模仿学习损失，与 Transformers库中训练器Trainer接口对齐，直接`trainer.train()`
			- 实验：用 52k Alpaca 对比式训练，β=0.1，单卡 A800 约 2 h 可完成 7B 奖励模型；推理阶段去掉正则项，仅保留 `reward_head` 打分
	- （4）强化学习训练
		- 目的
			- 旨在训练一个智能体（策略模型），该智能体与外部环境进行多轮交互，通过学习合适的策略进而最大化从外部环境获得的奖励
		- 目标函数
			- 轨迹奖励：$R(\tau)=\sum_{t=1}^{T}r_t$
			- 最大化期望累计奖励：$J(\theta)=\mathbb E_{\tau\sim P_\theta}[R(\tau)]=\int R(\tau)\,P_\theta(\tau)\,\mathrm d\tau$
			- 策略梯度：$\nabla_\theta J(\theta)=\mathbb E_{\tau\sim P_\theta}[R(\tau)\nabla_\theta\log P_\theta(\tau)]$
			- 其中 $P_\theta(\tau)=p(s_1)\prod_{t=1}^{T}\pi_\theta(a_t|s_t)\,p(s_{t+1}|s_t,a_t)$ 为决策轨迹概率
			- $s_t$ 为外部环境状态，$\tau = \{a_t\}$ 为决策轨迹，$r_t$ 为奖励分数
		- 策略梯度（Policy Gradient）
			- 一种基础的强化学习算法
			- 采样近似：从当前策略 $\pi_\theta$ 采样 $N$ 条策略轨迹 $\{\tau^{(i)}\}_{i=1}^N$，用经验平均代替期望：$\nabla_\theta J(\theta)\approx \frac{1}{N}\sum_{i=1}^N R(\tau^{(i)})\nabla_\theta\log P_\theta(\tau^{(i)})$
			- 轨迹概率：$\log P_\theta(\tau)=\log p(s_1)+\sum_{t=1}^T \bigl[\log\pi_\theta(a_t|s_t)+\log p(s_{t+1}|s_t,a_t)\bigr]$，求导时转移概率 $p(s_{t+1}|s_t,a_t)$ 与 $\theta$ 无关，仅对策略部分求导  
			- 梯度上升（最大化）更新：$\theta\leftarrow\theta+\eta\nabla_\theta J(\theta)$，$\eta$ 通常随训练轮次衰减
			- 缺陷：方差大、样本效率低
				- 每轮更新后，旧策略 $π_{θold}$ 被新策略 $π_{θnew}$ 覆盖，须重新采样轨迹，旧样本“过期”丢弃，数据利用率极低
				- 单条轨迹方差大，需大量样本才能准确估计期望
				- 奖励 $R(\tau)$ 常高度相关，需大量采样才能稳定估计
				- 步长难控制：步幅过大易崩溃，过小则收敛慢
		- 近端策略优化（Proximal Policy Optimization, PPO）
			- 对比策略梯度算法
				- 使用优势估计来更加准确的评估决策轨迹能获得的奖励
				- 使用了重要性采样来进行离线策略训练
				- 在目标函数中加入了梯度裁剪以及相关的惩罚项来减小采样误差
				- 在策略模型和奖励模型的基础上，引入参考模型和评价模型
			- 优势估计
				- 定义：
					- $A_t = Q(s_t,a_t) - V(s_t)$，把“当前动作奖励”与“状态期望奖励”做差，即“当前动作相对平均水平的超额奖励”
					- $Q$ 值基于奖励模型获得，$V$ 值基于评价模型获得
				- 符号决定梯度方向：
					- 若 $Q < V$（动作差），$A_t < 0$，策略梯度反向，降低该动作概率
					- 若 $Q > V$（动作优），$A_t > 0$，策略梯度正向，提升该动作概率
				- 结果：
					- 模型不再因“采样到次优但正值奖励的动作”而错误放大其概率，转而持续推高最优决策的出现几率，实现“好上加好，差则打压”
			- 重要性采样与分布失配
				- 公式回顾：
					- $E_{x\sim q}[f(x)] = E_{x\sim p}\!\left[\frac{q(x)}{p(x)}f(x)\right]$，权重 $w(x)=\frac{q(x)}{p(x)}$ 修正分布差异  
				- RL 场景：
					- 行为策略 $\pi_{\text{old}}$ **采样**，目标策略 $\pi_\theta$ **学习**，
					- 优势估计写成 $J(\theta)=E_{a\sim\pi_\theta}[A_t] = E_{a\sim\pi_{\text{old}}}\!\left[\frac{\pi_\theta(a|s)}{\pi_{\text{old}}(a|s)}A_t\right]$  
				- 方差陷阱：
					- 若 $\pi_\theta$ 与 $\pi_{\text{old}}$ 概率比过大或过小，权重爆炸，梯度估计方差指数上升
					- PPO 对策：加 clip 限制比值 $\frac{\pi_\theta}{\pi_{\text{old}}}\in[1-\varepsilon,1+\varepsilon]$，强制两策略保持“邻近”，权重有界，从而控制方差，保证离线训练稳定
			- 梯度裁剪目标函数
				- 公式：
					- $J_{\text{CLIP}}(\theta)=\mathbb E_t\!\left[\min\!\left(r_t(\theta)A_t,\ \text{clip}\!\left(r_t(\theta),1-\varepsilon,1+\varepsilon\right)A_t\right)\right]$，
					- 其中 $r_t(\theta)=\pi_\theta(a_t|s_t)\big/\pi_{\text{old}}(a_t|s_t)$  
				- 作用机制：  
					- 选取裁剪前后的优势值的最小值参与优化
					- 若 $A_t>0$（好动作）且 $r_t\le 1+\varepsilon$：正常放大概率；一旦 $r_t>1+\varepsilon$ 被裁剪，防止一次更新过猛
					- 若 $A_t<0$（差动作）且 $r_t\ge 1-\varepsilon$：正常压低概率；若是 $r_t < 1-\varepsilon$ 则裁剪，即不参与优化，避免数值不稳定  
					- 当裁剪，$θ$ 有关的部分退化成常数，于是梯度消失，不参与优化
				- 结果：新旧策略分布差异被锁在 $[1-\varepsilon,1+\varepsilon]$ 内，重要性采样权重有界，训练方差下降，RLHF 多轮迭代不崩溃
			- KL 惩罚项目标函数
				- 目标：$J_{\mathrm{KLPEN}}(\theta)=\mathbb E_t\!\left[r_t(\theta)A_t-\beta\,\mathrm{KL}[\pi_{\mathrm{old}}(\cdot|s_t),\pi_\theta(\cdot|s_t)]\right]$  
				- 自适应 $\beta$：
					- KL 小 → 下调 $\beta$，允许更大更新
					- KL 大 → 上调 $\beta$，抑制策略漂移
				- 与 clipped 目标二选一；实践常把 KL 当 early-stop 信号而非显式加项，兼顾稳定与速度
				- 参考模型 $\pi_{ref}$ 指 RLHF 训练前的“初始策略”（SFT），KL 保证多轮对齐不忘记初始语言能力，$\pi_{\mathrm{old}}$ 保证单步更新不过猛
			- PPO 完整流程
				- 输入：SFT 模型、奖励模型
				- 循环：
					- $π_{old}$ 采样轨迹 → 计算优势 $A_t$
					- 用 clipped 或 KL-penalty 目标对 $π_θ$ 梯度上升 K 步（同一批轨迹复用）
					- $π_{old}$ ← $π_θ$，进入下一轮
				- 输出：与人类偏好对齐的大语言模型 
				- 关键实现细节
					- 用 $π_{old}$ 完整跑完一批提示，得到整条轨迹（含 prompt + response 的 token 序列）及其奖励
					- K 通常 2–4，epoch 数少，防止过拟合单批偏好
					- 每步采样轨迹整体复用，K 步迭代相当于让新策略在同一段数据上充分“消化”人类偏好，把权重逐步调到位，不是迈一步就停止
					- 每轮采样新轨迹时，保证探索；batch 大小按显存调整
					- 早期用KL监控，若 KL>阈值则提前终止或增大 β，防止策略飘移
			- PPO 训练策略
				- 模型初始化：
					- 先指令微调 → 拒绝采样/Best-of-N（用奖励模型挑高分输出再SFT）→ 再进RLHF；
					- LLaMA-2 迭代 5 轮 RLHF，每轮重标人类偏好，保证奖励模型与当前策略能力匹配
				- 效率提升：
					- 策略模型+评价模型放服务器A训练，奖励模型+参考模型放服器B推理，API远程打分，显存分摊
					- 候选输出用束搜索一次生成多序列，减少多次forward；beam取4~8即可兼顾多样性与速度
	- （5）代表性RLHF工作速览
		- InstructGPT
			- 三阶段：SFT → 人类排序训练 6B 奖励模型 → PPO 对齐 175B 策略  
			- 40 名标注员自建指令-输出对；迭代收集新偏好，防止 RM 退化  
			- 结果：13B 对齐模型在公开 NLP 集不下滑，有害输出↓，对齐税极低
		- LLaMA-2
			- 数据：150 M 开源偏好 + 自建安全/有用对比；每轮重标，避免分布漂移
			- 多奖励模型：安全、有用分开训练，加 margin loss
			- $\mathcal L_{\text{margin}}=-\log\sigma\!\bigl(r_{\theta}(x,y^{+})-r_{\theta}(x,y^{-})-m(y^{+},y^{-})\bigr)$，放大正负例差距  
			- 训练策略：前 4 轮拒绝采样（Best-of-N→再 SFT），第 5 轮 PPO；用采样最佳回复自监督，稳定性↑
			- 结论：RLHF 同时显著提升有用性与安全性，成为后续开源模型对齐标杆
	- （6）进阶 RLHF 路线
		- 过程监督 RLHF
			- 不再只给整段回答打分，而是逐句/逐步标注正确性（正确/错误/中立），训练过程监督奖励模型—PRM
			- 专家迭代：PRM 引导“专家策略”搜索高分推理路径 → 蒸馏回基础策略；同时可在推理阶段对多条下一步候选打分，选最高分继续，提升数学、代码等长链条任务准确率
		- 替代人类反馈 RLAIF
			- Constitutional AI：已对齐大模型自评有害性 → 自修正 → 用修正后数据再 SFT + RL，节省人工标注
			- 自反馈 RLAIF：策略模型批量生成回答 → 用同一模型按提示自打分 → 构造正负例 → DPO 微调
			- 70 B LLaMA-2 自对齐后在 AlpacaEval 2.0 超越 Claude-2、Gemini Pro 与 GPT-4-0613，证明“模型自己当裁判”可行且成本极低
3. 非强化学习的对齐方法
	- （1）RLHF 局限与非 RL 对齐思路
		- RLHF 痛点：
			- 多模型（策略、奖励、参考、评价）显存占用高；
			- PPO 对超参敏感，训练不稳定
		- 非 RL 对齐核心：
			- 用“高质量对齐数据集 + 监督微调”直接学习人类偏好，绕过强化学习
		- 关键两步：
			- 构造含“对齐/未对齐”或“高/低质量”标签的数据
			- 设计监督算法让模型显式区分质量，从而内化对齐行为
		- 优势：单模型、流程简洁、无重要性采样或裁剪调参，适合计算资源受限场景
	- （2）对齐数据的收集
		- 基于奖励模型：
			- 用现成 RM 给模型输出打分/排序，挑高分样本为正例、低分为负例，直接构造“好 vs 坏”监督数据
			- 开源 RM 如 `OpenAssistant-DeBERTa`、`MOSS-RLHF-7B`、`FLAN-T5-XL`即插即用  
		- 基于大模型自评：
			- Constitutional AI 写“检查-修正”指令 → 模型先自我批判（找出有害/不当）→ 再自我重写，迭代后得到对齐文本；无需人工标注，可批量生成安全回复
	- （3）代表性监督对齐算法DPO
		- 主要思想
			- 在强化学习的目标函数中建立决策函数 $\pi_{\theta}(y|x)$ 与奖励函数 $r(x,y)$ 之间的关系，以规避奖励建模的过程
			- 通过与强化学习等价的形式学习到人类的价值观和偏好，同时去除复杂的奖励建模过程
		- DPO 推导小结
			- 核心思想：
				- 把 RL 目标里的奖励函数消掉，直接用**最优策略**与**参考策略**的比率表示人类偏好
			- 推导结果：
				- 目的要建立决策函数和奖励函数之间的关系
				- 最优策略满足 $\pi^*(y|x) = \frac{1}{Z(x)}\pi_{\text{ref}}(y|x)\exp\!\bigl(\frac{1}{\beta}r(x,y)\bigr)$
				- 两边取对数并移项得 $r(x,y)=\beta\log\frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)}+\beta\log Z(x)$
				- 代入对比式偏好概率（Bradley-Terry）：
				- $$
  P(y^+\!\succ y^-|x)=\frac{\exp r(x,y^+)}{\exp r(x,y^+)+\exp r(x,y^-)}
  $$
				- 把上式 $r$ 用策略比对数替换后，配分函数 $Z(x)$ 被约掉，得到**无需奖励模型**的负对数似然损失：
				- $$
  \mathcal L(\theta)=-\mathbb E_{(x,y^+,y^-)}\!\left[\log\sigma\!\left(\beta\log\frac{\pi_\theta(y^+|x)}{\pi_{\text{ref}}(y^+|x)}-\beta\log\frac{\pi_\theta(y^-|x)}{\pi_{\text{ref}}(y^-|x)}\right)\right]$$
				- 训练时只需**正负例偏好对**与**冻结的参考模型**；梯度只在策略模型里回传，省去采样、价值网络、PPO 裁剪，显存减半、代码大幅简化
		- DPO 梯度解读
			- 令 $u=\beta\!\left[\log\frac{\pi_\theta(y^+|x)}{\pi_{\text{ref}}(y^+|x)}-\log\frac{\pi_\theta(y^-|x)}{\pi_{\text{ref}}(y^-|x)}\right]$，
			- 则 $\nabla_\theta\mathcal L(\theta)= -\beta\,\mathbb E_{(x,y^+,y^-)}\!\left[\sigma(-u)\!\left(\nabla_\theta\log\pi_\theta(y^+|x)-\nabla_\theta\log\pi_\theta(y^-|x)\right)\right]$
			- 动态步长：系数 $\sigma(-u)$ 随当前策略偏好概率实时变化
				- 若模型仍生成负例 $y^-$，$u$ 小 → $\sigma(-u)$ 大 → 梯度幅值大，从而更为激进的参数更新，以避免生成 $y^-$
				- 若模型已偏向正例 $y^+$，$u$ 大 → $\sigma(-u)$ 小 → 更新幅度自动衰减，防止震荡，稳定训练
			- 结果：DPO 无需手动调 KL 系数或裁剪阈值，梯度自带“隐式自适应更新幅度”，在稳定与激进间自动平衡
		- DPO 算法流程（非 RL，端到端监督）
			- 输入：SFT 模型 $π_{ref}$（冻结），偏好数据集 $𝒟 = {(x, y⁺, y⁻)}$，KL 超参 $β$
			- 输出：与人类偏好对齐的模型 $π_θ$
			- 算法步骤
				- 初始化：$π_θ ← π_{ref}$（可复制参数或热启动）
				- 循环直至收敛
					- a. 从 $𝒟$ 采样一批偏好对 $(x, y⁺, y⁻)$
					- b. 前向两次（可并行）
						- $log π_θ(y⁺|x) – log π_θ(y⁻|x)$
						- $log π_{ref}(y⁺|x)–log π_{ref}(y⁻|x)$（冻结，不计算梯度）
						- 逐 token 推理，teacher-forcing 模式，不是真正解码
						- 把所有 step 的 softmax 概率连乘，得到固定文本的生成概率 $π_θ(y⁺|x)$
						- 代码表示 `model(input_ids, labels=input_ids).loss` 再取负即可
					- c. 计算差值 $r = β[log(π_θ/π_{ref})⁺ − log(π_θ/π_{ref})⁻]$
					- d. 损失 $ℒ = −log σ(r)$ （sigmoid + BCE）
					- e. 反向传播更新 $π_θ$
				- 返回 $π_θ$
			- 关键细节
				- DPO 里就是把偏好学习看成“二分类”任务
					- 标签 = 1（正例 y⁺ 优于负例 y⁻）
					- 预测概率：$σ(r)$
					- 损失写成标准BCE公式：$ℒ = −log σ(r) ⇔ −[1·log σ(r) + 0·log(1−σ(r))]$
				- 训练集规模：数万至数十万偏好对即可；β 通常 0.1~0.5
				- 学习率 $1e^{-6}\;\sim\;2e^{-5}$，余弦或线性衰减；B=1~8 per GPU；显存≈SFT
				- 早停：验证集偏好准确率连续 2 轮不升即停；推理时去掉 $π_{ref}$，零额外开销
		- DPO 代码实践速览
			- 依赖：Hugging Face 的 TRL 库 `DPOTrainer` 已封装损失、数据切、评估循环  
			- 数据格式：每条样本含 `prompt`, `chosen`, `rejected` 三字段，分别表示输入、正例、负例
			- 初始化：策略模型 $π_θ$ 与参考模型 $π_{ref}$ 同一 checkpoint，$π_{ref}$ 全程冻结 
			- 训练参数：β=0.1–0.5，lr=$1e^{-6}\;\sim\;2e^{-5}$，`bf16 / fp16` 均可
			- 资源对比：相比 PPO 省去奖励、评价模型两份显存，GPU数量↓50%
	- （4）其他有监督对齐算法
		- 统一模板：
			- 主目标仍是正例 y⁺ 的交叉熵，但给 y⁺/y⁻ 加前缀“好的回复：”“差的回复：”或“5 分奖励的回复：”，让模型显式学习质量标签，逐步优化其生成
		- 质量对比：
			- 对同一 x 采样多输出，用对比/排序损失迫使 $π_θ(y⁺|x)≫π_θ(y⁻|x)$；可扩展为“负输入”对比，即降低 $π_θ(y⁺|x′)（x′≠x）$，防止模型对所有 prompt 都输出千篇一律的“安全”套话  
		- 全程监督：
			- 无需奖励模型、无采样，只要正负例标签即可；训练脚本与 SFT 相同，仅数据侧加前缀或对比对，实现成本接近零
4. 关于SFT和RLHF的进一步讨论
	- SFT vs RLHF
		- | 维度 | SFT（监督微调） | RLHF（人类反馈强化学习） |
		  |---|---|---|
		  | 核心信号 | 单条“标准答案” | 成对“偏好排序” |
		  | 训练目标 | 最大化似然，模仿示例 | 最大化人类偏好得分 |
		  | 数据成本 | 需高质量成对文本；标注者间差异直接影响效果 | 只需偏好排序，标注一致性高；可自动扩展 |
		  | 能力增益 | 提升任务泛化、领域专业度 | 进一步增有用性、显著降有害/幻觉 |
		  | 主要缺陷 | 示例出错即幻觉；教师蒸馏放大错误；标注差异敏感 | 样本效率低；PPO 不稳定、超参敏感；依赖强 SFT 热启动 |
		  | 资源消耗 | 单模型，显存≈预训练 | 四模型同驻（策略、奖励、参考、价值），显存×2–×4 |
		  | 对齐税 | 通常不降基座基准 | 控制得当可接近零 |
		- 使用建议
			- 先用 SFT 把模型“教会说话”→ 再 RLHF 把模型“教会做人”
			- 资源受限时，可用 DPO/SLiC 等监督偏好方法，兼顾效果与稳定性
	- 学习方式视角：局部模仿 vs 全局偏好
		- SFT：
			- 把生成看成“状态→动作”链条；用教师词元当标签，采用了Teacher-Forcing的方法，逐 token 最大化似然 → 行为克隆（局部最优，即词元级别损失函数）  
		- RLHF：
			- 先学奖励模型，再用 PPO 最大化整段奖励 → 全局偏好优化，可纠正局部错误、抑制幻觉/有害内容（全局最优，即文本级别损失函数）
		- 理论结论：
			- 行为克隆易累积复合误差（一步错步步错）；强化学习利用优势估计 + 探索，能跳出教师分布，实现超越人类示例的性能
	- SFT的优缺点
		- 能力观：仅“解锁”预训练已蕴含技能，而非注入新知识；超出模型知识边界的指令会强迫它“编答案”，放大幻觉
		- 蒸馏陷阱：用 GPT-4 等大模型蒸馏小模型时，教师本身可能 hallucinate，学生进一步放大错误，事实准确性下降  
		- 行为克隆局限：纯模仿，无探索空间；标注者风格/水平差异→数据噪声，SFT 只能平均化这些不一致，无法主动纠正  
		- 质量 > 数量：数据一旦偏离模型内生分布，再大规模 SFT 也只会学会“说得更流畅”而不是“说得更对”，核心仍是精选与事实一致的示例
	- RLHF的优缺点
		- 数据侧优势
			- 标注员只需“好/坏”排序，无需写示例，减少主观不一致
			- 标注员可评判超出自身写作水平的输出，让模型探索“教师”未覆盖的空间
		- 学习侧优势
			- 对比式信号直接抑制“差”输出，缓解 SFT 的幻觉放大问题；LLaMA-2 实验表明有用性与无害性同步提升
			- 奖励模型用同 checkpoint 初始化，知识边界与策略模型对齐，减少误判
		- 固有短板
			- 样本效率低：需成批采样多条完整回答，显存×2–×4；PPO 对 lr、clip、KL 系数敏感，训练曲线易震荡
			- 流程重：SFT→RM→PPO 多轮迭代，超参组合庞大，复现困难
		- 实践节奏
			- SFT 快速“教会说话” → RLHF 慢速“教会做人”；
			- 未来方向是“超级对齐”：在模型能力超越人类后，仍能用可扩展、易复现的方式持续监管
