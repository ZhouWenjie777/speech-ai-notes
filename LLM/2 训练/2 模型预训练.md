1. 预训练任务
	- （1）语言建模
		- 标准语言建模 LM：
			- 自回归预测下一词元，损失 $\mathcal L=-\sum_t\log P(u_t\mid u_{<t})$；大规模数据下隐式多任务学习，蕴含情感、推理等能力
		- 前缀语言建模 Prefix-LM：
			- 随机切分序列 → 前缀双向注意力 + 后缀因果注意力，仅后缀损失 $\mathcal L=-\sum_{t=s+1}^T\log P(u_t\mid u_{\le s},u_{<t})$；适配前缀解码器架构
		- 中间填充 FIM：
			- 重排为\[前缀\]\[后缀\]\[中间\]，目标 $\mathcal L=-\log P(\text{middle}\mid\text{prefix})-\log P(\text{suffix}\mid\text{prefix})-\log P(\text{middle}\mid\text{prefix+suffix})$；兼顾代码补全，保持下一词预测能力
	- （2）去噪自编码 DAE
		- 损坏（替换/删除）文本 $\hat{\mathbf u}$ → 重建原始片段 $\mathbf u$，损失 $\mathcal L=-\log P(\mathbf u\mid\hat{\mathbf u})$；需设计掩码率、片段长度等，FLAN-T5 代表
	- （3）混合去噪器：又称UL2损失，统一视角，三类前缀标记
		- S-去噪器：前缀语言建模损失
		- R-去噪器：短片段（3-5 token）15% 掩码
		- X-去噪器：长片段（≥12 token）或 50% 高掩码
		- 输入首 token \[S\]/\[R\]/\[X\] 指示任务类型，UL2、PaLM-2 采用
2. 优化参数设置
	- 基于批次数据的训练
		- 批次大小动态升温 32 k → 3.2 M token，先小批次高噪声快速下降损失，后大批次平稳收敛模型
	- 学习率
		- 预热阶段：占训练步骤的 0.1 %–0.5 % 步数，线性升至峰值 $5\times10^{-5}$~$1\times10^{-4}$
		- 衰减阶段：线性、余弦或平方根倒数衰减，终值≈峰值 10 %
	- 优化器
		- `Adam/AdamW`为主，PaLM 用 `Adafactor`；精度 `BF16` 主流，FP16 仍存；梯度裁剪 1.0，权重衰减 0.1
		- `Adam/AdamW`：β₁=0.9，β₂=0.95，ε=1×10⁻⁸；
		- `Adafactor`：省显存，β₁=0.9，β₂=1.0-k⁻⁰·⁸，k 为训练步数
	- 稳定优化技术
		- 梯度裁剪：阈值 1.0，防损失尖峰
		- 训练恢复：定期存档，异常时回滚并跳过问题数据
		- 权重衰减：`AdamW`权重衰减系数 0.1，正则化技术稳定训练
		- Dropout：大模型因数据足 + LN，几乎不用
3. 可扩展的训练技术
	- 3D并行训练
		- 数据并行 DP
			- 模型副本放多卡，批次分片，梯度同步，等效大batch
			- 易扩展，框架内置
		- 流水线并行 PP
			- 层分段放不同卡，前向-后向串行致 GPU 空闲
			- 配合梯度累积，卡内连续跑多 micro-batch，提升吞吐
		- 张量并行 TP
			- 把单层矩阵按列/行切块分卡，并行乘后跨卡归约
			- 常见切 Attention $\mathbf W_Q/\mathbf W_K/\mathbf W_V$ 与 FFN $\mathbf W_1/\mathbf W_2$
			- Megatron-LM已集成
	- 零冗余优化器
		- ZeRO 优化器核心思想：数据并行下消除模型/优化器副本冗余，每张 GPU 只存部分参数与状态
		- 运行时按需跨卡通信取参，用完即释放显存，支持更大 batch & 更大模型
		- 三级策略：
			- ZeRO-1：仅分片优化器状态，如一阶动量 m（梯度的指数滑动平均）和二阶动量 v（梯度平方的指数滑动平均）
			- ZeRO-2：再分片梯度
			- ZeRO-3：再分片参数，显存需求最低
		- PyTorch 对应实现：完全分片数据并行 FSDP（Fully Sharded Data Parallel），API 兼容 DDP，已集成
	- 激活重计算（梯度检查点）
		- 一种用于优化反向传播时显存占用（计算参数梯度的激活值）的技术
		- 前向只存每层输入，反向时重新计算激活，显存 ↓50 %–70 %，额外计算 ≈20 %  
	- 混合精度训练
		- 主参数 FP32 副本，前向/反向用 FP16/BF16，梯度缩放防下溢，更新回 FP32
		- FP16：1位符号位、5位指数位、10位尾数位，表示范围为−65504到65504
		- BF16：1位符号位、8位指数位、7位尾数位，表示范围可以达到$10^{38}$数量级
		- BF16 指数 8 bit，范围 ≈$1e38$，优于 FP16，主流GPU如 A100 原生支持，吞吐 ↑≈2×，显存 ↓≈50 %
4. 模型参数量计算与效率分析
	- 参数量计算（Llama 例）
		- 总参数量：$2VH + H + L(4H^2 + 3HH' + 2H)$  
		- 其中，$V$：词表大小 $L$：层数 $H$：隐藏维 $H'$：FFN 中间维  
		- 7B 实例：$V{=}32\,000,\; L{=}32,\; H{=}4\,096,\; H'{=}11\,008$；参数量$6\,738\,415\,616$，与官方 7B 完全一致
	- 训练运算量估计
		- 注意力部分：$\text{FLOP}=12CL(H+N)$，$C{=}BT$，$H{=}ND$，占比小（$T\lesssim H$ 时 $<$ 线性 1/6）
		- 线性部分：
			- 参数量为 P 的模型在 C 个词元上进行预训练的总体运算量
			- 无重计算：$\text{FLOP}=6C\times\text{线性参数量}\approx 6CP$（线性参数量占 95 %）
			- 有重计算：$\text{FLOP}=8CP$ 
		- 示例：Llama-7B，$P{=}6.74\times10^9$，$C{=}1\times10^{12}$
		- 总运算量：$6{\times}6.74{\times}10^9{\times}10^{12}=4.04\times10^{21}\ \text{FLOP}$
	- 训练时间估计
		- 公式：
			- $\text{训练时间}=\dfrac{\text{总运算量}}{\text{GPU数}\times\text{GPU实际FLOPS}}$
			- 实际利用率取理论峰值 30 % – 70 %
		- Llama-65B 例：  
			- 运算量 $8CP=8\times6.5\times10^{10}\times1.4\times10^{12}=7.28\times10^{23}$ FLOP  
			- 2 048 张 A100，单卡理论 $3.12e^{14}\;BF16\;FLOPS$，按 $2e^{14}$ 实测  
			- 时间 $=7.28\times10^{23}/(2048\times2\times10^{14})\approx1.78\times10^{6}\ \text{s}\approx20.6\ \text{天}$（与论文 21 天吻合）
	- 训练显存估计
		- （1）模型参数与优化器的显存占用
			- P = 模型总参数量（单位：个，不是字节）
			- 混合精度下每参数占用：
				- 模型参数：2 字节 (FP16/BF16)
				- 模型梯度：2 字节 (FP16/BF16)
				- Adam 优化器状态：4 字节 (FP32 参数副本) + 4 字节 (FP32 动量 m) + 4 字节 (FP32 二阶矩 v)
				- 合计单参数 2 + 2 + 4 + 4 + 4 = 16 字节 → 总显存 $16P$ 字节
			- | 方案 | 单卡显存公式（字节） | 极限 $N_D$→∞ 占比 |
			  |---|---|---|
			  | 无 ZeRO | $16P$ | 100 % |
			  | ZeRO-1（仅优化器分片） | $4P + 12P/N_D$ | 25 % |
			  | ZeRO-2（+梯度分片） | $2P + 14P/N_D$ | 12.5 % |
			  | ZeRO-3（+参数分片） | $16P/N_D$ | $1/N_D$ |
			- 若再用张量/流水线并行，单卡显存再除以 $N_P×N_T$
		- （2）激活显存占用（LLaMA 例，单位：字节）
			- | 模块 | 保留张量 | 显存 |
			  |---|---|---|
			  | 多头自注意 | 输入$X,Q/K/V,Concat,QK^T$ | $2BTH+6BTH+2BTH+2BT^2N$ |
			  | SwiGLU FFN | 输入$X,W^GX,W^UX,σ\text{输出}$ | $2BTH+6BTH'$ |
			  | 归一化 | 两层输入 | $4BTH$ |
			  | 输出层 | 归一化输入、Y、softmax 输入($32b$) | $2BTH+4BTV$ |
			- 无优化总式：$\text{激活}=L(16BTH+6BTH'+2BT^2N)+4BTH+4BTV$
			- FlashAttention：去掉 $2BT^2N$ 项
			- 流水线并行：整体除以 $N_p$
			- 张量并行：$Q/K/V/Concat/FFN$中间量再除以$N_T$，但层输入仍全卡保存
			- 激活重计算：仅存每层输入和softmax 输入：$(4+2L)BTH+4BTV$
		- （3）其他显存开销
			- | 项目 | 大小 |
			  |---|---|
			  | PyTorch 内核 | 0.8–1 GB |
			  | ZeRO 管理/通信 | 1–4 GB（随 ZeRO 等级升高而增） |
			  | softmax 中间结果 | 额外 2× 输入 → $8BTV$ 字节 |
			  | 显存碎片 | 0.5–1 GB |
			- 实际规划时在理论值外再预留 ≈3–5 GB 安全垫
		- （4）总显存实例（LLaMA-7B，$2×A800$ $80G$）
			- | 模块 | 公式 | 数值 |
			  |---|---|---|
			  | 参数+优化器 | $16P/N_D$ | 50.20 GB |
			  | 激活（重计算+Flash） | $(4+2L)BTH+12BTV$ | 6.20 GB |
			  | softmax 中间结果 | $8BTV$ | 3.91 GB |
			  | 其他（框架+ZeRO+碎片） | ~ | 6 GB |
			  | **单卡总计** | ~ | **≈ 66 GB** |
			- 经验法则
				- 全参数训练至少留 **16×参数量** 字节显存
				- $13B$ → 208 GB → 最少 3×80 GB 卡，推荐 4 卡以上保证合理batch
				- $30B/65B$ 建议分别 8 张/16 张 80 GB 卡
5. 预训练代码实践（速览）
	- | 步骤 | 关键动作 | 代码/命令要点 |
	  |---|---|---|
	  | 1. 初始化 | 加载分词器 + 模型 | `AutoTokenizer` / `LlamaForCausalLM.from_pretrained(..., torch_dtype=torch.bfloat16)` |
	  | 2. 数据 | 流式读取 → 分块 → `DataCollatorForLanguageModeling` | 自定义 `IterableDataset` 按 `block_size` 切片，避免一次性加载 |
	  | 3. 并行启动 | 单机多卡 | `torchrun --nproc_per_node=8 train.py ...` |
	  | 4. 关键 flag | BF16、梯度累积、ZeRO、激活检查点 | `--bf16 --gradient_accumulation_steps 8 --deepspeed ds_config.json --gradient_checkpointing` |
	  | 5. DeepSpeed JSON（ZeRO-3 示例） | 显存 vs 通信权衡 | `"stage": 3, "stage3_max_live_parameters": 1e9, "stage3_max_reuse_distance": 1e9, "stage3_gather_16bit_weights_on_model_save": true` |
	  | 6. 扩展 ≥30 B / ≥128 卡 | 3D 并行 + 高速网络 | Megatron-LM 提供 PP+TP+DP 模板；硬件需（NVLink + NVSwitch）跨卡 +（InfiniBand）跨机 |
	- “小”规模（≤30 B，≤16 卡）用 Transformers+DeepSpeed ZeRO 即可
	- 超大模型务必切到 Megatron-LM 的 3D 并行，并配齐高带宽网络，否则显存/通信会立刻成为瓶颈
