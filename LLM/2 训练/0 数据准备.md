1. 预训练语料准备主要包括：原始数据收集、数据预处理、数据词元化、以及预训练过程中数据调度方法
2. 数据来源
	- 根据来源不同，预训练数据主要分为通用文本数据和专用文本数据两种类型
	- 通用文本数据
		- 增强模型语言建模能力
		- 网页
		- 书籍
	- 专用文本数据
		- 专用数据集有助于提升大语言模型解决特定下游任务的能力
		- 多语文本：增强模型的多语理解与生成能力
		- 科学文本：增强大语言模型对科学知识的理解
		- 代码：提高模型的代码能力以及结构化语义理解与逻辑推理能力（编程问答社区的数据以及开源项目仓库）
3. 数据预处理
	- 目的：消除低质量、冗余、无关甚可能有害的数据
	- 质量过滤
		- 目前主要使用两种数据清洗方法：基于启发式规则的方法、基于分类器的方法
		- 基于启发式规则的方法
			- 通过精心设计的规则来针对地识别和剔除低质量的文本数据
			- 基于语种的过滤（过滤掉目标外其他语言的文本数据）
			- 基于简单统计指标的过滤（如重复单词或句、点赞数、文档困惑度等）
			- 基于关键词的过滤（如HTML标签、以及IP地址等隐私信息）
		- 基于分类器的方法（更高精确度、更多计算资源消耗）
			- 训练用于判别数据质量的文本分类器，进行预训练语料的清洗
			- 可以使用多个分类器进行联合过滤或召回，实现对低质量文本高可信过滤
	- 敏感内容过滤
		- 包括有毒内容或隐私信息
		- 过滤有毒内容：采用分类器方法
		- 过滤隐私内容：使用启发式方法，如关键字识别（邮箱IP地址电话号码），通过正则替换的方式
	- 数据去重
		- 去重算法的设计可以基于不同的计算粒度以及匹配方法
		- 计算粒度：可以在句子级别、文档级别和数据集级别等多种粒度上进行
		- 匹配方法：可以使用精确匹配算法（即每个字符完全相同）或近似匹配算法（基于某种相似性度量，如最小哈希`MinHash`）
		- `MinHash`：一种估计两个集合之间相似度的技术，其核心思想是通过哈希处理集合元素，并选择最小的哈希值作为集合的表示
	- 数据对预训练效果的影响
		- 数据数量的影响：数据量的扩展性本质上来源于Transformer模型的可扩展性
		- 数据质量的影响
			- 整体影响：直接决定了模型的实际性能，使用大量低质量数据会导致模型训练过程不稳定，造成模型训练不收敛等问题
			- 重复数据：可能会削弱模型在上下文学习中的泛化能力
			- 有偏有毒隐私内容：产生侮辱性、攻击性或其他有害的输出
		- 数据集污染：在进行模型评测时，某些评估基准所包含的数据，已出现在预训练数据或者微调数据中，这种现象被称为基准泄漏或数据集污染
4. 词元化（分词）
	- 旨在将原始文本分割成模型可识别和建模的词元序列，作为大语言模型的输入数据
	- 基于词汇的分词在某些语言（如中文分词）中的问题
		- 可能对于相同的输入产生不同的分词结果，
		- 导致生成包含海量低频词的庞大词表
		- 可能存在未登录词（Out-of-vocabulary, `OOV`）
	- 子词分词器（Subword Tokenizer）被广泛应用于基于Transformer的语言模型中
		- `BPE` 分词
			- 合并：从一组基本符号（例如字母和边界字符）开始，迭代地寻找语料库中的两个相邻词元， 并将它们替换为新的词元
			- 每次迭代中，最频繁出现的一对词元会被选择与合并
			- 合并过程将一直持续达到预定义的词表大小
			- 字节级别的`BPE`：代表性语言模型包括GPT-2、BART和Llama
		- `WordPiece` 分词
			- 与`BPE`分词类似，通过迭代合并连续的词元
			- 训练一个语言模型对所有可能的词元对进行评分，每次合并时，选择使训练数据的似然性增加最多的词元对
		- `Unigram` 分词
			- 从语料库的一 组足够大的字符串或词元初始集合开始，迭代地删除其中的词元，直到达到预期的词表大小
			- 通过计算训练语料的似然增加情况，以此作为删除某个词元的选择标准
	- 分词器的选用
		- 分词器应具有高压缩率，即在给定文本数据的情况下，经过分词处理后的词元数量应尽可能少
		- 压缩比：原始文本的 `UTF-8` 字节数除以分词器生成的词元数（即每个词元的平均字节数）
5. 数据调度
	- 数据调度（Data Scheduling）：各数据源的混合比例、各数据源用于训练的顺序
	- 数据混合
		- 增加数据源的多样性
		- 优化数据混合，训练几个具有不同配比的小型语言模型，选择最理想的配比
		- 优化特定能力，通过增加特定数据源的比例来增强某些对应的模型能力
	- 数据课程
		- 更广泛来说，包括按照特定的顺序安排预训练数据进行模型训练
		- 代码能力：如通用词元 → 代码相关词元 → Python代码相关词元
		- 数学能力：如通用词元 → 代码相关词元 → 数学相关的词元
		- 长文本能力：如`2.5T`词元`4K`上下文窗口 → `20B`词元`16K`上下文窗口
	- 预训练阶段的一般流程和关键要点
		- 数据收集：多样化的数据来源
		- 数据清洗：质量过滤、去重、隐私去除以及词元化等
		- 数据调度：
			- 采用小模型的代理方法，首先确定语言配比，然后确定不同数据类型配比
			- 选择Llama的数据比例作为基础，每次仅调整某一类型数据比例进行实验
