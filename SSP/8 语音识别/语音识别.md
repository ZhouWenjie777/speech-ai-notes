# 1. 原理和系统组成
1. **语音识别（Speech Recognition）**：让机器“听懂”人类语音，并转成文字或命令
2. **实际应用**
	- **语音打字机**（Win 11 内置听写）
	- **数据库检索**（医院：“查询血压>140 的病例”）
	- **特定环境语音命令**（车载“下一首”）
	- **智能语音助手**（小爱、Siri、Alexa）
3. **识别两大任务**
	- ① **语音内容识别**（What was said?）→ 强调发音共性，抹平说话人差异
	- ② **说话人识别**（Who said it?）→ 强调个性，用于声纹门禁/取证
4. **系统 6 维分类**
	- （1）**识别单位**
		- 孤立词 | 连接词 | 连续语音 | 语音理解 | 会话语言（省略、倒置、口头禅）
	- （2）**词汇量**
		- **小词汇**：（1–20）→ 命令词
		- **中词汇**：（20–1 000）→ 语音菜单
		- **大词汇**：（>1 000）→ 听写、会议字幕
		- **原则**：词汇↑ → 识别基元↓（从整词→音素/音节）
	- （3）**讲话人**
		- 特定人（单人训练）| 多讲话人（有限名单）| 讲话者无关（任意人）
		- “无关”需大量多说话人数据训练，模型最大
	- （4）**环境**
		- 隔音室 | 安静办公室 | 公共场所 | 车载 | 工业噪声
		- 环境越噪，前端增强（DNN降噪、波束形成）越重要
	- （5）**说话人类型**
		- 男声 | 女声 | 童声
		- 基频范围差异 2 octaves，模型需覆盖或自适应
	- （6）**方法学**
		- 模板匹配（DTW）→ 随机模型（GMM-HMM）→ 概率语法→ 深度学习（DNN-HMM、E2E、Transformer）
5. **研究挑战**
	- （1）**协同发音与音变**：连续语流里“天安门”→/tʰiɛn an mən/，实际可能是[tʰiɛm˺ mən]，边界模糊→基元切分困难
	- （2）**语音变化性**：同一说话人今天/明天、感冒/正常，声道特性变化→模型要鲁棒
	- （3）**模糊性&同音词**：“期中考试” vs “期终考试” → 只能靠语言模型消歧
	- （4）**语境影响**：“我想吃苹果” vs “苹果想我” → 相同音素序列，语义相反，需上下文建模
6. **系统两大物理模块**
	- （1）**模块一：音频流 → 数字特征**
		- 采样 → 预加重 → 分帧 → 加窗 → FFT → MFCC / fbank / 波形采样
		- 输出：特征向量序列$O = o_1, o_2, …, o_T$
	- （2）**模块二：搜索引擎**，在「发音模型 + 语言模型 + 词典」里找最可能词序列：$W* = argmax_W P(O|W) · P(W)$
		- **发音模型**：声学模型（GMM/DNN/CNN/Transformer）
		- **语言模型**：N-gram、LSTM、GPT 式 Transformer
		- **词典**：词 → 音素序列（lexicon）
7. **流程两步走**
	- **Step-1 学习 & 训练**
		- 准备大量「语音-文本」平行语料 → 训练声学模型、语言模型
		- 发音模型：给每帧打音素后验
		- 语言模型：给句子打分
	- **Step-2 识别 & 测试**
		- 特征流输入 → 解码器（Viterbi / Beam Search / CTC / RNN-T）→ 输出文字序列
		- 实时系统加「流式解码」：每200ms出一次中间结果
## 1.1 预处理和参数分析
1. **语音采集**
	- **奈奎斯特**：fs ≥ 2 fmax，电话 8 kHz / 宽带 16 kHz
	- **抗混叠滤波器**：阻带 ≤ fs/2 − 400 Hz（留过渡带）
	- **量化**：16 bit，SNR ≈ 6.02×16 ≈ 96 dB，足够覆盖 40 dB 语音动态
2. **预加重（Pre-emphasis）**
	- **一阶FIR**：H(z)=1−αz⁻¹，α=0.95~0.97（8 kHz）/ 0.90~0.92 (16 kHz)
	- **目的**：补偿口-唇辐射 −6 dB/oct，令频谱平坦→相同信噪比→LPC/倒谱更准确
	- **逆滤波（de-emphasis）**：在合成端用 1/(1−αz⁻¹) 还原听感
3. **加窗**
	- **帧长**：20~25 ms（典型 400 @16 kHz）
	- **帧移**：10 ms
	- **汉明窗（Hamming）最常用**：主瓣宽 −3 dB，带宽 1.3/bin，旁瓣 −42 dB；若做相位声码器改用 汉宁窗（Hann）
	- **窗长短 vs 分辨率**：窄→时间好；宽→频率好，选 2~3 倍本地基音周期
4. **端点检测（VAD）**
	- **特征组合**
		- **短时能量**：$E = Σx²$
		- **过零率**：$ZCR = ½ Σ|sgn[x(n)]−sgn[x(n−1)]|$
		- **综合特征**：归一化 E 和 ZCR，再加权
		- **门限自适应**：silence 前 200 ms 统计 μ+3σ
	- **难点**
		- **能量低**：摩擦音/f/、鼻音 /m/，易被判成噪声，加谱熵或低频能量比可提高召回
		- **突发冲击**：键盘、咳嗽，用谱质心突变检测剔除
		- **车载/地铁**：先 DNN 降噪再做 VAD，否则门限全天漂移
5. **识别参数（Features）**
	- **低层物理**
		- 平均能量、ZCR、自相关峰值
		- 频谱、倒谱（MFCC 13~20 维）、PLP、Mel-滤波器组 40 维
		- 线性预测系数（LPC 10~16 阶）、反射系数（$k_i$）、LSF
		- 共振峰 F1~F3（根倒谱或 LPC 求根）
	- **高层超音段**
		- 音长、基音 F0、声调曲线
		- 一阶 Δ、二阶 ΔΔ（加速度）→ 39 维 MFCC+Δ+ΔΔ 标配
	- **现代趋势**：直接 80 维 Mel-滤波器组 → DNN 自动学变换，不再手工 MFCC
## 1.2 语音识别核心模块
1. **声学模型（AM）**
	- **任务**：$p(o|q)$，观察向量 $o_t$ ← 状态 $q_t$
	- **三代**：GMM-HMM → DNN-HMM → E2E（CTC/RNN-T/Transformer）
	- **状态数**：phone × 3 (B-M-E) 或单状态 CTC
	- **训练数据**：≥ 100 h 起步，16 kHz，标注到帧级（Force-align）
2. **语言模型（LM）**
	- **规则模型**：手写语法（适合命令词 20 以内）
	- **统计模型**：N-gram（2~5 元）、pruned arpa
	- **神经网络** ：LSTM、Transformer、GPT
	- **插值**：$P_{final} = λ P_{AM} + (1−λ) P_{LM}，λ≈0.3-0.5$
3. **连续语音自动分段（Segmentation）**
	- **目的**：从长 wav 里切出“识别基元”帧序列
	- **特征**：对数能量 + ZCR + LPC 反射系数
	- **算法**：Viterbi Force-align（已知文本）或 Bayesian Change-point（盲切）
4. **距离测度（Scoring）**
	- **欧氏距离**：$d_{EUC} = ||x−y||²$，简单但不对方差加权
	- **似然比**：$d_{LLR} = log(p(x|M1)/p(x|M2))$，需GMM超参数
	- **加权识别测度**：Mahalanobis距离，$d_{MAH} = (x−y)^T Σ^{-1}(x−y)$，白化后更准
5. **计算量 & 存储消减**
	- **特征降维**：PCA/LDA 40→13；矢量量化 (VQ) 码本 256 条
	- **模型剪枝**：GMM混合数 n →1~8；DNN 权重量化 32→8 bit
	- **帧率减半**：10 ms → 20 ms，几乎不丢精度，运算×0.5
6. **拒识 & 候补**
	- **拒识**：似然得分 < θ 或 posterior < 0.3 时返回“未识别”
	- **N-best 列表**：Beam Search 保留 Top-5，供用户二次选择或重打分
## 1.3 基本数据库
1. **词汇表（Lexicon）**
	- **词 → 音素序列**：覆盖 OOV 用 G2P 模型自动生成
	- **含重读/变调标记**：银行 háng vs 行走 xíng
2. **声学模型参数**
	- **GMM**：均值、方差、权重
	- **DNN**：权重矩阵、BNF 层
	- Speaker-map 或 i-vector 供自适应用
3. **语言模型参数**
	- **N-gram**：arpa 文件
	- **Neural-LM**：ckpt（checkpoint）+ tokenizer
	- **热词插槽**：用户可随时 add-word 并增量重训 LM
## 1.4 流程
1. 「预加重-分帧-特征」→「声学模型打分」→「语言模型约束」→「Viterbi/Beam 搜索」→「N-best 输出」→「拒识/候补」→ 文本
# 2. 模板匹配法
## 2.1 动态时间规整（DTW）
1. **定位**：把时间轴“拉伸/压缩”后再算距离，解决“说快说慢”导致的长度不一问题。无需分段，适合孤立词、小词汇、低资源场景。
2. **原理**：寻找一条最优时间归正路径 ϕ = {ϕx(i), ϕy(i)}，把测试序列 X(1…Nx) 映射到模板 Y(1…Ny)，使得累计距离最小。
3. **实现**：**动态规划（DP）**。
	- **逆序决策**：从 (Nx, Ny) 往前推，保证全局最优。
	- **递推（对称形式）**：$D(i,j) = d(i,j) + min[D(i-1,j), D(i-1,j-1), D(i,j-1)]$
	- **边界**：D(0,0)=0
	- **加权**：对角线 ×1，水平/垂直 ×2（防止过度压缩）
	- **约束**：
		- 单调非减
		- 局部连续性 |ϕx(i)−ϕx(i−1)| ≤1
		- 全局窗口 |i−j| ≤ r （常用 r = 10 % max(Nx,Ny)）
4. **步骤**
	- 特征帧提取（MFCC 13 维）
	- 构造 Nx×Ny 累积矩阵 D
	- 按递推公式填矩阵（O(Nx·Ny))
	- 从 (Nx,Ny) 回溯到 (0,0) 得最优路径 ϕ
	- 累计距离 D(Nx,Ny) 即为匹配得分 → 最小者判为识别结果
5. **经验参数**
	- **帧长** 25 ms，**帧移** 10 ms
	- **特征**：MFCC 13 + Δ 13 → 26 维
	- **距离**：欧氏 ||x−y||，可先 PCA 白化（先 PCA 去相关，再把每维方差压成 1）
	- **全局窗**：r = 0.1·max(Nx,Ny)
6. **优缺点**
	- **优点**：无需训练、小样本、算法 20 行代码、对孤立词鲁棒
	- **缺点**：存储随模板数线性增长、N>2000 时实时性下降、无法嵌入语言模型（纯声学匹配）
## 2.2 矢量量化方法（VQ）
1. **定位**：把高维特征空间划分成有限个区域（码字），用区域编号代替真实向量 → 数据压缩 + 距离加速
2. **训练（聚类）**：经典 LBG (Linde-Buzo-Gray) 算法
	- （1）初始码字 = 所有向量均值
	- （2）分裂：每个码字 ± ε 扰动 → 2× 码字
	- （3）最近邻划分 → 计算质心
	- （4）若失真下降 < threshold 停止，否则回 2
	- （5）结果：码本 $C = {c_1, c_2, …, c_K}$，K 常用 256~1024
3. **识别流程**
	- **训练阶段**：每个词的所有训练特征 → 聚成一份码本（K 码字）
	- **测试阶段**：把测试序列每帧先**量化**到最近码字 → 结合**码本距离阵**用码字索引与模板码本算平均量化误差
	- 选误差最小词作为结果
4. **变种**
	- 分裂 VQ、树形 VQ、乘积 PQ（用于 ANN 检索）
	- 半连续 HMM：GMM 混合权重用统一码本，参数共享 → 存储↓

2. **基于判别函数或准则的方法**
3. **基于动态时间规整（DTW）的方法**

## 3.1 修正型二次判别函数法（MQDF）
1. **原理简介**：MQDF 是对传统二次判别函数（QDF）的改进，用于解决高维特征下协方差矩阵估计不准确、识别性能下降的问题。
2. **核心思想**
	- 利用协方差矩阵的特征值和特征向量进行降维处理；
	- 将较小的特征值统一设为一个常数，减少参数估计误差；
	- 修正判别函数，提高鲁棒性
3. **优点**
	- 在高维特征空间中表现稳定；
	- 对小样本数据适应性强；
	- 计算效率高于标准 QDF
4. **缺点**
	- 仍依赖于高斯分布假设；
	- 对非线性分布数据处理能力有限。
5. **适用场景**
	- 孤立汉字识别；
	- 小词汇量语音识别任务。
## 3.1 概率尺度DP
1. **核心问题**：传统 DP 是确定性匹配，用欧氏距离等度量方式，容易受噪声、语速变化影响，鲁棒性差
2. **概率尺度的引入**：为了提升鲁棒性，将“距离”替换为“概率”，即不再最小化“距离”，而是最大化“匹配概率”。
3. **关键改进点**（与传统 DP 的区别）

| 项目   | 传统 DP     | 概率尺度 DP     |
| ---- | --------- | ----------- |
| 匹配度量 | 欧氏距离（确定性） | 概率得分（似然度）   |
| 目标   | 最小化总距离    | 最大化总概率      |
| 对齐路径 | 硬对齐       | 软对齐（考虑不确定性） |
| 鲁棒性  | 对噪声敏感     | 对噪声、语速变化更鲁棒 |
4. **实现方式**
	- **特征提取**（如 MFCC）
	- **模板建模**（如 GMM 或 HMM 状态输出概率）
	- **构建概率矩阵**（每个时间帧对模板状态的概率）
	- **DP 搜索最大概率路径**（Viterbi 或类似算法）
	- **识别决策**（选概率最大的模板作为识别结果）

# 5. 连续语音识别系统的性能评测
1. **系统识别率测度**

| 指标                    | 定义                       | 计算方法                                | 实际意义                       |
| --------------------- | ------------------------ | ----------------------------------- | -------------------------- |
| **正确数（Correct, C）**       | 识别结果中与参考文本完全匹配的词汇/字符数    | 人工或自动对齐后统计                          | 反映系统准确识别的基本单位数量            |
| **插入数（Insertions, I）**    | 识别结果中多出的词汇/字符数           | 对齐后多出的部分                            | 反映系统“过度识别”问题，可能由噪声或模型过拟合引起 |
| **置换数（Substitutions, S）** | 识别结果中与参考文本不同但位置对应的词汇/字符数 | 对齐后替换错误                             | 反映系统对相似音或词混淆的程度            |
| **脱落数（Deletions, D）**     | 参考文本中未被识别出的词汇/字符数        | 对齐后缺失部分                             | 反映系统“漏识别”问题，可能由弱信号或模型保守引起  |
| **正确率（Accuracy, Acc）**    | 正确识别的比例                  | $\frac{C}{N} \times 100\%$          | 初步衡量系统性能，但未考虑插入错误          |
| **错误率（Error Rate, ER）**   | 识别错误的综合比例                | $\frac{S + D + I}{N} \times 100\%$  | 最常用指标，越低越好                 |
| **识别精度（Precision）**       | 识别结果中有多少是正确的             | $\frac{C}{C + S + I} \times 100\%$  | 衡量识别结果的“纯净度”               |
| **召回率（Recall）**           | 参考文本中有多少被正确识别            | $\frac{C}{C + S + D} \times 100\%$  | 衡量系统“完整性”                  |
| **F1 分数**                 | 精度与召回的调和平均               | $2 \times \frac{P \times R}{P + R}$ | 综合衡量系统性能，适用于不平衡数据          |
2. **系统识别任务复杂性的测度**

| 指标                               | 定义             | 计算方法                 | 实际意义                                         |
| -------------------------------- | -------------- | -------------------- | -------------------------------------------- |
| **静态分支度（Static Branching Factor）**   | 每状态下平均可能的后续词数  | 统计语言模型中每个词后的平均候选数    | 反映语言模型的“歧义程度”，越大越难                           |
| **平均输出数（Average Number of Outputs）** | 每帧声学模型输出的平均候选数 | 统计每帧解码时的平均活跃路径数      | 反映声学模糊度，影响解码效率。值越高表示声学模型不确定性越大，搜索空间越大        |
| **识别任务的熵（Task Entropy）**             | 语言模型的不确定性      | 基于语言模型对测试集的预测分布计算信息熵 | 衡量识别任务本身的复杂度，熵越高表示语言结构越不确定，识别越困难             |
| **识别单位的分支度（Perplexity）**             | 语言模型对测试集的“困惑度” | 计算测试集上语言模型的平均负对数概率   | 衡量语言模型的预测能力，值越低越好。困惑度越高，模型对语言的预测越差，识别错误率可能越高 |
3. **综合评估需考虑的其他因素**

| 因素                         | 说明                    | 影响方向                 |
| -------------------------- | --------------------- | -------------------- |
| **词汇量大小（Vocabulary Size）**     | 系统支持的词表规模             | 越大，语言模型越复杂，错误率可能上升   |
| **话者依赖性（Speaker Dependency）**  | 是否针对特定话者训练            | 特定话者系统准确率更高，但通用性差    |
| **发音方式（Speaking Style）**       | 孤立词 vs 连续语音           | 连续语音更难，因存在连读、省略、语调变化 |
| **发音环境（Acoustic Environment）** | 安静、噪声、远场、混响等          | 噪声越大，识别率越低，需鲁棒性处理    |
| **语言类型（Language Type）**        | 拼音文字（如英语）vs 表意文字（如中文） | 中文无空格，需分词；同音字多，歧义大   |
| **领域适应性（Domain Adaptation）**   | 是否针对特定领域优化            | 医疗、法律等专业领域需专门语料微调    |
| **实时性要求（Latency）**             | 是否需实时识别               | 实时系统需在准确率和延迟间权衡      |
| **数据多样性（Data Diversity）**      | 训练集是否覆盖口音、年龄、性别等      | 多样性不足会导致系统偏见和鲁棒性差    |
4. **现代评测趋势（2020s+）**

| 趋势      | 说明                                         |
| ------- | ------------------------------------------ |
| **端到端系统评测** | 不再分声学/语言模型，直接评测 Attention/Transformer 输出   |
| **轻量级模型评测** | 移动端部署需考虑模型大小、推理速度、功耗                       |
| **多模态评测**   | 结合视觉（唇读）、文本（上下文）提升准确率                      |
| **鲁棒性测试集**  | 如 CommonVoice、AISHELL-4、WenetSpeech 含噪声、口音 |
| **公平性评测**   | 检查系统是否对某些群体（如方言、性别）存在偏见                    |
| **人类对比实验**  | 将 ASR 结果与人类转录员对比，评估“可接受误差”                 |
# 6. 基于DNN-HMM的语音识别系统
## 6.1 系统总览
1. **核心思想**：用 DNN 替换 GMM，估计 HMM 的状态发射概率$p(x_t|s_j)$ 
2. **整体链路**：语音波形 → 特征提取 → DNN 后验 → 转换似然 → HMM 解码 → 词序列
## 6.2 特征提取（Fbank 路线）
1. **预加重**： $1-0.97z^{-1}$，平衡高频
2. **分帧加窗**：25 ms 窗，10 ms 移，汉明窗
3. FFT → 功率谱 → 梅尔滤波器组（80 个三角滤波器，24 kHz 以下均匀分布）
4. **对数压缩**： $\log(\text{mel-filterbank energies})$
5. **可选**：倒谱提升（DCT→13 维 MFCC）或保持 40/80 维 Fbank
6. **拼接上下文**：±5 帧 → 440/880 维输入向量
7. **均值-方差归一化**（CMVN）：句子级或滑动窗 3 s
	**口诀**：预分窗谱梅尔，对数拼接归一化。
## 6.3 DNN 建模
1. **输入**：拼接 Fbank（440 维示例）
2. **隐层**：4–7 层，ReLU/tanh，1024–2048 节点，Dropout 0.2
3. **输出层**：softmax，节点数 = 三音素状态数（典型 1 500–9 000）
4. **目标**：交叉熵最小化，强制对齐来自 GMM-HMM（阶段 0）
## 6.4 似然转换
1. HMM 需要似然 $p(\mathbf{x}_t|s)$，而 DNN 输出后验 $p(s|\mathbf{x}_t)$
2. 利用贝叶斯公式：
$$
p(\mathbf{x}_t|s)=\frac{p(s|\mathbf{x}_t)p(\mathbf{x}_t)}{p(s)} \approx \frac{p(s|\mathbf{x}_t)}{p(s)} \qquad \text{忽略}p(\mathbf{x}_t)\text{，在解码比较中常恒}
$$
	其中先验$p(s)$用训练集状态出现频率估计，称“先验缩放”或prior scaling
## 6.5 解码与训练流程
1. **阶段 0**：训练 GMM-HMM → 生成状态级强制对齐
2. **阶段 1**：用对齐目标训练 DNN（CE）
3. **阶段 2**：用 DNN-HMM 重新对齐 → 再训练 DNN（可迭代 2–3 次）
4. **阶段 3**：序列训练（sMBR/MPE）微调，WER 相对下降 5–10 %
5. **解码**：维特比 + 语言模型（WFST 合成 H∘C∘L∘G）
