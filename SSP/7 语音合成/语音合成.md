# 1. 概述
1. **语音合成**：由人工制作出的语音。大体上可以分为参数合成法、波形拼接法、统计参数法。
2. **参数合成法**：包括共振峰合成，LPC合成和基于神经网络合成。需要分析调整大量参数，实用性差。
3. **波形拼接法**：从事先录制的音库中提取声学单元，按拼接准则合成语音，易出现语音不连续的问题。
4. **统计参数法**：利用统计参数建模，但语音自然度较低。
# 2. 共振峰合成法
1. **共振峰滤波器**：把声道视为谐振腔，利用腔体的谐振特性，如共振峰频率及带宽，以此为参数构成一个滤波器。
	- **声道模型**：声道被看作一个非均匀截面的管状谐振腔，其谐振频率即为共振峰（Formant）。
	- **滤波器形式**：每个共振峰用一个二阶谐振器（二阶IIR带通滤波器）模拟。
	- **特点**：每个滤波器只模拟一个共振峰，多个滤波器级联即可模拟整个声道的谐振结构。
2. **合成原理**：将多个共振峰滤波器组合起来模拟声道的传输特性，对激励信号进行调制，经过辐射即可得到合成语音。
	- **系统结构**：激励源 → 共振峰滤波器组（声道） → 辐射模型 → 合成语音
	- **调制过程**：激励信号（如冲激序列）通过共振峰滤波器组，其频谱被共振峰结构塑形，输出信号再经过辐射模型（如微分器$1-z^{-1}$ 模拟嘴唇辐射效应）
3. **参数更新**：由于音色各异的语音具有不同的共振峰模式，一般要求共振峰合成器的参数逐帧修正。如共振峰频率、带宽、增益。
4. **共振峰合成器的系统模型**
	- **冲激发生器**：生成周期冲激序列，对应浊音（如元音），频率 = 基频F0 
	- **噪声发生器**：生成伪随机白噪声，对应清音（如 /s/、/f/） 
	- **级联型调制器**：多个二阶谐振器串联 ，模拟元音的声道传输函数，结构简单，频谱平滑 
	- **并联型调制器**：各共振峰独立增益控制，模拟辅音、鼻音等复杂频谱，灵活性更高 
	- **基音调制**：控制浊音的基频F0 ，可逐帧变化，模拟语调、重音 
	- **辐射效应**：模拟嘴唇辐射，通常用一阶高通滤波器$1-z^{-1}$近似。
	- **激励源**：控制浊/清/混合音，分别对应周期冲激序列、伪随机序列和周期冲激调制噪声。
# 3. 线性预测合成法
1. **核心思想**：用p个过去样本的线性组合预测当前样本：$ŝ(n)=Σ_{k=1}^{p}a_k s(n−k)$把误差$e(n)=s(n)−ŝ(n)$当作激励，只要保存LPC系数$a_k$和激励参数即可重建语音 → 极低码率（1.2–2.4 kbps）。
2. **LPC分析-合成流程**
	- 分帧（20 ms，Hamming窗）
	- 自相关法/协方差法求LPC系数$a_k$（常用Levinson-Durbin，$O(p²)$）
	- 同时估计：清/浊音UV标志，浊音基频F0，增益G = √帧能量
	- 合成端按帧参数驱动LPC合成滤波器：1/A(z)，$A(z)=1−Σ_{k=1}^p a_k z^{−k}$
3. **合成滤波器结构**
	- **递归型（直接型）**：用$a_k$作为滤波器系数，简单但量化噪声累积，可能不稳定。稳定性判据：所有极点必须在单位圆内（可用Durbin中间量检查）。
	- **格型（Lattice）**：用反射系数$k_i$，$|k_i|<1$自动稳定，逐级正交化，抗量化好，适合硬件。运算量略高，但现代DSP可忽略。
4. **激励模型（LPC-10/CELP基础）**
	- **浊音**：冲激发生器，周期$δ(nT0)，T0=1/F0$
	- **清音**：白噪声发生器，N(0,1)
	- **增益G**：统一控制幅度，经典二元激励模型，简单但可能产生蜂鸣声。
5. **参数更新**
	- 帧移10 ms，参数线性插值$a_k, G, F0$，避免帧间跳变。
	- 反射系数域插值比直接$a_k$插值更稳定。
6. **优缺点**
	- **优点**：低数据率、低复杂度、低成本；分析合成框架奠定CELP/HE-VCX等标准。 
	- **缺点**：二元激励假设粗糙，共振峰带宽偏锐，音质机械，对背景噪声敏感。
# 4. 神经网络语音合成法
1. **基于DNN的统计参数语音合成**：声学建模、波形建模、频谱特征
2. **统计参数语音合成框架**
	- **语音合成系统主要分前端和后端两部分**
		- **前端**：对输入文本进行文本分析
		- **后端**：韵律建模和声学建模，通过声学预测合成语音
	- **语音合成框架主要分训练和合成两阶段**
		- **训练**：以音库声学特征参数和相应文本特征，训练声学模型，常用HMM
		- **合成**：结合输入文本特征，利用声学模型预测声学特征，通过声码器合成
3. **基于神经网络的声学建模方法**
	- **方法框架**：文本-->文本特征-->模型-->声学特征-->声码器-->语音
	- **基于DNN的声学建模方法**：输出特征包括清浊参数，基频参数等。
	- **基于RNN的声学建模方法**：DNN的延伸模型，其输入特征序列是静态的，而RNN输入特征序列是动态的。
# 5. 专用硬件简介
1. **LPC语音合成器功能模块**
	- **时序电路**：整个芯片的“指挥棒”，产生采样时钟、帧更新脉冲、RAM 读写时序、滤波器运算节拍
	- **RAM**：存放当前帧参数（LPC 系数$a_k、G、F0、UV$）和滤波器状态（过去p个样本）。
	- **数字滤波器**：计算s(n)并写回状态RAM，同时送DAC
	- **控制电路**：解释外部码流，把比特流→帧参数；协调各模块握手。
	- **音调处理器**：根据F0参数产生周期冲激地址（浊音）或静音指示（清音）。
	- **激励发生器**：按UV标志，产生当前样本的激励值e(n)，并乘增益G。
	- **DAC**：数模转换器，把数字样本s(n)转换成模拟电压，驱动喇叭。
# 6. PSOLA算法（Pitch-Synchronous OverLap-Add）
1. **技术定位**
	- 保持原始语音的主要音段特征，在音节拼接时灵活调整基音、能量和音长等韵律特征。
	- 无需显式声码器，属于波形级拼接合成，音质高、运算小；缺点是难以极端拉伸（>2×）或大幅变调（>4半音）。
2. **算法本质**
	- 短时傅里叶变换重构信号的叠结相加法。
	- 在基音周期标记处加窗 → 得到短时信号（pitch-synchronous frame）→ 修改时长/基频 → 重叠相加（OLA）重建。
	- 窗长 ≈ 2×本地基音周期，窗移 = 1×周期，保证临界采样。
3. **实现方式**
	- **时域基音同步叠加（TD-PSOLA）**：直接对时域波形做标记-抽帧-OLA，最常用。
	- **线性预测基音同步叠加（LPC-PSOLA）**：先用线性预测得到残差，仅对残差做 PSOLA，再经 LPC 合成滤波器重建，可减弱谱包络失真。
	- **频域基音同步叠加（FD-PSOLA）**：在 STFT 域修改每帧谱的相位斜率实现变调，再逆变换 OLA，避免时域标记误差，但计算量高。
4. **TD-PSOLA实现语音合成**
	- **基音同步分析**：对语音合成单元进行同步标记，即标记各个注音段基音周期起始位置。
		- 用「自相关+动态规划」求可靠**基音周期** T0(n)。
		- **标记周期**起始位置 tm（marker）。
		- **加窗抽帧**：xn(m)=x(m)w(m−tm)，窗函数常用汉宁窗，长度 2T0。
	- **基音同步修改**：在合成规则的指导下，调整同步标记，产生新的基音同步标记。
		- **时长调整**：删除/插入整周期帧 → 新标记 t′k。例：加速 2× → 隔一删一；减速 1.5× → 每两帧插一帧（线性插值能量）。
		- **基频调整**：拉伸/压缩标记间隔。例：升调 ½ 倍周期 → 标记间隔Δt′=Δt/2；降调 ×2 → Δt′=2Δt。
		- **能量调整**：每帧乘增益因子Gk。
	- **基音同步合成**：利用短时合成信号进行叠加合成。
		- 按新标记 t′k 放置修改帧，窗长与窗型保持原比例。
		- 重叠区域相加：$y(m)=Σ_k x^′_k(m)w(m−t^′_k)$。
		- 归一化窗能量，防止幅度跳变。
5. **关键参数与经验值**
	- **窗型**：汉宁窗（抑制频谱泄漏）。
	- **窗长**：2×本地T0（男声10 ms左右，女声5 ms）。
	- **最大拉伸率**：1.3~2.0（再高需 WSOLA/Phase Vocoder）。
	- **基频修改范围**：±2 半音内几乎无感知失真；±4 半音需 FD-PSOLA 或相位重置。
# 7. 文语转换系统（TTS）
## 7.1 TTS组成
1. **文本分析**：把“生文本”变成带发音、节奏、情感标签的「结构化语言描述」
	- **步骤一：纠错和剔除**，删 HTML/emoji、全半角统一、同音纠错（“系/是”）
	- **步骤二：分词和发音**，中文先分词 → 转拼音 → 多音字消歧
	- **步骤三：语气和轻重**，给每个词打词性、句法角色、重读等级
	- **方法**：
		- **基于规则**
			- **最佳匹配法**：词典最长匹配 + 人工规则消歧
			- **二次/双向扫描法**：左→右、右→左各一次，投票决定切分
			- **优点**：零资源；缺点：覆盖差，新词/网络语容易错
		- **数据驱动**
			- **二元/三元文法**：P($词_i|词*{i-1}$) 统计切分
			- **HMM/CRF**：把分词当序列标注（B-M-E-S）
			- **ANN/BERT**：直接端到端「字→词边界」分类，现最主流。
2. **韵律控制**
	- **特征维度**：语气、感情色彩、态度、说话节奏
	- **特征参数**：基频 F0、音长 Duration、音强 Energy、停顿 Pause。
	- **基于规则**
		- **手写“重读词典+句法规则”**：动词/名词重读，逗号 150 ms 停顿。
		- **情感模板**：疑问句尾 F0 ↑20%；生气句整体音量 +3 dB。
	- **数据驱动**
		- **决策树/XGBoost**：用文本特征（词性、句长、位置）→ 预测 F0 曲线
		- **HMM/BLSTM**：帧级 F0、duration 联合建模，再参数生成
		- **端到端 GST/Tacotron2**：自动学“潜在韵律嵌入”，无需人工标签
3. **语音合成**：主流仍是“选音 + 小修”（波形拼接法），而非纯参数生成。
	- **选音**
		- 音库切成半音节、音素、Diphone 或 WFST子词
		- 神经网络/统计模型打分，Viterbi 挑最短路径，得到“最优单元序列”
	- **拼接 & 修饰**
		- 用 PSOLA做边缘拉伸/压缩、基频微移，让接缝对齐
		- 只改局部10-30%，避免“机器腔”；运算量低，手机端可实时
	- **新趋势**
		- **神经声码器 WaveRNN、HiFi-GAN**：拼接后仍用神经声码器再编码一次，音质更润
		- **端到端E2E-TTS（VITS、YourTTS）**：直接 text→wave，跳过显式选音+PSOLA，但大模型、大算力
## 7.2 韵律特性
1. **韵律**：超音段信息，决定“说得是否自然、是否有感情”。
2. **转接与音渡规则**
	- **转接**：声母和韵母拼接过程中出现辅音和元音的过渡
		- **声学表现**：爆破峰、共振峰弯折（Formant Transition）
		- **音轨**：同一辅音造成的共振峰漂移的走向往往趋于同一点，该点为音轨
	- **音渡**：复合韵母之间出现元音和元音的过渡
3. **变调规则**：连续语流中，字与字之间的发音存在干扰现象，一字单独发音与连续发音声调有所不同
	- **普通话常见变调**（TTS 前端必须生成）
		- **三三变**：前 3→2，如“你好” ní hǎo → ní háo
		- **一不变**：单念“一”yī，“一年”yí nián，“一样”yí yàng
		- **七八变**：口语“七”qī→qí、“八”bā→bá（可选）
		- **轻声**：后缀“子、们”→失去原调，时长缩短 40%，音高随前字走向。
	- **数据驱动做法**
		- 用 CRF/BERT 做“字→实际调类”序列标注，把变调当多音字问题处
		- 输出给韵律模块时，已是“表面调值”，后续声学模型直接执行即可
4. **音长规则**
	- **层级比例**（普通话经验值）
		- **音素层**：塞音爆破段 10–30 ms；韵腹稳定段占全韵 60–70%
		- **音节层**：轻声 ≈ 0.12 s，正常四声0.18–0.25 s，重读可拉长 40%
		- **词层**：末音节平均+15%（词重音）；句层：句末停顿 200–400 ms，疑问句再+20%
	- **规则公式**
		- $DUR = DUR_{min} + (DUR_{max} − DUR_{min}) × Weight$
		- $Weight = 0.3×位置 + 0.2×重读 + 0.15×句法边界 + 0.1×情感$
	- **数据驱动**
		- **决策树/XGBoost**：文本特征→预测帧级duration
		- **端到端 Tacotron**：自动学「潜在时长」，但仍需Monaural正则化防止过快/过慢
## 7.3 文本分析方法
1. **文本分析**
	- 把“生文本”→ 计算机能吃的「结构化语言学符号序列」
	- 字/词边界 | 拼音 | 实际调类 | 重读等级 | 停顿级别 | 特殊符号归一化
2. **整体流程（5步）**
	- **步骤1**：提取句子（按标点切分）
	- **步骤2**：自动分词（系统词库 + 算法）
	- **步骤3**：多音字处理（多音字词库 + 规则/统计）
	- **步骤4**：声调调整（变调规则库 → 表面调）
	- **步骤5**：特殊符号处理（数字、英文、标点 → 可读字符串）
3. **步骤2：自动分词**
	- **经典机械匹配类**（速度极快，TTS 前端首选）
		- **最大匹配法 MM**：从左向右取最长词典词，剩余再切。
		- **逆向最大匹配法 RMM**：从右向左，对汉语偏正结构更友好（“南京市长”→ 南京市/长）。
		- **双向扫描法**：MM 与 RMM 都跑一遍，不一致处选切分次数少的 → 缓解 22% 歧义。
		- **二次扫描法**：第一次粗切，第二次用二元语法重打分，保留最高概率路径。
	- **基于规则/理解**
		- **词法-句法级规则**：介词、助词单独成词；量词强制向前合并。
		- 计算量大，一般只用于“校对层”而非主切分。
	- **统计/机器学习**（现代主流）
		- **二元/三元文法**：$P(词_i|词_{i-1})$，用 Viterbi 找最大概率路径。
		- **CRF/BERT字标注**：B-M-E-S四标记，端到端训练，OOV（新词）召回高
	- **TTS工程实践**：“MM + 二元文法重打分” 占资源最小；服务器端直接上 CRF 或 BERT 切分
4. **步骤3：多音字处理**
	- **词库法**：<多音字, 拼音, 词例>：“行” → háng（银行）| xíng（步行）| xìng（行伍，古音）；切词后直接用词例匹配，**命中**即出拼音。
	- **规则/统计后备**：**未命中** → 用前后词性 + 二元语言模型选最高概率读音：P(银行|háng) > P(银行|xíng) → 选 háng
	- **端到端**：BERT微调“字→读音”任务，把多音字当多分类，准确率 98%+，但需要 100 MB 模型。
5. **步骤4：声调调整**
	- **「字调」→「实际调类」**
	- **三三变调**：前 3→2（“你好”ní hǎo → ní háo）
	- **一不变**：yī→yí/yì 依据后字调
	- **轻声规则**：后缀“子、们、着”→ lose tone，时长−40%
	- **重读规则**：句首/句尾或语义焦点→ 调域扩大 + 时长+20%
	- **实现**：规则表查换（速度快，内存 0.5 MB）或 CRF 序列标注“表面调”直接输出
6. **步骤5：特殊符号处理**
	- **归一化**：“123”→“一百二十三”；“3×4”→“三乘四”；“……”→“等等”；“！”→ 语调标签
	- **策略**：正则 + 查表 + 微型语法树（分数、日期）
## 7.4 语音合成方法
1. **整体思想**
	- 不做传统“声码器-参数”路线
	- 把音节切成「声母过渡段 + 韵母平稳段（基音同步顿）+ 非平稳段」，直接从库中取出这些小积木 → 按声调曲线调整周期 → 顿-顿叠接（Overlap-Add）→ 合成波形
	- 波形级拼接，但粒度比整音节更细，兼顾韵律灵活性与自然度
2. **最小单元定义(听辨角度)**
	- **音节**：声母 + 韵母 + 声调曲线
	- **听辨角度**：指人耳可分辨的最小音段，此处借用来强调“库中样本=感知片段”
3. **基音同步帧叠接法（PSFC）**
	- （1）**合成基元（库中存什么？）**
		- **声母**
			- 不是单存 /b/, /d/⋯而是存「该声母 + 不同韵母起始 20–40 ms 的过渡段」集合。
			- **好处**：已包含 C→V 共振峰弯折，拼接时自然度↑，join cost↓
		- **韵母**
			- **平稳段**：明显周期，一个周期 = 一个基音同步顿（PS Pitch-Synchronous frame）。
			- **非平稳段**：① 声→韵过渡尾；② 鼻音尾；③ 短促 /ɚ/ 等；单独存成“短砖”
		- **声调曲线**：平稳段中基音同步帧的周期序列，描述声调变化及基音周期变化
			- **周期值序列**：$T = [T1, T2, …, Tn]$（单位：样点或 ms）
			- **库中同时存「声调修正曲线」**：升调模板、降调模板，供后续拉伸/压缩参考
		- **语音时长**
			- 理论上 = ΣTi （所有基音同步帧周期之和）
			- 实际合成时可删/插顿来伸缩时长，PSOLA 同款思路
		- **数据库**
			- ① 汉字-拼音索引库
			- ② 声母库（过渡段波形）
			- ③ 韵母库（平稳段 + 非平稳段波形）
			- ④ 合成基元库（已切好的 C/V 过渡、V 平稳、V 非平稳索引）
			- ⑤ 声调曲线库 & 修正曲线库（周期序列模板）
	- （2）**原理框图**
		![[合成系统原理框图.jpeg|350]]
	- （3）**合成步骤**
		- **Step1 选砖**
			- 输入拼音 “bai2”
				- → 22 声母表取 /b/→/ai/ 对应过渡段
				- → 38 韵母表取 /ai/ 平稳段、非平稳段（鼻尾无，此处为空）
				- → 声调曲线库取阳平“升调模板” $T = [8.0, 7.8, 7.5, 7.2 ms]×n$
		- **Step2 调砖**
			- 用目标 F0 曲线（可由韵律模块给）对每帧周期做线性映射：$T'_i = T_i × (F0_{target} / F0_{original})$
			- 再按新周期值对原帧做「拉伸/压缩 + OLA」→ 得到新平稳段波形
		- **Step3 叠砖**
			- 非平稳段 → 平稳段 → 声母过渡段
			- 顺序叠接（Join 处加 5 ms 汉宁窗 OLA），整体幅度归一 → 输出音节波形
	- （4）**与 PSOLA 的异同**
		- **相同**：都用「基音同步+OLA」做时长/基频修改
		- **不同**：PSOLA 是通用框架，单元可以是任意语料；PSFC 强制“声母过渡+韵母帧“细分库。PSFC 库更小（仅 22×38 组合），适合嵌入式 TTS；PSOLA 需大音库
	- （5）**优缺点**
		- **优点**：波形级、无需参数声码器、运算量小、嵌入式友好
		- **缺点**：库需手工精细切分+标注，鲁棒性低（新词、方言需重新录库）；极端拉伸会听出机械感

2. 基音同步帧周期的调整
3. 合成语音振幅的调整
4. 声调曲线的修正
5. 由文本信息获取合成与修正信息
