# 1. 词嵌入（Word2Vec）
1. 预训练文本表示概览
	- 核心思想：利用大规模无标注语料，通过自监督目标（上下文预测、掩码重建等）先学得通用语言表示，再微调下游任务，避免昂贵人工标注。
	- 静态 vs 动态：
		- word2vec/GloVe：同一词元对应固定向量，无法处理多义；
		- 基于 Transformer 的深层模型（BERT 系列）：上下文相关，同一词在不同句中向量不同，可捕捉多义、句法、语义依赖。
	- 流水线：上游预训练 → 下游微调，一套权重适配情感分析、文本蕴涵、问答等多种任务。
2. Word2vec 要点
	- 独热缺陷：维数=词表大小 𝑉，两两正交，余弦相似度恒为 0，无法表达语义相近。  
	- Word2vec 核心：自监督学习低维稠密向量（通常 50–300 维），使语义/语法相似词在向量空间接近。  
	- 两大模型：
		- Skip-gram：用中心词预测上下文词
		- CBOW：用上下文词预测中心词
	- 训练目标：最大化语料中相邻词的条件概率（负采样或层次 Softmax 加速），无需人工标签。
3. 跳元模型（Skip-Gram）
	- 目标：最大化似然函数
  $$
  \prod_{t=1}^T\prod_{-m\le j\le m,\,j\ne 0}P(w_{t+j}\mid w_t)
  $$
	  其中 $P(w_o\mid w_c)=\displaystyle\frac{\exp(\mathbf u_o^\top\mathbf v_c)}{\sum_{k\in\mathcal V}\exp(\mathbf u_k^\top\mathbf v_c)}$。
	- 参数：每个词两套向量
		- 中心词向量 $\mathbf v_c\in\mathbb R^d$（训练完作为最终词嵌入）
		- 上下文词向量 $\mathbf u_o\in\mathbb R^d$（辅助，训练后弃用）
	- 梯度：需遍历全词表 → 用负采样或层次 Softmax 近似加速。
	- 输出：中心词向量矩阵 $\mathbf V\in\mathbb R^{|\mathcal V|\times d}$ 即语义空间，余弦相似度可直接度量词关系。
4. 连续词袋模型（CBOW）
	- 目标：最大化似然函数
  $$
  \prod_{t=1}^T P\!\bigl(w_t\mid w_{t-m},\dots,w_{t-1},w_{t+1},\dots,w_{t+m}\bigr)
  $$
	  其中上下文向量平均后 $\bar{\mathbf v}=\frac{1}{2m}\sum_{j=1}^{2m}\mathbf v_{o_j}$，再计算  
  $$
  P(w_c\mid\text{context})=\frac{\exp(\mathbf u_c^\top\bar{\mathbf v})}{\sum_{k\in\mathcal V}\exp(\mathbf u_k^\top\bar{\mathbf v})}.
  $$
	- 参数：每个词仍两套向量（向量符号与跳元模型中相反），但训练后取上下文词向量 $\mathbf v_i$ 作为最终词嵌入。
	- 梯度：同样需全词表求和 → 同样用负采样/层次 Softmax 加速。
	- 特点：上下文平均使 CBOW 对高频词更稳、训练速度优于 Skip-gram，但对罕见词效果稍弱。
5. 小结
	- 词向量：表示词意义的向量，或可看作词的特征向量
	- 词嵌入：把词映射到实向量，语义/语法相近词在向量空间距离近。  
	- 两大自监督模型：
		- Skip-gram：中心词 → 预测上下文，中心词向量作最终表示
		- CBOW：上下文平均 → 预测中心词，上下文词向量作最终表示
	- 训练：均用负采样或层次 Softmax 加速，避免全词表归一化；输出 50~300 维静态词向量，可直接用于下游任务或作为深度模型初始值
	- 避免全词表归一化
		- 原始 softmax 中计算归一化概率，分母要对词典里所有词（|V| 个）全局求和，在百万级词表下极其耗时
		- 负采样（Negative Sampling）：只拿“正样本 + 少量负样本”做二分类
		- 层次 Softmax：把全局求和替换成一条二叉树路径上的求和
# 2. 近似训练
1. 负采样
	- 目标：把 softmax 全局归一化改成 K+1 个二分类（1 正 + K 负），每步计算量从 O(|V|) 降到 O(K)
	- 分布 P(w)：常用幂律分布 $∝ U(w)^{3/4}$，既保证高频词被充分负采样，又避免罕见词永远抽不到
	- 损失：对正例最大化 $log\;σ(u_o^⊤v_c)$（正例与中心词越近似），对 K 个噪声词最大化 $log\; σ(-u_k^⊤v_c)$（负例与中心词越远异），等价于带负例的交叉熵
	- 梯度：只涉及每步 K+1 个向量更新，与词表大小无关，可轻松应对百万级词汇。
2. 层序 Softmax
	- 结构：二叉树，|V| 个叶节点对应词表；每个内部节点含向量 $\mathbf u_{\text{node}}$。
	- 概率分解：把“归一化”变成长度 $O(\log_2|V|)$ 的路径连乘  
  $$
  P(w_o\mid w_c)=\prod_{j=1}^{L(w_o)-1}\sigma\!\bigl([\![\text{left}]\!]\cdot\mathbf u_{n(w_o,j)}^\top\mathbf v_c\bigr)
  $$
	  其中 $[\![\text{left}]\!]$ 取 $+1$（下一步左转）或 $-1$（下一步右转）。
	- 特性：
		- 无需计算全词表归一化因子，训练步复杂度从 $O(|V|)$ 降到 $O(\log_2|V|)$
		- 树结构保证 $\sum_{w}P(w\mid w_c)=1$ 严格成立
	- 实现：常用 Huffman 树，高频词路径更短，进一步减少平均计算量。
# 3. 用于预训练词嵌入的数据集
1. PTB 数据集预处理
	- 规模：42 069 句（训练集），约 100 万词，词汇表 6 719 词（min_freq≥10，低频转 `<unk>`
	- 格式：每行已分词，直接按空格切分即可；保留原始 `<unk>` 标记，与自行合并的 `<unk>` 一致
	- 后续：以句为单位采样中心词与上下文，构造 Skip-Gram/CBOW 正例，再按幂律分布抽负样本，组成小批量
2. 下采样
	- 目的：削弱高频词（the/a/in 等）对训练的主导，加速计算并提升低频词嵌入质量
	- 公式：词 w 被保留概率  
  $$
  P(\text{keep})=\min\!\Bigl(1,\ \sqrt{\frac{t}{f(w)}}\Bigr),\quad t=10^{-4}
  $$
	  其中 f(w) 为词频占比；仅当 f(w)>t 时才可能被丢弃，且频率越高丢弃概率越大。 
	- 效果：PTB 中 “the” 从 50 k → 2 k（<5%），句子平均长度缩短约 30%，训练加速；稀有词如 “join” 100% 保留。
3. 中心词-上下文提取
	- 随机窗口：每句中心词固定，窗口大小在 1~max_window_size 均匀采样，保证训练多样性
	- 正例规模：PTB 下采样后共约 1.58 M 对；窗口=5 时，平均每中心词 ≈ 5.3 个上下文词
	- 实现：返回两列表 `centers` 与 `contexts`，一一对应，可直接用于 Skip-Gram 正例 + 负采样。
4. 负采样实现细节
	- 采样权重：按词频0.75 次幂 $（f(w)^{0.75}）$ 构造，削弱高频、提升低频被抽概率，符合 Mikolov 经验
	- 逻辑：每对正例配 K=5 个噪声词；用 `RandomGenerator` 缓存 10 000 个预抽样，保证高效且可复现
	- 过滤：噪声词不能与当前上下文词重复，避免“负例”其实是正例
	- 输出：`all_negatives` 与 `all_contexts` 一一对应，可直接组成 (center, context, negative) 三元组喂入 Skip-Gram 损失。
5. 小批量封装
	- 动态填充：同一 batch 内 context+negative 总数不同，用 0 填充到 max_len，保证张量对齐
	- 掩码 masks：1 表示有效位置，0 对应填充，用于后续损失计算时排除 padding
	- 标签 labels：1 表正例（context），0 表负例（negative），配合掩码可快速实现负采样交叉熵
	- 输出：4 个张量，可直接喂入带掩码的 Skip-Gram 损失函数
		- `centers: (B, 1)`
		- `contexts_negatives: (B, max_len)`
		- `masks: (B, max_len)`
		- `labels: (B, max_len)`
# 4. 预训练word2vec
1. Skip-Gram 前向
	- 嵌入层：
		- `embed_v` 中心词矩阵（|V|×d）
		- `embed_u` 上下文/噪声词矩阵（|V|×d）  
	- 输入：
		- `center (B, 1)` → 查得 `v (B, 1, d)`
		- `contexts_and_negatives (B, max_len)` → 查得 `u (B, max_len, d)`
	- 输出：批量矩阵乘 `v @ u.transpose` → `pred (B, 1, max_len)`即为所有正/负样本的原始得分（后续接 sigmoid 与掩码计算负采样损失）
2. Skip-Gram 训练
	- 损失：二元交叉熵 + 掩码，只计算有效位置，再按掩码平均归一化
		- 参数：两套嵌入均 Xavier 初始化；中心词矩阵 `net[0]` 与上下文矩阵 `net[1]` 共同训练
	- 加速：GPU 上 5 epoch 约几分钟；lr=0.002，Adam 优化，最终 loss ≈ 0.41
	- 输出：训练完取 `net[0].weight` 即中心词向量，可作为静态词嵌入或下游模型初始化
# 5. 全局向量的词嵌入（GloVe）
1. 全局语料统计视角
	- 损失重写：利用全局共现计数 $x_{ij}$（词 $w_j$ 作为 $w_i$ 上下文的次数），Skip-Gram 的负对数似然等价于  
  $$
  \mathcal L = -\sum_{i\in\mathcal V}\sum_{j\in\mathcal V} x_{ij}\log q_{ij}
  $$
	  即让模型条件分布 $q_{ij}$ 逼近经验分布 $p_{ij}=x_{ij}/x_i$。
	- 问题：
		- 归一化 $q_{ij}$ 需遍历全词表，计算昂贵
		- 交叉熵对大量稀有共现 ($x_{ij}=1$) 过敏感，易过拟合，权重失衡
	- 解决思路：放弃精确 softmax，改用负采样或层次 softmax，并引入平方误差（Glove），缓解罕见事件主导
2. GloVe模型
	- 损失函数：平方误差 + 权重 + 偏置  
  $$
  \mathcal L=\sum_{i,j\in\mathcal V}h(x_{ij})\,(\mathbf u_j^\top\mathbf v_i+b_i+c_j-\log x_{ij})^2
  $$
	  其中 $h(x)$ 在 $x<100$ 取 $(x/100)^{0.75}$，否则为 1，抑制罕见共现、保留高频信号
	- 全局统计：非零 $x_{ij}$ 预计算，训练只采样这些项，复杂度 ≈ 非零元素数，与词表大小无关。
	- 对称性：$x_{ij}=x_{ji}$，中心向量 $\mathbf v_i$ 与上下文向量 $\mathbf u_i$ 数学等价；最终表示取二者之和，性能略优。
	- 相比 word2vec：直接拟合对数共现矩阵，无需滑动窗口重复采样，训练更快，内存占用与语料词汇相关。
3. 条件概率比值视角
	- 直觉：共现概率比值 $P(w_k|w_i)/P(w_k|w_j)$ 能区分“与 $w_i$ 相关但 $w_j$ 无关”，“与 $w_j$ 相关但 $w_i$ 无关”，“与两者皆相关/无关” 的词。  
	- 设计目标：寻找标量函数 $f$ 使  
  $$
  f(\mathbf u_j,\mathbf u_k,\mathbf v_i)\approx\frac{P(w_j|w_i)}{P(w_k|w_i)}
  $$
	  取 $f(a)=exp(a)$ 并令 $a=(\mathbf u_j-\mathbf u_k)^\top\mathbf v_i$ 即满足 $f(a)f(-a)=1$
	- 推导：由 $exp(\mathbf u_j^\top\mathbf v_i)\propto P_{ij}=x_{ij}/x_i$ 取对数可得$\mathbf u_j^\top\mathbf v_i+b_i+c_j\approx\log x_{ij}$，加权平方误差即 GloVe 损失，解释其合理性。
		- 目标：用三个词向量拟合共现概率比值  
  $$
  \frac{P(w_j|w_i)}{P(w_k|w_i)} = \exp\!\bigl((\mathbf u_j - \mathbf u_k)^\top \mathbf v_i\bigr)
  $$
		  取对数后得  
  $$
  (\mathbf u_j - \mathbf u_k)^\top \mathbf v_i = \log P(w_j\mid w_i) - \log P(w_k\mid w_i).
  $$
		- 引入全局计数：令 $P(w_j\mid w_i)=x_{ij}/x_i$，上式化为  
  $$
  \mathbf u_j^\top \mathbf v_i - \mathbf u_k^\top \mathbf v_i = \log x_{ij} - \log x_{ki}.
  $$
		- 为使等式对任意 $k$ 成立，把与 $k$ 无关的部分吸收进可学习偏置：  
  $$
  \mathbf u_j^\top \mathbf v_i + b_i + c_j = \log x_{ij}
  $$
		   其中 $b_i$ 中心偏置 $c_j$ 上下文偏置为标量参数
		- 最终损失：对全部非零 $x_{ij}$ 加权最小二乘  
  $$
  \mathcal L=\sum_{i,j} h(x_{ij})\!\bigl(\mathbf u_j^\top \mathbf v_i + b_i + c_j - \log x_{ij}\bigr)^2.
  $$
4. GloVe 小结
	- 全局语料统计：利用词-词共现计数，改 Skip-Gram 负对数似然为平方误差，避免全词表归一化，缓解交叉熵对罕见事件过敏感。
	- 中心词与上下文词：数学等价，训练后取二者之和作为最终词嵌入
	- 条件概率比值：从共现概率比值角度解释 GloVe，设计标量函数拟合比值，引入偏置项，最终得到加权平方误差损失
# 6. 子词嵌入
1. fastText
	- 子词粒度：3–6 字符 n-gram，加边界标记 `< >`；例如 where → `<wh, whe, her, ere, re>, <where>`。  
	- 中心词向量：词 w 的表示 $v_w = Σ_{g∈G_w} z_g$，即所有子词向量之和，实现形态共享
	- 训练：其余与 Skip-Gram 相同（负采样 + 二元交叉熵），但词典变为子词集合，参数量增大
	- 优势：
		- 罕见词 & OOV 词可用其子词组合得到向量，缓解稀疏问题
		- 形态丰富语言（芬兰语、土耳其语）效果优于 word2vec/GloVe
2. 字节对编码（Byte Pair Encoding）
	- 原理：从字符粒度出发，统计词内连续字符对频率，迭代合并最频繁对，生成新符号，直到达到预设合并次数或词表大小
	- 实现：
		- 初始化符号表为字符集 + 特殊符号（UNK、词尾符等）
		- 对词典中词按字符拆分，统计连续字符对频率
		- 找到最频繁对，合并为新符号，更新词典
		- 重复步骤 2-3，直至满足条件。  
	- 优势：
		- 固定词表大小，可容纳任意长度子词
		- 保留形态信息，对形态丰富的语言（如德语、俄语）效果好
		- 应用于 GPT 系列、RoBERTa 等预训练模型，提升对罕见词、词缀的处理能力
3. 子词嵌入小结
	- fastText：基于 Skip-Gram，用子词向量和表示中心词，引入形态信息，缓解罕见词稀疏问题，但计算复杂度高、参数量大
	- BPE：迭代合并最频繁字符对，生成变长子词，固定词表大小，适用于形态丰富语言，广泛用于预训练模型输入表示
	- 子词嵌入优势：提升罕见词、词表外词表示质量，保留形态信息，增强对变形词、词缀的处理能力
# 7. 词的相似性和类比任务
1. 词相似度
	- 利用预训练词向量，通过余弦相似度找到与输入词语义相近的词
	- 实现 `knn` 函数，输入查询词向量、词表向量矩阵，输出最相似的 k 个词及相似度
	- 示例：
		- “chip” → “chips” (0.856)、“intel” (0.749)、“electronics” (0.749) 
		- “baby” → “babies” (0.839)、“boy” (0.800)、“girl” (0.792) 
		- “beautiful” → “lovely” (0.921)、“gorgeous” (0.893)、“wonderful” (0.830)
2. 词类比
	- 任务：a : b :: c : ?，寻找 d 使 $vec(d) ≈ vec(c) + vec(b) − vec(a)$。  
	- 实现：用 `knn` 找最近邻，返回 top-1 词。  
	- 示例：
		- 语义（male-female）：`man-woman :: son- → daughter`
		- 语义（首都-国家）：`beijing-china :: tokyo- → japan`
		- 句法（原级-最高级）：`bad-worst :: big- → biggest`
		- 句法（现在-过去）：`do-did :: go- → went`
	- 结论：预训练词向量同时编码语义与句法关系，支持零样本类比推理
# 8. 来自Transformers的双向编码器表示（BERT）
1. BERT 核心思想
	- 双向上下文：使用 Transformer 编码器同时看左右上下文，解决 GPT 只能左到右的局限；同一词在不同句中向量不同，天然消歧
	- 任务无关架构：预训练仅依赖自监督任务（MLM + NSP），下游只需极轻量微调,换输出层或加简单分类头即可
	- 预训练任务：
		- Masked Language Model（MLM）：随机遮盖15% token，预测原词
		- Next Sentence Prediction（NSP）：判断两句是否相邻，学习句子级关系
2. BERT 输入表示
	- 三元相加：
		- 词元嵌入（token embedding）
		- 片段嵌入（segment embedding）：0 表第一句，1 表第二句
		- 可学习位置嵌入（positional embedding），长度固定 `max_len`
	- 形状：
		- 输入 `tokens (B, seq_len)` + `segments (B, seq_len)`
		- 输出 `(B, seq_len, num_hiddens)`，与 Transformer 编码器一致
	- 特殊标记：
		- `[CLS]` 置于首位，其输出用于分类/下一句预测
		- `[SEP]` 分隔两句，助模型识别句子边界
3. 掩蔽语言模型（MLM）
	- 掩蔽策略：15 % 词元被选中，其中
		- 80 % 替换为 `[MASK]`
		- 10 % 替换为随机词
		- 10 % 保持不变
		- 防止预训练-微调 mismatch，并鼓励模型不过度依赖被掩位置
	- 实现：
		- `MaskLM` 为单层MLP(双层线性)：`num_inputs → num_hiddens → vocab_size`
		- 输入 `encoded_x` (B, L, H) 与 `pred_positions` (B, K) 
		- 输出 `mlm_Y_hat` (B, K, |V|) 
		- 损失：交叉熵仅在被掩位置计算，忽略其余 token
4. 下一句预测（NSP）
	- 任务：二元分类，判断 B 输入序列中第二句是否为第一句的真实下一句
	- 数据构造：50 % 正例（连续句对），50 % 负例（随机句对），标签自动产生，无需人工标注
	- 实现：
		- 取 `[CLS]` 对应隐藏状态 `(B, H)` → 单层线性分类器 → 输出 `(B, 2)`
		- 损失：交叉熵，与 MLM 联合优化
	- 作用：让 `[CLS]` 表示蕴含句子级关系，为下游句子对任务（NLI、QA）提供良好初始表示
5. BERT小结
	- 上下文敏感：同一词在不同句中向量不同，天然处理多义与复杂语义
	- 双向编码：Transformer 编码器同时看左右上下文，弥补 GPT 单向局限
	- 任务无关：预训练后仅需最小架构更改（换输出层或加分类头）即可适配各类下游任务
	- 输入嵌入：词元 + 片段 + 位置 三者相加，支持单句或句对输入
	- 预训练任务：
		- MLM：掩蔽语言模型，双向编码单词
		- NSP：下一句预测，建模句间关系
	- 效果：GLUE、SQuAD 等 11 项 NLP 任务刷新 SOTA，开启“预训练+微调”新范式
# 9. 用于预训练BERT的数据集
1. 预训练数据生成
	- 下一句预测：
		- 以 50 % 概率把下一句或随机句作为 `tokens_b`，返回 `(tokens_a, tokens_b, is_next)`
		- 仅保留两句总长 ≤ `max_len` 的样本，组成 NSP 正/负对
	- 掩蔽语言模型：
		- 候选位置跳过 `[CLS]`/`[SEP]`，随机选 15 % token
		- 80 % → `[MASK]`，10 % → 随机词，10 % → 原词；返回 `(masked_tokens, pred_positions, pred_labels)`
		- 保证每句预测量 ≈ 15 %，用于后续 MLM 交叉熵损失
# 10. 预训练BERT
1. BERT 预训练总结
	- 模型规模：
		- BASE：12 层（Transformer编码器块），768 隐藏，12 头 ≈ 1.1 亿参数
		- LARGE：24 层（Transformer编码器块），1024 隐藏，16 头 ≈ 3.4 亿参数  
	- 表示能力：
		- 同一词元在不同句子中向量不同，验证双向上下文编码有效消歧
	- 使用方式：预训练完成后，无需改动主体，即可提取表示供下游任务使用
		- 整句表示（`[CLS]` 向量）
		- 词级表示（任意 token 向量）
2. `[CLS]` 向量
	- BERT 在每个输入序列最前面的特殊词元`[CLS]`，对应的隐藏状态（最后一层输出的第 0 个位置向量），被当作整个输入序列的语义压缩表示
	- 原因
		- `[CLS]`本身没有语义，可以“公平”地吸收整句信息
		- 预训练时的 NSP 任务用它来做二分类，迫使它学会聚合句子级特征
		- 微调阶段只需在这一向量上加简单分类层即可，无需再对整序列做池化
# 11. 序列级和词元级应用：微调BERT
1. BERT 微调范式总览
	- 统一思路：无论序列级还是词元级，唯一新增一两层全连接；预训练权重全部可微调
	- 序列级（单句）：
		- 输入 `[CLS]` 文本 → 取 `[CLS]` 向量 → 线性层 → 分类/回归输出
		- 例：情感分析、主题分类
	- 序列级（句对）：
		- 输入 `[CLS]` 句A `[SEP]` 句B → `[CLS]` → 线性层 → 三分类（蕴含/中立/矛盾）或相似度打分
		- 例：NLI、语义相似度、问答匹配
	- 词元级：
		- 输入序列得每个 token 向量 → 线性层 → 逐标签预测
		- 例：词性标注（POS）、命名实体识别（BIO 标签）、SQuAD 答案起止位置
	- 优势：一套权重 + 最小顶层改动，端到端微调，GLUE/SQuAD 等十余任务当年一次性刷新 SOTA
# 12. 自然语言推断：微调BERT
1. 自然语言推断
	- 任务类型：文本对三分类（蕴含 / 中立 / 矛盾），属序列级问题
	- 架构：预训练 BERT → `[CLS]` 向量 → 单层 MLP 分类器（隐藏→3 类）
	- 参数策略：
		- 下游层随机初始化
		- BERT 主体可全量微调（也可仅微调顶层）
2. SNLI-BERT 数据集
	- 格式：
		- 每样本 = (前提 tokens, 假设 tokens, 标签)，打包成单条 BERT 输入 `[CLS] 前提 [SEP] 假设 [SEP]`
	- 截断策略：
		- 保留3个特殊符位置，循环pop较长句尾，直到总长度 `≤ max_len − 3`
	- 并行加速：
		- 4 进程 `Pool().map` 把 55 万训练 + 1 万测试句对转成 `(token_ids, segments, valid_len, label)`
	- 输出：
		- `DataLoader`返回 `token_ids` (B, 128) `segments` (B, 128) `valid_len` (B,)  `labels` (B,) 可直接喂给 BERT+MLP 微调
3. 微调 BERT
	- 分类头：仅保留 `bert.encoder` + 单层隐藏 `self.hidden(768→256)` + 输出 `self.output(256→3)`，组成 `BERTClassifier`。  
	- 参数更新：
		- `encoder` 与 `hidden` 全量微调；
		- 预训练 `MaskLM/NSP` 头参数虽在图中，但不被前向调用，梯度 stale，需 `ignore_stale_grad=True`
	- 训练：`Adam lr=1e-4，epoch=5，batch=256，max_len=128`；SNLI 训练集 549 k，测试集 9.8 k
