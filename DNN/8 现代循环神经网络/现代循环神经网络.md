# 1. 门控循环单元（`GRU`）
1. 梯度异常的实际意义
	- 早期关键信息（如校验和）需长期保存，若无机制存储，须赋予极大梯度，易爆炸
	- 无关词元（如HTML标签）应被跳过，避免干扰隐状态
	- 逻辑中断（章节/市场切换）需重置内部状态，防止旧信息污染新段
2. 重置门与更新门
	- 输入 $X_t\in\mathbb{R}^{n\times d}$，上一隐状态 $H_{t-1}\in\mathbb{R}^{n\times h}$
	- 重置门 $R_t = \sigma(X_t W_{xr} + H_{t-1} W_{hr} + b_r)$
	- 更新门 $Z_t = \sigma(X_t W_{xz} + H_{t-1} W_{hz} + b_z)$
	- $W_{xr},W_{xz}\in\mathbb{R}^{d\times h},\ W_{hr},W_{hz}\in\mathbb{R}^{h\times h},\ b_r,b_z\in\mathbb{R}^{1\times h}$
	- `sigmoid` 输出区间 (0,1)，广播机制自动应用于批内样本
3. 候选隐状态
	- 候选隐状态 $\tilde{H}_t = \tanh\!\bigl(X_t W_{xh} + (R_t\odot H_{t-1})W_{hh} + b_h\bigr)$
	- $W_{xh}\in\mathbb{R}^{d\times h},\ W_{hh}\in\mathbb{R}^{h\times h},\ b_h\in\mathbb{R}^{1\times h}$
	- `Hadamard`积 $\odot$ 按元素相乘；$R_t\approx 1$ 时恢复标准`RNN`，$R_t\approx 0$ 时重置旧状态
	- 输出区间 $(-1,1)$
4. 隐状态
	- 最终更新 $H_t = Z_t\odot H_{t-1} + (1-Z_t)\odot\tilde{H}_t$
	- $Z_t\approx 1$：保留旧状态，跳过当前步；$Z_t\approx 0$：采纳候选状态
	- 凸组合缓解梯度消失，长距离依赖可直接传递
		- 因为更新门$Z_t$可以一直≈1，于是$H_t = 1·H_{t-1} + 0·H̃_t$
		- 旧状态被原封不动地复制到下一时刻，不经过任何矩阵乘法或激活函数
		- 既不会缩小也不会放大数值，梯度反向传播时也就不会因为反复连乘而产生 vanishing/exploding
		- 因此从时间步 1 到 T 的信息可以一字节不差地传过去，实现长距离依赖的直接传递
5. `GRU`显著特征
	- 重置门有助于捕获序列中的短期依赖关系
	- 更新门有助于捕获序列中的长期依赖关系
	- 重置门打开时，门控循环单元包含基本循环神经网络；更新门打开时，门控循环单元可以跳过子序列

# 2. 长短期记忆网络（`LSTM`）
1. LSTM记忆元与三门
	- 记忆元C与隐状态H同形，专用于存储附加信息
	- 输入门i：决定何时读入新数据
	- 遗忘门f：决定何时重置单元内容
	- 输出门o：决定何时输出条目
	- 通过专用门控机制显式控制“记/忘/输出”，比GRU更精细
2. LSTM三门计算
	- 输入门 $I_t = \sigma(X_t W_{xi} + H_{t-1} W_{hi} + b_i)$
	- 遗忘门 $F_t = \sigma(X_t W_{xf} + H_{t-1} W_{hf} + b_f)$
	- 输出门 $O_t = \sigma(X_t W_{xo} + H_{t-1} W_{ho} + b_o)$
	-  $W_{x\bullet}\in\mathbb{R}^{d\times h},\,W_{h\bullet}\in\mathbb{R}^{h\times h},\,b_\bullet\in\mathbb{R}^{1\times h}$
	-  `sigmoid` 输出区间(0,1)，同一小批量内广播
3. 候选记忆元
	- 候选记忆元 $\tilde{C}_t = \tanh(X_t W_{xc} + H_{t-1} W_{hc} + b_c)$
	- $W_{xc}\in\mathbb{R}^{d\times h},\,W_{hc}\in\mathbb{R}^{h\times h},\,b_c\in\mathbb{R}^{1\times h}$
	-  输出区间 $(-1,1)$，供输入门筛选后写入记忆元
4. 记忆元更新
	- 记忆元更新 $C_t = F_t\odot C_{t-1} + I_t\odot\tilde{C}_t$ 
	-  $F_t\approx 1,\,I_t\approx 0$：保留全部历史，跳过新输入；梯度无压缩，长依赖可直接传递
	-  设计目的：缓解梯度消失，捕获长距离依赖
5. 隐状态
	- 隐状态 $H_t = O_t\odot\tanh(C_t)$
	- 输出门 $O_t\approx 1$：记忆信息全部送入预测层；$O_t\approx 0$：仅保存记忆元，当前步$H_t\approx 0$
	-  保证 $H_t\in(-1,1)$
6. 长短期记忆网络的隐藏层输出包括“隐状态”和“记忆元”。
	- 只有隐状态会传递到输出层，记忆元完全属于内部信息
	- 长短期记忆网络可以缓解梯度消失和梯度爆炸

# 3. 深度循环神经网络
1. 深度`RNN`
	- 第l层隐藏状态 $H_t^{(l)}=\phi_l\bigl(H_t^{(l-1)}W_{xh}^{(l)}+H_{t-1}^{(l)}W_{hh}^{(l)}+b_h^{(l)}\bigr)$，$H_t^{(0)}=X_t$
	- 输出层 $O_t=H_t^{(L)}W_{hq}+b_q$，$W_{hq}\in\mathbb{R}^{h\times q},\,b_q\in\mathbb{R}^{1\times q}$
	- 隐藏层数$L$与隐藏单元数$h$为超参数；可用`GRU/LSTM`替换隐状态得到深度门控循环神经网络或深度长短期记忆神经网络
2. 高级API实现
```python
from d2l import torch as d2l
batch_size, num_steps = 32, 35 
train_iter, vocab = d2l.load_data_time_machine(batch_size, num_steps)
# 通过num_layers的值来设定隐藏层数
vocab_size, num_hiddens, num_layers = len(vocab), 256, 2 
num_inputs = vocab_size 
device = d2l.try_gpu() 
lstm_layer = nn.LSTM(num_inputs, num_hiddens, num_layers) 
model = d2l.RNNModel(lstm_layer, len(vocab)) 
model = model.to(device)
# 训练
num_epochs, lr = 500, 2 
d2l.train_ch8(model, train_iter, vocab, lr, num_epochs, device)
```
3. 在深度循环神经网络中，隐状态的信息被传递到当前层的下⼀时间步和下⼀层的当前时间步

# 4. 双向循环神经网络
1. 隐马尔可夫模型（hidden Markov model，HMM）
	- 在任意时间步 $t$，设存在某个隐变量 $h_t$，通过概率 $P(a_t \mid h_t)$ 控制我们观测到的 $a_t$
	- 任何 $h_t \rightarrow h_{t+1}$ 转移都是由一些状态转移概率 $P(h_{t+1} \mid h_t)$ 给出。这个概率图模型就是一个HMM
2. HMM 推断目标
	- 观测除 $α_j$ 外的全部序列，求 $P(αⱼ | α_{-j})$
	- 需对 $k^{T}$ 个隐变量组合求和，不可行
	- 动态规划把求和拆成递归，复杂度降至 $O(kT)$
	- 与 RNN 类比：$π_t$ 相当于可学习的 $H_t$，转移/发射概率由 $W,b$ 参数化
3. 前向递归与RNN对比
	- 前向概率递归：$\pi_{t+1}(h_{t+1})=\sum_{h_t}\pi_t(h_t)P(\alpha_t|h_t)P(h_{t+1}|h_t)$
	- 初始化 $\pi_1(h_1)=P(h_1)$；简写 $\pi_{t+1}=f(\pi_t,\alpha_t)$，与RNN隐状态$H_t$更新形式一致
	- 动态规划把指数级求和转为线性递归，复杂度从$O(k^T)$降到$O(kT)$ 
	- 观点：RNN可视为“可学习的前向算法”，隐状态$H_t$被可学习参数$W,b$替代
4. 双向RNN与HMM对比
	- 增加“从后向前”运行的反向隐藏层，形成双向循环网络
	- 前向+后向同时提供上下文，类似HMM前向-后向算法
	- 区别：HMM步骤有明确统计意义，双向RNN仅为可学习通用函数
	- 设计原则：先借鉴经典模型结构，再参数化为深度网络
5. 双向RNN更新公式
	- 前向隐状态 $\overrightarrow{H}_t = \phi(X_t W_{xh}^{(f)} + \overrightarrow{H}_{t-1} W_{hh}^{(f)} + b_h^{(f)})$
	- 反向隐状态 $\overleftarrow{H}_t = \phi(X_t W_{xh}^{(b)} + \overleftarrow{H}_{t+1} W_{hh}^{(b)} + b_h^{(b)})$
	- 合并隐状态 $H_t = [\overrightarrow{H}_t;\overleftarrow{H}_t]\in\mathbb{R}^{n\times 2h}$
	- 输出 $O_t = H_t W_{hq} + b_q,\ W_{hq}\in\mathbb{R}^{2h\times q},\,b_q\in\mathbb{R}^{1\times q}$
	- 两向隐藏单元数可不同，深度网络中$H_t$作为下一双向层输入
6. 双向RNN的局限与应用
	- 训练时可用双向信息，测试时未来未知→预测下一词元精度差
	- 前向+后向+梯度链长，计算慢，训练成本高，双向层使用少
	- 适用：填充缺失词、命名实体识别、机器翻译编码等无需自回归场景

# 5. 机器翻译与数据集
1. 英-法数据集预处理
	- 源语言：英语；目标语言：法语
	- 步骤：空格替换不间断空格、全小写、单词与标点间插空格
	- 清洗后每行保留制表符分隔的双语句对，供后续分词与构造词表
2. 词元化策略
	- 采用单词级词元化（非字符级），每个词元为单词或标点
	- 函数返回两列表：`source[i]、target[i]`分别对应第i句英/法词元序列
	- 为后续构建词表、截断/填充批量提供基础
3. 双语词表构建
	- 分别为源语言与目标语言构建独立词表
	- 单词级词元使词表增大，低频次<2的词元合并为\<unk\>
	- 保留特殊词元：\<pad\>（填充）、\<bos\>（开始）、\<eos\>（结束）
	- 示例：源语言词表大小10012
4. 截断与填充
	- 固定长度num_steps：不足补\<pad\>，超长截断
	- 每句末尾加\<eos\>表示结束
	- 返回填充后数组与有效长度（不含\<pad\>），供后续mask或损失计算使用
5. 数据迭代器与总结
	- `load_data_nmt`返回：`train_iter, src_vocab, tgt_vocab`
	- 每个批量含`(X, X_valid_len, Y, Y_valid_len)`
	- X/Y形状`(batch_size, num_steps)`，`valid_len`记录实际长度（不含pad）
	- 小结：单词级词元化词表大，低频\<unk\>处理；截断/填充保证等长批量
# 6. 编码器-解码器架构
1. 编码器-解码器架构
	- 可以将长度可变的序列作为输入和输出，适用于机器翻译等序列转换问题
	- 编码器：接收长度可变的序列作为输入，将其转换为具有固定形状的编码状态
	- 解码器：将具有固定形状的编码状态映射为长度可变的序列
```python
class EncoderDecoder(nn.Module):
	"""编码器-解码器架构的基类"""
	def __init__(self, encoder, decoder, **kwargs):
		super().__init__(**kwargs)
		self.encoder = encoder
		self.decoder = decoder
	
	def forward(self, enc_X, dec_X, *args):
		enc_outputs = self.encoder(enc_X, *args)
		dec_state = self.decoder.init_state(enc_outputs, *args)
		return self.decoder(dec_X, dec_state)
```

# 7. 序列到序列学习（`seq2seq`）
1. 编码器-解码器seq2seq
	- 编码器：可变长输入→固定形状隐状态，信息被编码到h_T
	- 解码器：以h_T（或每步h_T）为上下文，自回归生成输出；输入首词元\<bos\>，隐状态初始化为编码器最终h_T
	- 训练：采用teacher forcing，标签整体右移一位，目标序列首词元为\<bos\>，末词元为\<eos\>
![[seq2seq.png|425]]
2. 编码器技术细节
	- 单向RNN：$c = h_T$，仅编码“过去”信息
	- 双向RNN：$c = [h→_T; h←_1]$，同时编码整个序列上下文
	- 使用嵌入层：`vocab_size × embed_size` 矩阵，按索引取行向量作为 $x_t$
	- 实现：多层门控循环单元（GRU）作为循环层，输出最后一层最终隐状态或拼接结果得到上下文变量 c
3. 解码器实现要点
	- 隐状态更新 $s_{t'} = g(y_{t'-1}, c, s_{t'-1})$，输入：上一输出 + 上下文 + 上一隐状态
	- 初始隐状态 $s_0 = h_T$（编码器末状态），因此对于RNN，要求编解码器的层数与隐藏单元数一致
	- 每一步将 $c$ 与 $y_{t'-1}$ 拼接后输入解码器，最后一层全连接 + softmax 输出 $P(y_{t'}|y_{<t'},c)$
	- 训练用 teacher-forcing，测试可自回归采样
4. 填充屏蔽与损失扩展
	- sequence_mask：按valid_len把填充位置置0（或指定值），支持多维
	- 损失计算：初始掩码全置为1，根据有效长度将填充对应位置置0；交叉熵损失乘以掩码，排除\<pad\>贡献
	- 结果：仅有效词元参与梯度更新，防止填充干扰训练
5. 强制教学（teacher forcing）
	- 训练阶段：把“\<bos\>”与原始输出序列（不含\<eos\>）拼成解码器输入，即标签作为下一步输入
	- 替代方案：用上一时间步模型预测词元作为当前输入（自回归）
	- 好处：加速收敛、减少误差累积；
	- 缺点：训练与测试分布不一致
6. 自回归预测
	- 测试阶段：解码器输入为上步时间预测词元，首步输入\<bos\>
	- 逐词生成，直到输出\<eos\>或达最大长度
	- 过程：编码器一次性编码源序列；解码器循环生成，每步用最新预测作为下一步输入
7. BLEU（bilingual evaluation understudy）评估
	- 常用的评估方法，通过测量预测序列和标签序列之间n-gram的匹配度来评估预测
	- 定义 $\mathrm{BLEU} = \exp\!\left(\min\!\left(0,\,1-\frac{\mathrm{len_{label}}}{\mathrm{len_{pred}}}\right)\right)\prod_{n=1}^{k}p_n^{1/2^n}$
	- $k$：用于匹配的最长n-gram；$p_n$：n-gram精确度 = 匹配数 / 预测n-gram总数
	- 完美匹配时BLEU=1；长n-gram权重更高；长度惩罚项抑制过短输出
# 8. 束搜索
1. 贪心搜索
	- 每步仅取局部最大 $y_t^*=\arg\max_y P(y|y_{<t},c)$
	- 无法保证全局最优，早期次优选择可能引导出更高整体概率序列
2. 穷举搜索
	- 枚举全部$|V|^T$序列，选条件概率最大者，保证最优
	- 复杂度$O(|V|^T)$，现实不可行，仅理论基准，实际使用束搜索或贪心搜索近似
3. 束搜索
	- 步1：选条件概率最高的k个词元作为k条候选首词
	- 步t>1：对每条候选续$|V|$个词，保留所有候选中得分最高的k条新序列
	- 终止：出现\<eos\>或达最大长度即停止，返回得分最高序列
	- 超参数k控制精度-速度权衡；可结合长度归一化避免过短输出
	- 束搜索细节
		- 束宽k=2，每步保留top-k序列
		- 得分：长度归一化log概率和 $\dfrac{1}{L^\alpha}\sum_{t=1}^L\log P(y_t|y_{<t},c)$，α=0.75抑制长序列
		- 复杂度$O(k|V|T)$，介于贪心$(O(|V|T))$与穷举$(O(|V|^T))$之间
		- $k=1$ 即退化为贪心搜索
		- $k = |V|^{T}$即退化为穷举搜索，top-k 就等价于全部候选，不再做剪枝
