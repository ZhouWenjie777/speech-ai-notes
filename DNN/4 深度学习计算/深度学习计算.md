# 1. 层和块
1. 块（block）：描述单个层、多个层组成的组件、整个模型
2. 从编程的角度，块由类nn.Module表示。由于自动微分，定义块时只需考虑前向传播函数forward()和构造函数__init__()，除非实现⼀个新的运算符，否则不必担心反向传播函数或参数初始化，系统将自动生成这些。
```python
class MLP(nn.Module):
	# 模型参数声明层。这⾥，声明两个全连接的层
	def __init__(self):
		super().__init__()
		self.hidden = nn.Linear(20, 256) # 隐藏层
		self.out = nn.Linear(256, 10) # 输出层
	
	# 定义模型的前向传播，即如何根据输入X返回所需的模型输出
	def forward(self, X):
		# 注意，这⾥我们使⽤ReLU的函数版本，其在nn.functional模块中定义。
		return self.out(F.relu(self.hidden(X)))
```
3. 构建顺序块MySequential()：
	- 每个`nn.Module`都带一个`_modules`属性，让子模块既能被注册，又能按定义顺序被找到，还能保证保存/加载 checkpoint 时顺序不变。但工程上永不推荐直接写，而是通过`nn.ModuleList`走公共 API，内部调`self.register_module`
	- “不注册”后果
		- 不会进入  `_parameters` → 优化器看不到
		-  不会进入 `_buffers` →`state_dict()`里没有它，`torch.save()` 也不会存它
		- 设备不联动 → 调用`model.cuda()`后它仍留在 CPU
		- 前向还能用，但一保存/加载就掉
```python
class MySequential(nn.Module):
	def __init__(self, *args):
		super().__init__()
		for idx, module in enumerate(args):
		# 变量_modules中。module的类型是OrderedDict
		self._modules[str(idx)] = module
		
		# self.layers = nn.ModuleList(args)
	
	def forward(self, X):
		# OrderedDict保证了按照成员添加的顺序遍历它们
		for block in self._modules.values():
			X = block(X)
		return X
```
4. 如何注册

| 目标        | 做法                               |
| --------- | -------------------------------- |
| 动态/列表式子模块 | 用`nn.ModuleList`或`nn.Sequential` |
| 需要学习的额外张量 | 用`nn.Parameter`包装                |
| 不需要学习的缓存  | 用`self.register_buffer`          |
```python
class GoodBlock(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.ModuleList([nn.Linear(10, 10) for _ in range(3)])
        self.weight = nn.Parameter(torch.randn(5, 5))         # 可训练
	    self.register_buffer('rand_weight', torch.ones(5, 5)) # 不训练，但存档
```

# 2. 参数管理
1. 参数访问：通过Sequential类定义模型时，模型如同列表，可以通过索引来访问任意层，每层的参数都在其属性中。`print(net[2].state_dict())`
	- 参数：参数都表示为参数类的实例。参数是复合的对象，包含值、梯度和额外信息：`print(net[2].bias)`，`net[2].bias.data`，`net[2].weight.grad`
	- ⼀次性访问所有参数：`print(*[(name, param.shape) for name, param in net.named_parameters()])`
	- 从嵌套块收集参数：因为层是分层嵌套的，所以也可以像通过嵌套列表索引⼀样访问。`net = nn.Sequential(), net.add_module(f'block {i}', block1())`。如访问第⼀个主要的块中、第二个子块的第⼀层偏置项`rgnet[0][1][0].bias.data`
2. 参数初始化：默认情况下，PyTorch会根据⼀个范围均匀地初始化权重和偏置矩阵，这个范围是根据输入和输出维度计算出的
	- 内置初始化
		- `nn.init.normal_(m.weight,mean=0,std=0.01)` 
		- `nn.init.zeros_(m.bias)` 
		- `nn.init.constant_(m.weight, 1)` 
		- `nn.init.xavier_uniform_(m.weight)` 
	- 自定义初始化：直接设置参数，如`net[0].weight.data[:] += 1` 
3. 参数绑定：给共享层⼀个名称，以便引用
	- `shared = nn.Linear(8, 8), net = nn.Sequential(shared, nn.ReLU(), shared)`
	- 反向传播期间共享层的梯度会加在⼀起
# 3. 延后初始化
1. 延后初始化（defers initialization）：先搭模型结构，数据第⼀次通过模型传递时，框架会动态地推断出每个层的大小。
2. 有了该技术。在编写代码时无须知道维度就可以设置参数，这大大简化定义和修改模型的任务
# 4. 自定义层
1. 不带参数的层：只需继承基础层类并实现前向传播功能
```python
import torch.nn.functional as F
from torch import nn
class CenteredLayer(nn.Module):
	def __init__(self):
		super().__init__()
	
	def forward(self, X):
		return X- X.mean()
```
2. 带参数的层：使用内置函数来创建参数
```python
class MyLinear(nn.Module):
	def __init__(self, in_units, units):
		super().__init__() 
		self.weight = nn.Parameter(torch.randn(in_units, units)) 
		self.bias = nn.Parameter(torch.randn(units,)) 
	
	def forward(self, X): 
		linear = torch.matmul(X, self.weight.data) + self.bias.data 
		return F.relu(linear)
```

3.`model.eval()`是 “把模型切换到推理模式” 的开关，最主要的行为就是`Dropout`和`BatchNorm`，忘记切模式，Dropout会乱dropout，BN会乱算，推理结果不稳定甚至直接错误。

| 模块        | 训练模式 ( model.train() ) | 推理模式 ( model.eval() ) |
| --------- | ---------------------- | --------------------- |
| Dropout   | 按概率随机置 0（正则）           | 全部保留，不做随机丢弃           |
| BatchNorm | 用当前批次的均值/方差并更新运行统计量    | 用历史上累积的运行均值/方差，不再更新   |
4. 同时加载和保存“架构 + 参数”
```python
save_path = 'mlp_full.tar'
torch.save({'net': net,                  # 整个模型对象（含架构）
            'state': net.state_dict(),   # 再存一份参数，可恢复也可单独用
            'epoch': 10},                # 或其他训练实验记录
           save_path)
print('已保存完整模型！')

ckpt = torch.load(save_path)             # 把文件读回来
clone = ckpt['net']                      # 架构+参数都在
clone.eval()                             # 切换到推理模式
```
- 采用“架构+参数”完整 pickle 方式时：
	- 类定义必须在加载时能 import 到（同名、同路径、同构造函数签名）；
	- 不能出现  lambda 、嵌套函数、未绑定外部类等无法 pickle 的对象；
# 6. GPU
1. 使用`nvidia-smi`命令查看显卡信息。
2. 在PyTorch中，每个数组都有⼀个设备（device），我们通常将其称为环境。默认情况下，数据在主内存中创建，然后使用CPU进行计算
3. 在PyTorch中，CPU和GPU用`torch.device('cpu')`和`torch.device('cuda')`表示
4. 如果有多个GPU，使用`torch.device(f'cuda:{i}')`来表示。cuda:0和cuda是等价的
5. 使用`torch.cuda.device_count()`查询可用gpu的数量。
6. 深度学习框架要求计算的所有输入数据都在同一设备上。
```python
# 1.查看硬件
nvidia-smi                  # 驱动 / 显存 / 进程
nvidia-smi -L               # 只列 GPU 序号与名称
torch.cuda.device_count()   # 可用 GPU 数量
torch.cuda.get_device_name(0)  # 0 号卡名称

# 2.device 对象
device_cpu  = torch.device('cpu')
device_gpu0 = torch.device('cuda:0')   # 第 1 张卡
device_gpu1 = torch.device('cuda:1')   # 第 2 张卡

# 3.创建时直接放到 GPU（省一次内存复制）
X = torch.ones(2, 3, device=device_gpu0)   # 显存里诞生
Y = torch.randn(2, 3, device=device_gpu0)  # 同上

# 4.已存在张量的跨设备移动
X_cpu = torch.ones(2, 3)        # 先在内存
X_gpu = X_cpu.to(device_gpu0)   # 复制到 0 号卡
X_gpu = X_cpu.cuda(0)           # 等价写法

# 5.模型整体搬运，cuda()/to()都是“返回新对象”，不是“原地改”；忘记重新赋值等于没搬
net = MyModel().to(device_gpu0)  # 所有参数 + buffer 一起过去
net = net.cuda()   # 必须重新赋值，把GPU版重新赋给net

# 6.计算图中所有张量 + 模型参数必须在同一设备。推荐写法：先选device，再统一to
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = MyModel().to(device)
```
