# 1. 注意力提示
1. 生物学中的注意力提示
	- 注意力是稀缺资源，需机会成本支付；环境信息远超限，只能聚焦小部分
	- 双组件框架：
		- 非自主性提示：突出性/易见性驱动（红杯子自动吸引黄斑区）
		- 自主性提示：认知与主观意愿控制（主动聚焦书本）
	- 黄斑区（macula）
		- 视网膜中心、正对瞳孔的一块高分辨率感光区域，含最密集的视锥细胞
		- 负责清晰锐利的中/高视力（阅读、人脸识别等）
		- 当注意力被突出刺激自动吸引时，大脑会把眼球转到使目标影像恰好落在黄斑区的方向，从而获得最清晰的视觉信息——这就是“非自主性提示”的生理基础
	- 大脑利用历史数据（经验）学习“什么值得看”，实现资源明智分配
2. 查询、键与值
	- 自主性提示：查询(query)
	- 感官输入：值(value)
	- 每个值附带非自主性提示：键(key)
	- 注意力汇聚(attention pooling)：query与key匹配，得到最相关value的加权组合
	- 注意力机制区别于全连接/汇聚层：显式引入query实现自主选择性
# 2. 注意力汇聚：Nadaraya-Watson核回归
1. 非参数注意力汇聚
	- 定义 $f(x) = \sum_{i=1}^n \alpha(x,x_i)y_i$，权重 $\alpha(x,x_i)=\frac{K(x-x_i)}{\sum_{j=1}^n K(x-x_j)}$
	- 高斯核 $K(u)=\exp\!\bigl(-\tfrac{u^2}{2}\bigr)$ ⇒ $weight=softmax(-\tfrac{1}{2}‖x-x_i‖²)$
	- 键 $x_i$ 接近查询 $x$ 则权重 $\alpha(x,x_i)$ 大，实现“非参数平滑加权平均”
	- 模型无参数，仅由训练点${(x_i,y_i)}$决定，预测线平滑且贴近真实
2. 带参数注意力汇聚
	- 在查询与键距离中引入可学习参数 $w：α(x,x_i)=softmax(-\tfrac{1}{2}((x−x_i)w)²)$
	- 模型仍保持高斯核形式，但带宽由数据学习，收敛更快
	- 目标：通过最小化训练误差端到端学习 w，使注意力权重更贴合任务需求
3. 带参模型平滑度下降
	- 带参模型学习 $w$ 后，注意力权重 $α(x,x_i)$ 在高密度区更尖锐（峰值大）
	- 权重更集中：加权平均仅由少数邻近点主导，曲线出现“过拟合”尖峰
	- 非参数模型权重固定（高斯带宽固定），天然平滑；可学习 $w$ 放大局部差异，降低平滑度
	- 缓解：增加正则项、减小学习率、采用核带宽平滑约束
4. 批量矩阵乘法
	- 输入张量形状(n,a,b)与(n,b,c)，输出(n,a,c)
	- 示例：`torch.bmm(X,Y)`完成n组独立矩阵乘法
	- 注意力场景：`weights(n,1,L) @ values(n,L,1)→(n,1,1)`，实现小批量加权平均
	- 避免 for-loop，GPU并行加速
```python
X = torch.ones((2, 1, 4)) 
Y = torch.ones((2, 4, 6)) 
torch.bmm(X, Y).shape 
# torch.Size([2, 1, 6])

weights = torch.ones((2, 10)) * 0.1 
values = torch.arange(20.0).reshape((2, 10)) 
torch.bmm(weights.unsqueeze(1), values.unsqueeze(-1)).shape 
# torch.Size([2, 1, 1])
```
5. 总结
	- Nadaraya-Watson核回归 = 带注意力机制的加权平均，权重 = 键-查询相似度函数
	- 分非参数型（固定核）与带参数型（可学习 $w$ ），后者收敛更快但可能过拟合局部尖峰
	- 为后续自注意力、多头注意力奠定“键-查询-值”框架基础
# 3. 注意力评分函数
1. 注意力评分函数框架
	- 评分函数 $a(\mathbf q,\mathbf k)$ 将查询-键映射到标量，经 `softmax` 得注意力权重
	- 输出 $f(\mathbf q,\{\mathbf k_i,\mathbf v_i\}) = \sum_i \mathrm{softmax}\bigl(a(\mathbf q,\mathbf k_i)\bigr)\mathbf v_i \in \mathbb R^v$
	- 不同 $a(\cdot,\cdot)$ 对应不同注意力机制；下文介绍两种主流评分函数
2. 掩蔽 softmax
	- 指定有效长度 $\ell$，对超出位置掩蔽：$\text{mask}_{ij}=0\;(j> \ell_i)$
	- logits 置 $-\infty$ 后再 softmax，掩蔽输出 $\tilde{\alpha}_{ij}=\begin{cases}0 & j> \ell_i\\\displaystyle\frac{\exp(l_{ij})}{\sum_{k\le\ell_i}\exp(l_{ik})}&j\le\ell_i\end{cases}$
	- 用途：忽略填充 $<pad>$，仅对有意义词元求注意力权重
	- 实现：$\text{masked\_softmax}(L,\boldsymbol\ell)$，内部生成长度掩码，返回同形的概率矩阵
```python
def masked_softmax(X, valid_lens):
	"""通过在最后⼀个轴上掩蔽元素来执⾏softmax操作"""
	# X:3D张量，valid_lens:1D或2D张量
	if valid_lens is None:
		return nn.functional.softmax(X, dim=-1)
	else:
		shape = X.shape
		if valid_lens.dim() == 1:
			valid_lens = torch.repeat_interleave(valid_lens, shape[1])
		else:
			valid_lens = valid_lens.reshape(-1)
		# 最后一轴上被掩蔽的元素使用⼀个非常大的负值替换，从而其softmax输出为0
		X = sequence_mask(X.reshape(-1, shape[-1]), 
							  valid_lens, value=-1e6)
		return nn.functional.softmax(X.reshape(shape), dim=-1)

def sequence_mask(X, valid_len, value=0): 
	"""在序列中屏蔽不相关的项""" 
	maxlen = X.size(1) 
	mask = torch.arange(maxlen, dtype=torch.float32, 
						 device=X.device)[None, :] < valid_len[:, None] 
	X[~mask] = value
	return X

"""考虑由2X4矩阵表示的两个样本，valid_lens:1D 指定两个样本的有效长度"""
masked_softmax(torch.rand(2, 2, 4), torch.tensor([2, 3]))

"""考虑由2X4矩阵表示的两个样本，valid_lens:2D 指定两个样本的有效长度"""

```
3. 加性注意力评分函数
	- 查询 $\mathbf q\in\mathbb R^q$，键 $\mathbf k\in\mathbb R^k$
	- 评分函数 $a(\mathbf q,\mathbf k) = \mathbf w^\top \tanh(W_q\mathbf q + W_k\mathbf k)\in\mathbb R$
	- 可学习参数：$W_q\in\mathbb R^{h\times q},\;W_k\in\mathbb R^{h\times k},\;\mathbf w\in\mathbb R^h$
	- 隐藏层 h 为超参数；tanh 激活，无偏置
```python
classAdditiveAttention(nn.Module):
	"""加性注意力"""
	def __init__(self,key_size,query_size,num_hiddens,dropout,**kwargs):
		super().__init__(**kwargs)
		self.W_k = nn.Linear(key_size, num_hiddens, bias=False)
		self.W_q = nn.Linear(query_size, num_hiddens, bias=False)
		self.w_v = nn.Linear(num_hiddens, 1, bias=False)
		self.dropout = nn.Dropout(dropout)
	
	def forward(self, queries, keys, values, valid_lens):
		queries, keys = self.W_q(queries), self.W_k(keys)
		#在维度扩展后， 
		# queries的形状：(batch_size，查询的个数，1，num_hidden) 
		# key的形状：(batch_size，1，“键－值”对的个数，num_hiddens) 
		# 使用广播方式进⾏求和
		features = queries.unsqueeze(2) + keys.unsqueeze(1)
		features = torch.tanh(features)
		# self.w_v仅有⼀个输出，因此从形状中移除最后那个维度。 
		# scores的形状：(batch_size，查询的个数，“键-值”对的个数)
		scores = self.w_v(features).squeeze(-1)
		self.attention_weights = masked_softmax(scores, valid_lens)
		# values的形状：(batch_size，“键－值”对的个数，值的维度)
		returntorch.bmm(self.dropout(self.attention_weights), values)

"""示例"""
# 查询、键和值的形状为（批量大小，步数或词元序列长度，特征大小）
# 注意力汇聚输出的形状为（批量大小， 查询步数，值的维度）

# values的小批量，两个值矩阵是相同的

							  dropout=0.1)
attention.eval() 
attention(queries, keys, values, valid_lens)
```
4. 缩放点积注意力
	- 评分函数 $a(\mathbf q,\mathbf k)=\dfrac{\mathbf q^{\top}\mathbf k}{\sqrt{d}}$
	- 方差稳定：独立零均值单位方差下点积方差为$d$，除以$\sqrt{d}$后恒为1
	- 批量形式 $\mathrm{softmax}\!\left(\dfrac{QK^{\top}}{\sqrt{d}}\right)V$，其中$Q\in\mathbb{R}^{n\times d},K\in\mathbb{R}^{m\times d},V\in\mathbb{R}^{m\times v}$，输出$\mathbb{R}^{n\times v}$
	- 实现可加入dropout正则化
```python
class DotProductAttention(nn.Module):
	"""缩放点积注意力"""
	def __init__(self, dropout, **kwargs):
		super().__init__(**kwargs)
		self.dropout = nn.Dropout(dropout)
	# queries的形状：(batch_size，查询的个数，d) 
	# keys的形状：(batch_size，“键－值”对的个数，d) 
	# values的形状：(batch_size，“键－值”对的个数，值的维度) 
	# valid_lens的形状：(batch_size，)或者(batch_size，查询的个数)
	
	def forward(self, queries, keys, values, valid_lens=None):
		d = queries.shape[-1]
		scores = torch.bmm(queries, keys.transpose(1,2)) / math.sqrt(d)
		self.attention_weights = masked_softmax(scores, valid_lens)
		return torch.bmm(self.dropout(self.attention_weights), values)
```
5. 小结
	- 输出均为值的加权平均，评分函数不同→注意力权重分布不同
	- 查询键长度不同：采用可加性注意力；长度相同：缩放点积注意力计算效率更高
	- 点-积是相同长度下的首选评分函数
		- 点积 $qᵀk$ 各自方向与幅度的完整交互（余弦×模长）$\mathbf{q}^\top\mathbf{k} = \|\mathbf{q}\|\cdot\|\mathbf{k}\|\cdot\cos\theta$；
		- 向量求和既损失交互，又增加复杂度（额外将向量映射到标量），无显著优势
# 4. Bahdanau注意力
1. Bahdanau可微注意力
	- 不再使用单一固定上下文变量，而为每个解码步 t′ 动态计算上下文向量
	- 上下文 = 注意力加权求和：$\displaystyle \mathbf{c}_{t'}=\sum_{t=1}^{T}\alpha_{t',t}\mathbf{h}_t$  
	- 对齐权重 $\alpha_{t',t}\propto\exp\!\bigl(a(\mathbf{s}_{t'-1},\mathbf{h}_t)\bigr)$ 由当前解码隐状态 $\mathbf{s}_{t'-1}$ （查询）与编码器各隐状态 $\mathbf{h}_t$ （健和值）相关性决定。具体可使用上节所定义的加性注意力评分函数计算
	- 实现“软对齐”——解码器仅关注源序列中与当前预测相关的部分
2. 带有Bahdanau注意力的RNN解码器
	- 继承Decoder，新增attention_weights属性，供可视化对齐矩阵
	- 初始化解码器状态
		- 编码器末层全部隐状态$\mathbf{h}_{1:T}$→作为键值
		- 末时刻隐状态$\mathbf{h}_T$→初始化解码器隐状态$\mathbf{s}_0$
		- 有效长度→屏蔽\<pad\>，避免注意力落在填充位置
	- 每步解码：上一隐状态$\mathbf{s}_{t'-1}$作查询，得到注意力输出$c_{t'}$，再与上一预测词元嵌入$e_{t'-1}$沿特征维拼接，后输入GRU计算
		- 当前应该关注哪些源语言信息（来自 $c_{t'}$）
		- 上一步生成了什么词（来自 $e_{t'-1}$）
		- 基于这两部分信息决定当前步 t′ 隐状态和输出分布
3. 小结
	- 上下文变量=加性注意力池化输出，解码每一步动态关注源序列相关部分
	- 查询：上一解码隐状态；键/值：编码器全部时间步隐状态
	- 实现软对齐，缓解固定上下文信息瓶颈，提升长序列翻译质量
# 5. 多头注意力
1. 多头注意力动机
	- 同一注意力机制需捕捉不同范围依赖（短/长距离）
	- 使用h组独立可学习线性投影将Q,K,V映射到h个子空间
	- 各子空间并行计算注意力→h个输出头
	- 最终拼接h个头并通过线性层融合，形成 richer 表示
2. 多头注意力数学形式
	- 给定查询 $\mathbf q\in\mathbb R^{d_q}$、键 $\mathbf k\in\mathbb R^{d_k}$、值 $\mathbf v\in\mathbb R^{d_v}$
	- 第 i 头：$\mathbf h_i = f\!\bigl(W_i^{(Q)}\mathbf q,\,W_i^{(K)}\mathbf k,\,W_i^{(V)}\mathbf v\bigr)\in\mathbb R^{p_v}$
	- 投影矩阵：$W_i^{(Q)}\in\mathbb R^{p_q\times d_q},\,W_i^{(K)}\in\mathbb R^{p_k\times d_k},\,W_i^{(V)}\in\mathbb R^{p_v\times d_v}$
	- 输出融合：$\mathrm{MultiHead}(\mathbf q,\mathbf k,\mathbf v)=W_O\!\bigl[\mathbf h_1;\cdots;\mathbf h_h\bigr],\;W_O\in\mathbb R^{p_o\times h p_v}$
	- 每个头关注不同子空间，拼接后线性变换得到最终表示
3. 多头注意力实现细节
	- 设定 $p_q = p_k = p_v = p_o / h$，避免参数量爆炸
	- 四个线性变换输出维度 $p_q h = p_k h = p_v h = p_o$，可一次性并行计算 $h$ 个头
	- 使用缩放点积注意力作为每个头的汇聚函数
	- 参数 `num_hiddens` 对应 $p_o$，实现中通过 reshape 将批量拆成 $h$ 个头并行计算
```python
class MultiHeadAttention(nn.Module):
	"""多头注意力"""
	def __init__(self, key_size, query_size, value_size, num_hiddens, 
				 num_heads, dropout, bias=False, **kwargs):
		super().__init__(**kwargs)
		self.num_heads = num_heads
		self.attention = d2l.DotProductAttention(dropout)
		self.W_q = nn.Linear(query_size, num_hiddens, bias=bias)
		self.W_k = nn.Linear(key_size, num_hiddens, bias=bias)
		self.W_v = nn.Linear(value_size, num_hiddens, bias=bias)
		self.W_o = nn.Linear(num_hiddens, num_hiddens, bias=bias)
		
	def forward(self, queries, keys, values, valid_lens):
		# queries，keys，values的形状：
		# (batch_size，查询或“键-值”对的个数，num_hiddens)
		# valid_lens 的形状：(batch_size，)或(batch_size，查询的个数)
		# 经过并行变换后，queries，keys，values 的形状:
		# (batch_size*num_heads，查询或“键-值”对的个数，num_hiddens/num_heads)
		queries = transpose_qkv(self.W_q(queries), self.num_heads)
		keys = transpose_qkv(self.W_k(keys), self.num_heads)
		values = transpose_qkv(self.W_v(values), self.num_heads)
		
		if valid_lens is not None:
			# 在轴0，将第⼀项（标量或者矢量）复制num_heads次，然后如此复制之后项
			valid_lens = torch.repeat_interleave(valid_lens, 
							repeats = self.num_heads, dim=0)
			
		# output的形状：(batch_size*num_heads，查询数，num_hiddens/num_heads)
		output = self.attention(queries, keys, values, valid_lens)
		
		# output_concat的形状：(batch_size，查询数，num_hiddens)
		output_concat = transpose_output(output, self.num_heads)
		return self.W_o(output_concat)

def transpose_qkv(X, num_heads):
	"""为了多注意力头的并行计算而变换形状"""
	# 输入形状：(batch_size，查询或“键-值”对的个数，num_hiddens)
	# 输出形状：(batch_size*num_heads，查询或“键-值”对数，num_hiddens/num_heads)
	X = X.reshape(X.shape[0], X.shape[1], num_heads, -1)
	X = X.permute(0, 2, 1, 3)
	return X.reshape(-1, X.shape[2], X.shape[3])
	
def transpose_output(X, num_heads):
	"""逆转transpose_qkv函数的操作"""
	X = X.reshape(-1, num_heads, X.shape[1], X.shape[2])
	X = X.permute(0, 2, 1, 3)
	return X.reshape(X.shape[0], X.shape[1], -1)

"""示例测试"""
num_hiddens, num_heads= 100, 5
attention = MultiHeadAttention(num_hiddens, num_hiddens, num_hiddens, 
							   num_hiddens, num_heads, 0.5)
attention.eval()
batch_size, num_queries = 2, 4
num_kvpairs, valid_lens = 6, torch.tensor([3, 2])
X = torch.ones((batch_size, num_queries, num_hiddens)) 
Y = torch.ones((batch_size, num_kvpairs, num_hiddens))
attention(X, Y, Y, valid_lens).shape # torch.Size([2, 4, 100])
```
4. 多头注意力小结
	- 融合h个头不同子空间知识，统一查询、键、值空间下学习多种依赖模式
	- 并行实现：一次矩阵乘法完成h组投影，再reshape分头计算并拼接
	- 为标准Transformer/自注意力机制核心组件，提升表示容量与训练效率
	- 对于一个训练好的多头注意力模型，可以设计实验修剪最不重要的头以提高预测速度。设计核心思路：让每个头“单独缺席”一次，看任务指标下降多少，下降越大，头越重要。
# 6. 自注意力和位置编码
1. 自注意力机制
	- 查询、键、值来自同一输入序列，称为自注意力（self-attention）
	- 输入 $\mathbf{x}_1,\dots,\mathbf{x}_n$，输出同长序列 $\mathbf{y}_i=\sum_j\alpha_{ij}\mathbf{x}_j$
	- 权重 $\alpha_{ij}=\mathrm{softmax}\!\left(\dfrac{\mathbf{x}_i^\top\mathbf{x}_j}{\sqrt{d}}\right)$，捕获全局依赖
	- 实现：多头注意力对张量（批量大小, 序列长度, d）计算，输出形状相同
2. 架构对比总结
	- 计算复杂度：RNN $O(nd²)$，CNN $O(knd²)$，自注意力 $O(n²d)$
	- 顺序操作：RNN $O(n)$，CNN $O(1)$，自注意力 $O(1)$ → 后两者可并行
	- 最大路径长度：RNN $O(n)$，CNN $O(n/k)$，自注意力 $O(1)$ → 自注意力最有利于捕获长距离依赖
	- 代价：自注意力随序列长度呈二次方增长，极长序列计算昂贵
3. 位置编码
	- 目的：为并行化的自注意力注入绝对/相对位置信息
	- 固定位置编码：$P\in\mathbb{R}^{n\times d}$，$P_{i,2j}=\sin\!\Bigl(\dfrac{i}{10000^{2j/d}}\Bigr),\;P_{i,2j+1}=\cos\!\Bigl(\dfrac{i}{10000^{2j/d}}\Bigr)$
	- 输出：$\mathbf{X}+\mathbf{P}$，直接与词元嵌入相加；维度不变，计算高效
```python
class PositionalEncoding(nn.Module):
	"""位置编码"""
	def __init__(self, num_hiddens, dropout, max_len=1000):
		super().__init__()
		self.dropout = nn.Dropout(dropout)
		# 创建⼀个足够长的P
		self.P = torch.zeros((1, max_len, num_hiddens))
		X = torch.arange(max_len, dtype=torch.float32).reshape(-1, 1) 
			/ torch.pow(10000, torch.arange(0, num_hiddens, 2,
						dtype=torch.float32) / num_hiddens)
		# X的Shape为：(max_len, num_hiddens/2)
		# 算术广播可以“缺头维”，索引赋值不行且只能复制长度为 1 的轴
		X = X.unsqueeze(0)
		self.P[:, :, 0::2] = torch.sin(X)
		self.P[:, :, 1::2] = torch.cos(X)
	
	def forward(self, X):
		X = X + self.P[:, :X.shape[1], :].to(X.device)
		return self.dropout(X)
```
4. 位置编码解释
	- 沿嵌入维度单调降频：低维对应高频（短周期），高维对应低频（长周期）
	- 类比二进制位：低位交替快，高位交替慢；位置编码用三角函数实现类似“位”结构
	- 此类连续表示法比二进制表示法更节省空间，可唯一标识0~n-1位置，同时便于梯度传播
5. 相对位置信息
	- 对于任意偏移 $s$，$\mathbf p_{i+s}$ 可表示为 $\mathbf p_i$ 的线性投影：$\begin{bmatrix}\!\cos(sw_j)&\!\sin(sw_j)\\[2pt]-\!\sin(sw_j)&\!\cos(sw_j)\end{bmatrix}\begin{bmatrix}p_{i,2j}\\p_{i,2j+1}\end{bmatrix}=\begin{bmatrix}p_{i+s,2j}\\p_{i+s,2j+1}\end{bmatrix}$
	- 2×2 投影矩阵仅依赖偏移 $s$ 与频率 $w_j=1/10000^{2j/d}$，与绝对位置 $i$ 无关
	- 模型因而天然捕获相对位置关系，增强对平移的泛化能力
# 7. Transformer
1. Transformer
	- 编码器
		- 每层两子层：多头自注意力 + 位置前馈网络；残差连接后接层归一化
		- 自注意力查询/键/值来自前一编码层输出；输出每位置 d 维向量
	- 解码器
		- 额外插入第三子层：编码器-解码器注意力，查询来自解码器，键/值来自编码器最终输出
		- 掩蔽自注意力：查询/键/值来自上一解码层，但仅允许看到当前位置及之前，保持自回归特性

2. 基于位置的前馈网络（Position-wise FFN）
	- 同一MLP独立作用于每个位置：输入形状(batch, seq, d) → 输出(batch, seq, ffn_out)
	- 结构：Linear(d, h) → ReLU → Linear(h, ffn_out)
	- 参数共享于各时间步；当所有位置输入相同时输出亦相同
	- 作用：提供逐位置非线性变换，增强模型表达能力
	- “逐位置”指的是 seq（序列时间步）维的每个位置
		- 输入张量形状  (batch, seq, d)
		- 同一 MLP 只作用于最后一个维度 d，对每个 `batch-idx` 和每个 `seq-idx` 都独立地做一次相同的线性-ReLU-线性变换
		- 因此 seq 维度的每个时间步都经历同一套参数的变换，但不同时间步之间互不影响——这就是“逐位置（Position-wise）”的含义
```python
class PositionWiseFFN(nn.Module):
	"""基于位置的前馈网络"""
	def __init__(self, ffn_num_input, ffn_num_hiddens, ffn_num_outputs, 
				 **kwargs):
		super().__init__(**kwargs)
		self.dense1 = nn.Linear(ffn_num_input, ffn_num_hiddens)
		self.relu = nn.ReLU()
		self.dense2 = nn.Linear(ffn_num_hiddens, ffn_num_outputs)
	
	def forward(self, X):
		return self.dense2(self.relu(self.dense1(X)))
```
3. 残差连接与层规范化
	- 子层输出为 F(x)，执行 x + F(x) 后再做层规范化
	- 层规范化：沿特征维度 d 计算均值/方差，对变长序列更友好；批量规范化沿 batch 归一化，对 NLP 效果差
	- 二者组合保证深模型可训练，缓解梯度消失与内部协变量偏移
```python
class AddNorm(nn.Module):
	"""残差连接后进行层规范化"""
	def __init__(self, normalized_shape, dropout, **kwargs):
		super().__init__(**kwargs)
		self.dropout = nn.Dropout(dropout)
		self.ln = nn.LayerNorm(normalized_shape)
	
	def forward(self, X, Y): 
		return self.ln(self.dropout(Y) + X)
```
4. Transformer编码器层
	- `EncoderBlock` 含两子层：多头自注意力 + 位置前馈；均用残差连接与层规范化
	- 前向：`Y = AddNorm(X, MHAtt(X,X,X,valid_len)) → AddNorm(Y, FFN(Y))`
	- 输入输出形状不变`(batch, seq, d)`；堆叠`num_layers`个`Block` 形成完整编码器
	- 嵌入值先乘 `sqrt(d)` 再与固定位置编码相加，以匹配量级
		- 嵌入值通常服从 N(0,1) 或类似的小幅随机分布，数值量级 ≈1
		- 正弦-余弦位置编码的每个元素在 \[-1,1\] 之间波动，量级也 ≈1
		- 训练初期反向传播平均把嵌入矩阵更新得更小（靠近 0），使得实际标准差 << 1，故L2 期望 ≪ √d，乘 √d 是提前补偿这一趋势。如果直接把两者相加：
			- 嵌入值相对太小 → 位置编码“淹没”语义信息
			- 训练初期嵌入矩阵更新缓慢 → 模型几乎只看位置、忽略词义
```python
class EncoderBlock(nn.Module):
	"""transformer编码器块"""
	def __init__(self, key_size, query_size, value_size, num_hiddens,
				 norm_shape, ffn_num_input, ffn_num_hiddens, num_heads, 
				 dropout, use_bias=False, **kwargs):
		super().__init__(**kwargs)
		self.attention = d2l.MultiHeadAttention(key_size, query_size, 
				value_size, num_hiddens, num_heads, dropout, use_bias)
		self.addnorm1 = AddNorm(norm_shape, dropout) 
		self.ffn = PositionWiseFFN(ffn_num_input, ffn_num_hiddens, 
								   num_hiddens) 
		self.addnorm2 = AddNorm(norm_shape, dropout)
	
	def forward(self, X, valid_lens): 
		Y = self.addnorm1(X, self.attention(X, X, X, valid_lens)) 
		return self.addnorm2(Y, self.ffn(Y))

class TransformerEncoder(d2l.Encoder):
	"""transformer编码器"""
	def __init__(self, vocab_size, key_size, query_size, value_size, 
				 num_hiddens, norm_shape, ffn_num_input, ffn_num_hiddens, 
				 num_heads, num_layers, dropout, use_bias=False, **kwargs): 
		super().__init__(**kwargs)
		self.num_hiddens = num_hiddens 
		self.embedding = nn.Embedding(vocab_size, num_hiddens) 
		self.pos_encoding = d2l.PositionalEncoding(num_hiddens, dropout) 
		self.blks = nn.Sequential()
		for i in range(num_layers): 
			self.blks.add_module("block"+str(i), 
				EncoderBlock(key_size, query_size, value_size, num_hiddens, 
							 norm_shape, ffn_num_input, ffn_num_hiddens, 
							 num_heads, dropout, use_bias))
	
	def forward(self, X, valid_lens, *args): 
	# 嵌入值先乘sqrt(d)再与固定位置编码相加，以匹配量级
		X = self.pos_encoding(self.embedding(X) * 
							  math.sqrt(self.num_hiddens)) 
		self.attention_weights=[None] * len(self.blks)
		for i, blk in enumerate(self.blks):
			X = blk(X, valid_lens)
			self.attention_weights[i] =  
				blk.attention.attention.attention_weights
		return X
```
5. Transformer解码器层
	- 三子层：掩蔽自注意力 + 编码器-解码器注意力 + 位置前馈；均残差&层归一
	- 掩蔽自注意力：查询键值来自上一解码层，但掩码阻止窥见未来位置，保持自回归
	- 编码器-解码器注意力：查询来自解码器，键值来自编码器最终输出，实现源-目标对齐
	- 参数 `dec_valid_lens` 控制可见长度，训练可并行，预测逐词扩展
		- 训练阶段一次喂入整个目标序列，`dec_valid_lens = [1,2,3,…,seqLen] (batch-wise)`，掩码矩阵一次性生成下三角，并行完成所有位置自回归计算
```python
class DecoderBlock(nn.Module):
	"""解码器中第i个块"""
	def __init__(self, key_size, query_size, value_size, num_hiddens, 
				 norm_shape, ffn_num_input, ffn_num_hiddens, num_heads, 
				 dropout, i, **kwargs): 
		super().__init__(**kwargs) 
		self.i = i 
		self.attention1 = d2l.MultiHeadAttention(key_size, query_size, 
				value_size, num_hiddens, num_heads, dropout) 
		self.addnorm1 = AddNorm(norm_shape, dropout) 
		self.attention2 = d2l.MultiHeadAttention(key_size, query_size, 
				value_size, num_hiddens, num_heads, dropout) 
		self.addnorm2 = AddNorm(norm_shape, dropout) 
		self.ffn=PositionWiseFFN(ffn_num_input,ffn_num_hiddens,num_hiddens) 
		self.addnorm3 = AddNorm(norm_shape, dropout)
	
	def forward(self, X, state): 
		enc_outputs, enc_valid_lens = state[0], state[1]
		# 训练阶段，输出序列的所有词元都在同一时间处理
		# 因此state[2][self.i]初始化为None
		# 预测阶段，输出序列通过词元逐一解码
		# 因此state[2][self.i]包含直到当前时间步第i个块解码的输出表示
		if state[2][self.i] is None:
			key_values = X
		else:
			key_values = torch.cat((state[2][self.i], X), axis=1)
		state[2][self.i] = key_values
		if self.training:
			batch_size, num_steps, _ = X.shape
			# dec_valid_lens的初始值:(batch_size, num_steps)
			# 其中每一行是[1,2,...,num_steps]
			dec_valid_lens = torch.arange(
				1, num_steps + 1, device=X.device).repeat(batch_size, 1)
		else: 
			dec_valid_lens = None
		
		# 自注意⼒ 
		X2 = self.attention1(X, key_values, key_values, dec_valid_lens)
		Y = self.addnorm1(X, X2) 
		# 编码器-解码器注意力。 
		# enc_outputs: (batch_size,num_steps,num_hiddens)
		Y2 = self.attention2(Y, enc_outputs, enc_outputs, enc_valid_lens)
		Z = self.addnorm2(Y, Y2) 
		return self.addnorm3(Z, self.ffn(Z)), state

def case()

class TransformerDecoder(d2l.AttentionDecoder):
	def __init__(self, vocab_size, key_size, query_size, value_size, 
				 num_hiddens, norm_shape, ffn_num_input, ffn_num_hiddens, 
				 num_heads, num_layers, dropout, **kwargs):
		super().__init__(**kwargs)
		self.num_hiddens = num_hiddens 
		self.num_layers = num_layers 
		self.embedding = nn.Embedding(vocab_size, num_hiddens) 
		self.pos_encoding = d2l.PositionalEncoding(num_hiddens, dropout) 
		self.blks = nn.Sequential()
		for i in range(num_layers):
			self.blks.add_module("block"+str(i), 
				DecoderBlock(key_size, query_size, value_size, num_hiddens,
							 norm_shape, ffn_num_input, ffn_num_hiddens, 
							 num_heads, dropout, i))
		self.dense = nn.Linear(num_hiddens, vocab_size)
		
	def init_state(self, enc_outputs, enc_valid_lens, *args): 
		return [enc_outputs, enc_valid_lens, [None] * self.num_layers]
	
	def forward(self, X, state):
		X = self.pos_encoding(self.embedding(X)*math.sqrt(self.num_hiddens))
		self._attention_weights = [[None]*len(self.blks) for _ in range(2)]
		for i, blk in enumerate(self.blks): 
			X, state = blk(X, state)
			# 解码器自注意力权重
			self._attention_weights[0][i] = 
				blk.attention1.attention.attention_weights
			# 编码器-解码器注意力权重
			self._attention_weights[1][i] = 
				blk.attention2.attention.attention_weights
		return self.dense(X), state
	
	@property
	def attention_weights(self):
		return self._attention_weights
```

7. Transformer 小结
	- 编码器-解码器架构实践，可单独使用编码器或解码器
	- 多头自注意力表示输入/输出序列；解码器靠掩蔽保持自回归
	- 残差连接+层规范化保障深度模型可训练
	- 位置前馈网络：同一MLP独立变换每个序列位置表示
8. 练习，如何设计语言模型架构
	- 语言模型 = 纯自回归生成 ⇒ 只用 Transformer 解码器（Decoder-Only）即可，无需编码器，也无需交叉注意力
	- 若需“编码-解码”任务（如机器翻译）才用编码器-解码器完整架构；纯语言建模 ⇒ Decoder-Only 最简洁高效
	- 设计要点（GPT 路线）
		- 结构
			- 去掉编码器整个分支
			- 保持 掩蔽自注意力 + 位置前馈 + 残差 & LayerNorm
			- 层数 L、隐藏维度 d、头数 h 按规模选取（GPT-2 1.5 B: L=48, d=1600, h=25）
		- 输入 / 输出
			- 输入： `<bos> token1 token2 ... tokenn`
			- 目标：整体右移 1 位  `token1 token2 ... tokenn <eos>`
			- 损失：标准语言模型交叉熵，仅计算非 pad 位置
		- 位置编码
			- 固定正弦或可学习绝对位置编码；最长序列长度预先设定
		- 训练技巧
			- Teacher forcing 并行训练
			- 梯度裁剪 + 学习率预热 + 余弦退火
			- Token 级 dropout / weight decay
		- 推理
			- 自回归采样：每次用缓存中的完整历史 + 一个新 token 做前向，缓存 Key/Value 矩阵加速
			- 可用 top-k / nucleus (top-p) 采样或 beam search
				- 模型输出的是整个词表上的概率分布 π
				- 贪心：直接 argmax 取概率最大词
				- top-k：只保留概率最高的 k 个词，重归一化后再按概率采样
				- top-p（nucleus）：保留累积概率 ≥ p 的最小集合，再采样
				- 采样 ≠ 训练，是推理阶段引入随机性的策略，避免重复、增加多样性；k 或 p 越小，文本越保守。
