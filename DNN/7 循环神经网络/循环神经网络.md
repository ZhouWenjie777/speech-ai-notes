# 1. 序列模型
1. 循环神经网络（recurrent neural network，RNN）通过引入状态变量存储过去的信息和当前输入确定当前的输出，可以更好地处理序列信息。
2. 外推法（extrapolation）：对超出已知观测范围进行预测。
3. 内插法（interpolation）：在现有观测值之间进行估计。
4. 序列模型建模统计工具
	- 自回归模型
		- 策略1：截断长度$τ$，仅用$α_{t-1},...,α_{t-τ}$预测$α_t$，参数固定→自回归模型
		- 策略2：维护隐状态$h_t$，联合更新预测与状态
			- 预测 $α_t \sim P(α_t|h_t)$
			- 状态更新 $h_t = g(h_{t-1}, α_{t-1})$
			- 由于$h_t$不可观测，称为隐变量自回归模型
	- 马尔可夫模型
		- 用$α_{t-1},...,α_{t-τ}$近似$α_1,...,α_{t-1}$，若近似足够精确，则序列满足马尔可夫条件
		- 一阶马尔可夫模型($τ=1$)：$P(\alpha_1,\dots,\alpha_T)=P(\alpha_1)\prod_{t=2}^T P(\alpha_t|\alpha_{t-1})$
		- 初始分布：$P(\alpha_1|\alpha_0)\equiv P(\alpha_1)$
5. 对于直到时间步t的观测序列，其在时间步t+k的预测输出是“k步预测”。随着k值的增加，误差会快速累积导致预测质量极速下降。
# 2. 文本预处理
1. 文本常见的预处理步骤通常包括
	- 将文本作为字符串加载到内存中
	- 将字符串拆分为词元（如单词和字符）
	- 建立⼀个词表，将拆分的词元映射到数字索引
	- 将文本转换为数字索引序列，方便模型操作
2. 词表（vocabulary）：将字符串类型的词元映射到从0开始的数字索引中。
	- 合并训练文档得语料，统计唯一词元
	- 按频率降序编号，稀有词移除或映射\<unk\>
	- 特殊词元：\<pad\>、\<bos\>、\<eos\>、\<unk\>
# 3. 语言模型和数据集
1. 语言模型：给定文本序列 $\alpha_1,\alpha_2,\dots,\alpha_T$，估计联合概率$P(\alpha_1,\alpha_2,\dots,\alpha_T)$
2. 马尔可夫模型与n元语法（n-gram）
	- 一元：$P(\alpha_1,\alpha_2,\alpha_3,\alpha_4)=P(\alpha_1)P(\alpha_2)P(\alpha_3)P(\alpha_4)$
	- 二元：$P(\alpha_1,\alpha_2,\alpha_3,\alpha_4)=P(\alpha_1)P(\alpha_2|\alpha_1)P(\alpha_3|\alpha_2)P(\alpha_4|\alpha_3)$
	- 三元：$P(\alpha_1,\alpha_2,\alpha_3,\alpha_4)=P(\alpha_1)P(\alpha_2|\alpha_1)P(\alpha_3|\alpha_1,\alpha_2)P(\alpha_4|\alpha_2,\alpha_3)$
	- 分别对应 `unigram`、`bigram`、`trigram` 模型
3. 长序列读取策略
	- 随机采样：在原始长序列上任意位置截取子序列，样本间无顺序关系
	- 顺序分区：按时间顺序将长序列划分为连续子序列，样本间保持时间连续性
# 4. 循环神经网络
1. 隐变量语言模型
	- n-gram需存储$|V|^n$参数，参数指数增长；改用隐状态$h_{t-1}$存储历史信息
	- 条件概率：$P(\alpha_t|\alpha_{t-1},\dots,\alpha_1)\approx P(\alpha_t|h_{t-1})$，$h_{t-1}$ 是隐状态（hidden state），也称为隐藏变量（hidden variable），存储到时间步 $t-1$ 的序列信息。
	- 隐状态更新：$h_t=f(\alpha_t,h_{t-1})$，函数$f$可存储完整历史，但计算与内存代价高
2. 无隐状态的神经网络（单隐藏层MLP）
	- 隐藏层输出 $H = \phi(XW_{xh} + b_h)$，$W_{xh}\in\mathbb{R}^{d\times h},\ b_h\in\mathbb{R}^{1\times h}$
	- 输出层 $O = HW_{hq} + b_q$，$W_{hq}\in\mathbb{R}^{h\times q},\ b_q\in\mathbb{R}^{1\times q}$
	- 分类任务：输出概率 $\mathrm{softmax}(O)$
3. 有隐状态的循环神经网络（RNN）
	- 隐状态更新 $H_t = \phi(X_t W_{xh} + H_{t-1} W_{hh} + b_h)$，$W_{xh}\in\mathbb{R}^{d\times h},W_{hh}\in\mathbb{R}^{h\times h}$
	- 输出计算 $O_t = H_t W_{hq} + b_q$，$W_{hq}\in\mathbb{R}^{h\times q}$
	- 参数共享：不同时间步共用同一组 $W_{xh},W_{hh},W_{hq},b_h,b_q$，参数数量与时间步无关
	- 隐状态 $H_t$ 充当网络记忆，保留历史信息
	- 隐状态的更新计算是循环的（recurrent）。于是基于循环计算的隐状态神经网络被称为循环神经网络（recurrent neural network）。
4. 隐状态更新矩阵视角
	- 将批量样本按时间步堆叠，$X_t\in\mathbb{R}^{n\times d},\ H_t\in\mathbb{R}^{n\times h}$
	- 一次矩阵乘法完成小批量所有样本的隐状态更新与输出计算。
	- 隐状态中 $X_t W_{xh} + H_{t-1} W_{hh}$ 的计算，相当于 $X_t$ 和 $H_{t-1}$ 的拼接与 $W_{xh}$ 和 $W_{hh}$ 的拼接的矩阵乘法
5. 序列交叉熵
	- 平均交叉熵 $-\dfrac{1}{n}\sum_{t=1}^{n}\log P(\alpha_t|\alpha_{<t})$
	- 衡量模型压缩效率，长度归一化便于不同序列比较
6. 困惑度（perplexity）
	- 度量评价语言模型的质量
	- 定义 $\displaystyle \mathrm{PPL} = \exp\!\left(-\frac{1}{n}\sum_{t=1}^{n}\log P(\alpha_t|\alpha_{<t})\right)$
	- 最佳情况：概率恒为1 → PPL=1
	- 最坏情况：概率为0 → PPL=$+\infty$
	- 均匀基线：PPL=|V|（词表大小），为实际上限

2. 初始化模型参数
```python
def get_params(vocab_size, num_hiddens, device):
	num_inputs = num_outputs = vocab_size
	
	def normal(shape): 
		return torch.randn(size=shape, device=device) * 0.01
	
	# 隐藏层参数 隐藏单元数num_hiddens是⼀个可调的超参数
	W_xh = normal((num_inputs, num_hiddens)) 
	W_hh = normal((num_hiddens, num_hiddens)) 
	b_h = torch.zeros(num_hiddens, device=device) 
	# 输出层参数 
	W_hq = normal((num_hiddens, num_outputs)) 
	b_q = torch.zeros(num_outputs, device=device) 
	# 附加梯度 
	params = [W_xh, W_hh, b_h, W_hq, b_q]
	for param in params: 
		param.requires_grad_(True) 
	return params
```
3. 隐状态初始化

4. `RNN`函数
```python
def rnn(inputs, state, params): 
	# inputs的形状：(时间步数量，批量⼤⼩，词表⼤⼩) 
	W_xh, W_hh, b_h, W_hq, b_q = params 
	H, = state 
	outputs = [] 
	# X的形状：(批量⼤⼩，词表⼤⼩)
	for X in inputs: 
		H = torch.tanh(torch.mm(X, W_xh) + torch.mm(H, W_hh) + b_h) 
		Y = torch.mm(H, W_hq) + b_q 
		outputs.append(Y)
	return torch.cat(outputs, dim=0), (H,)
```

```python
class RNNModelScratch:
	"""从零开始实现的循环神经⽹络模型""" 
	def __init__(self, vocab_size, num_hiddens, device, 
				 get_params, init_state, forward_fn): 
	self.vocab_size, self.num_hiddens = vocab_size, num_hiddens 
	self.params = get_params(vocab_size, num_hiddens, device) 
	self.init_state, self.forward_fn = init_state, forward_fn
	
	def __call__(self, X, state): 
		X = F.one_hot(X.T, self.vocab_size).type(torch.float32) 
		return self.forward_fn(X, state, self.params)
	
	def begin_state(self, batch_size, device):
		return self.init_state(batch_size, self.num_hiddens, device)
```
6. 预热与预测
	- 遍历prefix字符时只更新隐状态，不输出，称为预热（warm-up）
	- 预热后隐状态已包含prefix信息，再逐步生成并输出后续字符
```python
def predict_ch8(prefix, num_preds, net, vocab, device):
	"""在prefix后⾯⽣成新字符"""
	state = net.begin_state(batch_size=1, device=device)
	outputs = [vocab[prefix[0]]]
	get_input = lambda: torch.tensor([outputs[-1]], 
									  device=device).reshape((1, 1))
	for y in prefix[1:]: # 预热期 
		_, state = net(get_input(), state) 
		outputs.append(vocab[y])
	for _ in range(num_preds): # 预测num_preds步 
		y, state = net(get_input(), state) 
		outputs.append(int(y.argmax(dim=1).reshape(1)))
	return ''.join([vocab.idx_to_token[i] for i in outputs])
```
7. 梯度裁剪
	- 梯度范数可能爆炸，令优化无法收敛
	- 裁剪规则：$\displaystyle \mathbf{g}\leftarrow \min\!\left(1,\frac{\theta}{\|\mathbf{g}\|}\right)\mathbf{g}$
	- 效果：范数上限$\theta$，方向不变；限制单批次影响，提升稳定性
	- 快速缓解梯度爆炸，需配合其他技术使用
		- 没有解决梯度仍然很大的根源（模型结构、权重初始化、学习率过大等）
		- 对梯度消失完全无效
		- 长期依赖问题（权重矩阵特征值接近1）依旧存在
		- 所以通常还要同时用
			- 合理初始化（Xavier/He）
			- 适当学习率 + 调度器
			- 门控结构（LSTM / GRU）或残差连接
			- 权重正则化、批量归一化等
```python
def grad_clipping(net, theta):
	"""裁剪梯度""" 
	if isinstance(net, nn.Module): 
		params = [p for p in net.parameters() if p.requires_grad] 
	else: 
		params = net.params
	norm = torch.sqrt(sum(torch.sum((p.grad ** 2)) for p in params))
	if norm > theta:
		for param in params:
			param.grad[:] *= theta / norm
```
8. 训练
	- 顺序分区：仅周期开始初始化隐状态，末尾样本状态传给下一批量；训练前分离梯度，限制计算范围以减少计算量
	- 随机采样：每个批量重新初始化隐状态，无需跨批量传递
	- 更新前执行梯度裁剪，防止爆炸
	- 评价指标：困惑度，保证不同长度序列可比
```python
from d2l import torch as d2l
def train_epoch_ch8(net, train_iter, loss, updater, device,use_random_iter):
	"""训练⽹络⼀个迭代周期"""
	state, timer = None, d2l.Timer()
	metric = d2l.Accumulator(2) #训练损失之和,词元数量
	for X, Y in train_iter:
		if state is None or use_random_iter: 
			# 在第⼀次迭代或使⽤随机抽样时初始化state 
			state = net.begin_state(batch_size=X.shape[0], device=device)
		else:
			if isinstance(net, nn.Module) and not isinstance(state, tuple):
				#state对于nn.GRU是个张量 
				state.detach_()
			else: 
			# state对于nn.LSTM或对于我们从零开始实现的模型是个张量 
				for s in state: 
					s.detach_()
		y = Y.T.reshape(-1)
		X, y = X.to(device), y.to(device)
		y_hat, state = net(X, state)
		l = loss(y_hat, y.long()).mean()
		if isinstance(updater, torch.optim.Optimizer):
			updater.zero_grad() 
			l.backward() 
			grad_clipping(net, 1) 
			updater.step()
		else: 
			l.backward() 
			grad_clipping(net, 1) 
			#因为已经调⽤了mean函数 
			updater(batch_size=1)
		metric.add(l * y.numel(), y.numel())
	# 困惑度ppl，词元/秒speed
	return math.exp(metric[0] / metric[1]), metric[1] / timer.stop() 
```
# 6. 循环神经网络的简洁实现
1. `RNN`的高级API实现
```python
batch_size, num_steps = 32, 35
num_hiddens = 256
rnn_layer = nn.RNN(len(vocab), num_hiddens)
# 隐状态形状（隐藏层数，批量大小，隐藏单元数）
state = torch.zeros((1, batch_size, num_hiddens))

X = torch.rand(size=(num_steps, batch_size, len(vocab))) 
Y, state_new = rnn_layer(X, state) 
# 输出Y为每个时间步的隐状态，供后续输出层使用；state_new为最终隐状态
Y.shape, state_new.shape
```

2. 函数封装
```python
class RNNModel(nn.Module):
	"""循环神经网络模型"""
	def __init__(self, rnn_layer, vocab_size, **kwargs):
		super().__init__(**kwargs)
		self.rnn = rnn_layer
		self.vocab_size = vocab_size
		self.num_hiddens = self.rnn.hidden_size
		#如果RNN是双向的，num_directions应该是2，否则应该是1
		if not self.rnn.bidirectional:
			self.num_directions = 1
			self.linear = nn.Linear(self.num_hiddens, self.vocab_size)
		else:
			self.num_directions = 2
			self.linear = nn.Linear(self.num_hiddens * 2, self.vocab_size)
	
	def forward(self, inputs, state):
		X = F.one_hot(inputs.T.long(), self.vocab_size)
		X = X.to(torch.float32)
		Y, state = self.rnn(X, state)
		# 全连接层首先将Y的形状改为(时间步数*批量⼤⼩, 隐藏单元数)
		# 它的输出形状是(时间步数*批量⼤⼩, 词表⼤⼩)。
		output = self.linear(Y.reshape((-1, Y.shape[-1])))
		return output, state
	
	def begin_state(self, device, batch_size=1):
		if not isinstance(self.rnn, nn.LSTM):
		# nn.GRU以张量作为隐状态
		return torch.zeros((self.num_directions * self.rnn.num_layers,
							batch_size, self.num_hiddens), device=device)
		else:
		# nn.LSTM以元组作为隐状态
		return (torch.zeros((self.num_directions * self.rnn.num_layers, 
							 batch_size, self.num_hiddens), device=device), 
				torch.zeros((self.num_directions * self.rnn.num_layers, 
							 batch_size, self.num_hiddens), device=device))
```
# 7. 通过时间反向传播
1. 循环网络反向传播（BPTT）
	- 损失 $L = \frac{1}{T}\sum_{t=1}^{T}\ell(y_t,o_t)$
	- 梯度 $\frac{\partial L}{\partial W_h}=\frac{1}{T}\sum_{t=1}^{T}\frac{\partial\ell(y_t,o_t)}{\partial o_t}\frac{\partial o_t}{\partial h_t}\frac{\partial h_t}{\partial W_h}$
	- 关键项 $\frac{\partial h_t}{\partial W_h}$ 递归依赖历史：$\frac{\partial h_t}{\partial W_h}=\frac{\partial f}{\partial W_h}+\frac{\partial f}{\partial h_{t-1}}\frac{\partial h_{t-1}}{\partial W_h}$
	- 展开后得 $\frac{\partial h_t}{\partial W_h}=\frac{\partial f}{\partial W_h}+\sum_{i=1}^{t-1}\frac{\partial f}{\partial W_h}\prod_{j=i+1}^{t}\frac{\partial f}{\partial h_{j-1}}$，链随 t 指数增长，需截断或梯度检查等技巧
2. 截断与随机截断策略
	- 完全计算：遍历全部历史，慢且易梯度爆炸，几乎不用
	- 截断时间步：仅最近τ步求和，真实梯度近似，偏向短期依赖，称截断BPTT
	- 随机截断：用变量$s_t∈{0,π_t^{-1}}，E[s_t]=1，P(s_t=0)=1−π_t$；梯度$∂h_t/∂W_h$替换为$z_t = ∂f/∂W_h + (∂f/∂h_{t−1})*(∂h_{t−1}/∂W_h)*s_t$，期望正确且提前终止，长序列权重自动放大
3. 三种BPTT策略对比
	- 完全BPTT：计算不可行，几乎不用
	- 常规截断：固定长度子序列，轻度正则化，短依赖已足够，最常用
	- 随机截断：理论有吸引力，实际效果≈常规截断；增加方差，长梯度优势被抵消
4. 线性RNN的数值不稳定性
	- 对任何时间步t展开梯度含矩阵的高次幂 $(W_{hh}^T)^{T-i}$，特征值<1消失，>1发散
	- 表现：梯度消失/爆炸，数值不稳定
	- 缓解：截断时间步；后续引入LSTM等门控机制进一步解决
5. 为了计算的效率，”通过时间反向传播（BPTT）“在计算期间会缓存中间值。
