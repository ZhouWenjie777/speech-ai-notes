1. 友元
	- 目的：
		- 打破封装壁垒，让全局函数、其它类或指定成员函数直接访问本类私有/保护成员，常用于运算符重载
	- 声明方式：
		- 在类体内写 `friend 函数原型;` 或 `friend class 类名;`
		- 位置任意，通常集中放在最前或最后
	- 普通友元函数：
		- `friend Time operator+(const Time &t1, const Time &t2);` 
		- 实现对称加法，可平等访问两个对象的私有 hours/minutes
	- 友元类：
		- class B 被声明为 class A 的友元，则 B 的所有成员函数都能访问 A 的私有成员；单向关系，A 不能因此访问 B。
	- 友元成员函数：
		- 可只把另一个类的某个成员函数声明为友元
		- 语法 `friend 返回类型 类名::函数名(形参表);` 提供更细粒度控制
	- 输出运算符重载：
		- <<须重载为非成员（全局或友元），因为左操作数是`ostream`而非本类对象
		- 原型`ostream& operator<<(ostream &os, const Time &t)`返回`ostream&` 支持链式 `cout << t1 << " + " << t2;` 若返回 void 则无法连续输出
2. 友元详解
	- 友元函数
		- 全局非成员函数，可访问宿主类全部私有/保护成员；常用于运算符重载（`<<、>>、+、==`）以保持对称性
		- 声明：`class TV { friend void remotePower(TV& t); }` 全局函数
	- 友元类
		- 整个类所有成员函数均获得访问权；适用于“专属服务类”或“配对组件”：例如遥控器必须操作电视内部状态
		- 声明：
			- `class TV { friend class Remote; }` 
			- Remote 所有成员都可访问 TV 私有
			- 位置随意，通常集中放在类定义最前或最后，并加注释说明意图
	- 友元成员函数（细粒度控制）
		- 只把另一个类的某一个成员函数设为友元，减少暴露面
		- 声明
			- `class Remote { void changeInput(TV& t); }`
			- `class TV { friend void Remote::changeInput(TV&); }`
		- 注意：必须让编译器先看到 `Remote` 的类声明，再在`TV`里写友元声明，最后实现 `changeInput`，否则出现“未定义类型”错误
	- 权限特性
		- 友元关系单向： `Remote`是`TV`的友元，不等于`TV`能访问`Remote`私有
		- 不可传递：若`A`是`B`的友元，`B`是`C`的友元，不表示`A`能访问`C`
		- 不可继承：派生类不会自动成为原类的友元；若`SmartRemote`继承`Remote`，仍需单独`friend class SmartRemote`;
	- 使用建议
		- 优先通过 public 接口解耦；当性能或语义要求必须突破封装时再使用友元
		- 友元类/函数应与宿主类共存于同一库/模块，避免外部代码依赖私有细节
		- 头文件中友元声明会暴露实现细节，可考虑 `PIMPL` 或内部类替代，减少编译依赖
