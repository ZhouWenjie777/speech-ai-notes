1. 标准模板库（STL）与 vector 速览
	- STL 组成（四大件）
		- **容器**（Containers）：存储数据的模板类，如 `vector`, `list`, `map`
		- **迭代器**（Iterators）：泛化指针，用于遍历容器；支持 `++`, `--`, `*`, `->`
		- **算法**（Algorithms）：头文件 `<algorithm>` 提供排序、查找、复制等泛型函数，通过迭代器操作容器
		- **函数对象**（Functors）：重载了 `operator()` 的类或普通函数指针，可内联、可带状态，供算法定制行为。
	- vector 本质
		- 头文件 `<vector>` 的**顺序容器**，内存连续，支持随机访问（下标/迭代器），容量自动倍增，可视为“可变长数组”。
	- 常用构造与赋值
		- `vector<int> v;               // 空`
		- `vector<int> v(5, 1);         // 5 个 1`
		- `vector<string> titles(NUM);  // NUM 个默认空串`
		- `titles = otherVec;           // 整体深拷贝赋值`
	- 容量管理
		- `size()`   当前元素个数
		- `capacity()`   已分配总空间
		- `empty()`   判空
		- `reserve(n)`   提前分配，减少重新拷贝
		- `shrink_to_fit()`   请求归还多余容量（C++11）
	- 元素访问
		- 下标  `v[i]`  // 不检查越界
		- `v.at(i)`  // 越界抛  `out_of_range`
		- `front() / back()`  // 首尾元素引用
		- 迭代器：
			- `for (vector<string>::iterator it = titles.begin(); it != titles.end(); ++it) cout << *it;`
			- C++11 范围 for：`for (auto& t : titles) cout << t;`
	- 修改接口
		- `push_back(x)`  尾插，可能触发扩容（ amortized O(1) ）
		- `pop_back()`  删除末元素
		- `insert(pos, x) / erase(pos)`  任意位置增删，后续元素搬移
		- `clear()`  清空所有元素，容量通常保留
		- `emplace_back(args...)` 原地构造，避免额外拷贝/移动（C++11）
	- 异常安全
		- `push_back`提供强异常保证：若拷贝/移动抛异常，vector 保持原状态
		- 元素析构抛异常则调用`terminate()`
	- 与原始数组互操作
		- `data()` 返回`T*`指向首元素（C++11），可与 C API 对接
		- 保证连续内存，可用`memcpy`批量读写 POD 类型
	- 性能提示
		- 随机访问 O(1)；尾插平均 O(1)；中间插入/删除 O(n)
		- 扩容策略：通常 1.5 倍或 2 倍，使均摊复制次数为常数
		- 提前`reserve`已知元素量，可大幅减少重新分配与复制
	- 适用场景
		- 需要“数组”+“自动扩容”+“随机访问”时首选 `vector`；
		- 若频繁中间插入/删除，考虑 `list` / `deque`；
		- 若键值查找，换 `map` / `unordered_map`
2. 迭代器（iterator）——STL 的“通用指针”
	- 本质
		- 对每个容器内部实现的一个**内嵌类**，行为像指针：可 `++`、可 `*`、可 `==`，但具体走位由容器自己决定（数组连续 +1、链表跳 next、红黑树中序遍历）
	- 核心操作（任何迭代器都必须支持）
		- 解除引用 `*p`                    访问当前元素
		- 自增 `++p / p++`               移动到下一位置
		- 相等 `p == q / p != q`     判断是否指向同一元素
		- 赋值 `p = q`                      拷贝迭代器状态
	- 容器配套接口
		- `container::iterator       it;   // 读写迭代器`
		- `container::const_iterator cit;  // 只读迭代器`
		- `c.begin()                       // 指向首元素`
		- `c.end()    // 指向“超尾”——最后一个元素的下一个位置（one-past-end）`
	- 通用遍历模式：
		- `for (auto it = v.begin(); it != v.end(); ++it) { /* use *it */ }`
	- 与“下标”区别
		- 下标仅适用连续内存容器（vector、string、array）；迭代器适用于所有容器
		- 下标不能插入/删除；迭代器配合算法可`erase(it)` / `insert(it, val)`
	- find 泛型化
		- 旧式（数组）
			- `int* find(int* begin, int* end, int val) {`
			- `for (int* p = begin; p != end; ++p)`
			- `if (*p == val) return p;`
			- `return end; }          // 找不到返回超尾`
		- STL 通用版
			- `template<class InputIt, class T>`
			- `InputIt find(InputIt begin, InputIt end, const T& val) {`
			- `for (; begin != end; ++begin)`
			- `if (*begin == val) return begin;`
			- `return end; }`
			- 同一套代码，对 `vector<int>`、`list<string>`、`deque<double>`都能工作，因为各自迭代器都满足`InputIterator`概念
	- 迭代器类别（由弱到强）
		- Input / Output 只读/写一次，`++`单向
		- Forward 可多次读写，`++`单向
		- Bidirectional 支持 `++` 与 `--`（`list、map`）
		- Random Access 支持 `+n、-n、[]、<、>`（`vector、deque、string`）
	- `const_iterator`意义
		- 当容器本身为`const`或只需读时，编译器强制使用`const_iterator`，防止意外修改元素
	- 异常安全（迭代器失效规则）
		- `vector`只有在**重新分配**时所有迭代器失效；尾插/尾删仅`end()`失效
		- `list/map`仅**被删除元素**的迭代器失效，其余稳定
	- C++11 语法糖
		- `for (const auto& x : container) { ... }   // 编译器自动用 begin/end`
	- 自写容器时的最小迭代器
		- 按需求实现`iterator`与`const_iterator`内嵌类
		- 重载`*、++、==、!=`，并提供`begin()/end()`，即可立刻享用所有 STL 算法
3. 函数对象（Functor）——“像函数的对象”
	- 定义
		- 重载了`operator()`的类/结构体实例，语法上像函数调用，实为对象调用
		- 好处：可内联、可带状态、可模板化，比普通函数指针更高效灵活
	- 示例
		- `struct add { void operator()(int& x) const { ++x; } };`
		- `for_each(v.begin(), v.end(), add()); // 对每个元素执行 add()`
	- 函数指针 vs 函数对象
		- 函数指针：无法内联；无状态；类型固定
		- 函数对象：编译器可内联，效率 = 手写循环；可携带成员变量，多次调用间保存数据；可用模板，一套代码适用多类型
	- STL 提供的预定义函数对象（头文件`<functional>`）
		- 算术：
			- `plus<T>   operator+` 
			- `minus<T>   operator-`
			- `multiplies<T>   operator*`
			- `divides<T>   operator/`
			- `modulus<T>   operator%`
			- `negate<T>  一元取反`
			- 注：`plus<T>`本身只提供函数调用运算符，调用类型 T 的 `operator+`，而不是自己重载 `+`，其他同理
		- 关系：
			- `equal_to<T>   ==`
			- `not_equal_to<T>   != `
			- `greater<T>   >`
			- `less<T>   <`
			- `greater_equal<T>   >=`
			- `less_equal<T>   <=`
		- 逻辑：
			- `logical_and<T>   &&`
			- `logical_or<T>   ||`
			- `logical_not<T>   !`
	- 使用示范
		- `merge(a, b, c, 10, plus<int>());   // c[i] = a[i] + b[i]`
		- `merge(s1, s2, s3, 3, plus<string>()); // 字符串拼接`
		- `for_each(v.begin(), v.end(), output); // 打印`
		- `for_each(v.begin(), v.end(), add);    // 每个元素 +1`
	- 一元 / 二元限制
		- `for_each` 要求一元函数对象（参数 = 容器元素）
		- `transform` / `merge` 可接受二元函数对象（两输入序列中各取一元素）
	- 状态示例——计数器
		- `struct counter { mutable std::size_t n = 0; void operator()(int) const { ++n; } };`
		- `counter c; for_each(v.begin(), v.end(), c); std::cout << "elements count = " << c.n;`
	- 与 lambda 的关系
		- C++11 起 lambda 表达式自动生成匿名函数对象；旧代码手写 functor 仍适用，且可读性、可复用性更高
	- 性能提示
		- 函数对象通常零成本抽象：编译器把`operator()`直接内联到调用点，效率 ≥ 手写循环；比通过函数指针调用快
	- 最佳实践
		- 算法需要“自定义操作”时，优先用 STL 预定义函数对象
		- 逻辑复杂或需状态，写小型函数对象
		- 仅一次使用且简单，用 lambda 更简洁
4. STL 算法速览（`<algorithm>` 非成员函数版）
	- （1）统一调用模式
		- 绝大多数算法接受一对迭代器 `[first, last)` 表示范围，额外参数可再传入函数对象/值
	- （2）排序 `sort`
		- 默认升序
			- `sort(v.begin(), v.end());          // 用 operator<`
		- 显式比较器
			- `sort(v.begin(), v.end(), less<int>());    // 同默认`
			- `sort(v.begin(), v.end(), greater<int>()); // 降序`
		- 复杂度：平均 O(N log N)，内省排序（`Introsort`）
	- （3）查找 `find`
		- `auto it = find(container.begin(), container.end(), value);`
		- 找到返回对应迭代器，否则返回 `container.end()`
		- 线性扫描 O(N)；若已排序可用 `binary_search` / `lower_bound`
	- （4）遍历 `for_each`
		- `for_each(v.begin(), v.end(), output); // 对每个元素执行 output(*it)`
		- 返回值为传入的函数对象（可带状态）
	- （5）拷贝 `copy` / `copy_if`
		- `copy(src.begin(), src.end(), dst.begin()); // 按顺序拷贝，dst需够空间`
		- `copy(src.begin(), src.end(), back_inserter(dst)); // 动态扩容`
	- （6）反转 `reverse`
		- `reverse(v.begin(), v.end());`
	- （7）去重 `unique`（连续重复）
		- `auto newEnd = unique(v.begin(), v.end());   // 逻辑去重+返回新尾`
		- `v.erase(newEnd, v.end());    // 真正删除尾部[newEnd, v.end())残余`
	- （8）计数 `count` / `count_if`
		- `int c = count(v.begin(), v.end(), 5);       // 等于 5 的个数`
		- `int c2 = count_if(v.begin(), v.end(), [](int x){ return x>3; });`
	- （9）最值 `min_element` / `max_element`
		- `auto it = min_element(v.begin(), v.end());  // 返回迭代器`
		- `cout << *it;`
	- （10）合并`merge`（已序区间）
		- `vector<int> dst(src1.size() + src2.size());`
		- `merge(src1.begin(), src1.end(), src2.begin(), src2.end(), dst.begin(), less<int>());   // 目标也需有序`
	- （11）堆算法 
		- `make_heap / push_heap / pop_heap`
		- 把 vector 原地构建为二叉堆，用于优先队列实现
	- （12）数值算法（头文件`<numeric>`）
		- `accumulate`：求和 / 自定义累积
		- `inner_product`：内积
		- `partial_sum / adjacent_difference`：前缀和 / 相邻差
	- （13）迭代器适配器（补充）
		- `back_inserter(container)`：尾部插入迭代器
		- `front_inserter / inserter`：头部 / 任意位置插入
		- `istream_iterator<T> / ostream_iterator<T>`：把流当容器读写
	- （14）复杂度
		- 非序算法（`find、count、for_each`：O(N)
		- 排序相关（`sort、binary_search、merge`）：O(N log N) 或 O(log N)
		- 堆算法：O(N) 建堆，O(log N) 插入/删除
	- （15）使用要诀
		- 算法不依赖容器，只依赖迭代器能力；换容器代码不动
		- 尽量用 auto 接收返回迭代器，减少冗长类型名
		- 排序后再用二分搜索（`binary_search / lower_bound`）可提速查找
		- 需要修改容器大小时配合`inserter`或`back_inserter`（动态扩容），避免容量不足崩溃
