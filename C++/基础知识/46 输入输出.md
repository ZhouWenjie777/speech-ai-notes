1. 输入输出概述与重定向
	- 流（stream）模型
		- C++ 把 I/O 看成**字节序列**，不关心来源/去向；库 `<iostream>` 提供类族实现
			- **输入流**：设备 → 内存（`std::istream`）
			- **输出流**：内存 → 设备（`std::ostream`）
	- 预定义全局对象（`<iostream>` 已构造）
		- `std::cin`：关联设备（键盘）；有缓冲；标准输入
		- `std::cout`：关联设备（显示器）；有缓冲；标准输出
		- `std::cerr`：关联设备（显示器）；**无缓冲**；标准错误，立即输出
		- `std::clog`：关联设备（显示器）；有缓冲；标准日志，与`cerr`同设备但可缓冲
	- 缓冲机制
		- **行缓冲**（`cout/clog`）：遇 `\n` 或缓冲区满或程序正常结束 → 刷新
		- **无缓冲**（`cerr`）：每次插入立即写设备
		- 手动刷新：`std::flush` 或 `std::endl`（先 `\n` 再 flush）
	- 命令行重定向（shell 级，与 C++ 代码无关）
		- 重定向：
			- 通过命令行替换数据源/目的地
			- `<` 把文件变键盘
			- `>` 把文件变屏幕
		- 输入重定向：
			- `可执行文件 < in.txt`  // 键盘 → 文件内容
		- 输出重定向：
			- `可执行文件 > out.txt`  // 显示器 → 写入文件（覆盖） 
			- `可执行文件 >> out.txt`  // 追加模式
		- 同时重定向：
			- `可执行文件 < in.txt > out.txt`
		- 错误分离：
			- `可执行文件 > out.txt 2> err.txt` // 标准输出与标准错误分别进文件
			- `可执行文件 > out.txt 2>&1`   // 把 2 合并到 1，同文件输出
	- 程序内检测重定向
		- 可用 `isatty(fileno(stdin))` / `stdout` / `stderr` 判断某流是否仍关联终端；返回 0 表示已被重定向到文件
	- 二进制流 vs 文本流
		- 默认文本模式：`\n` ↔ 系统换行符转换；
		- 二进制模式：`std::ofstream ofs("data.bin", std::ios::binary);` 无换行符转换，适合非文本数据
	- 刷新控制
		- `std::cout << "msg" << std::flush;   // 只刷新，不换行`
		- `std::cout << "msg" << std::endl;    // 换行 + 刷新`
		- `std::cin.tie(&std::cout);       // 关联：每次 cin 前自动刷新 cout`
	- 最佳实践
		- 用户提示信息写 `std::cerr`或`std::clog`，避免被 `>` 重定向“埋没”
		- 大批量输出先用 `std::ostringstream` 拼接，再一次性`cout << oss.str();`减少刷新次数
		- 写日志时`std::clog`带缓冲，性能高于`std::cerr`，但关键错误立即 `std::cerr` 确保落盘
2. `cout` 进阶输出（`ostream` 方法）
	- 重载 `<<` 运算符链
		- 返回值都是 `ostream&`，因此可以无限拼接：`cout << x << " abc " << y << flush;`
	- `put(char)`：单字符输出
		- 原型：`ostream& put(char ch);`
		- 支持链式：`cout.put('a').put('b');   // 输出 ab`
	- `write(const char* s, streamsize n)` ：块输出
		- 不处理换行符、不添加结尾空字符，按字节原样写出
		- `const char* tmp = "abcdefg";` `cout.write(tmp, 5);  // 输出 abcde`
		- 常用于二进制数据或高效批量文本
	- 缓冲区刷新时机
		- 缓冲区满自动刷新
		- 遇到 `endl` → 先写 `\n` 再 `flush`
		- 遇到 `flush` → 仅刷新，不换行
		- 程序正常结束自动刷新
		- 关联输入时 →  `cin >> x` 前自动 `cout.flush()`（默认 `cin.tie(&cout)`）
	- 关联（tie）机制
		- `streambuf* old = cin.tie();   // 查询当前关联`
		- `cin.tie(nullptr);             // 取消关联，加速交互式大量输入`
		- 一个输出流最多关联一个输入流；反之亦然
	- 数值类型内部 → 文本流程（以 int 为例）
		- 编译器调用对应 `operator<<(ostream&, int)`
		- 按符号/进制转换生成字符序列
		- 把序列写入 `streambuf`
		- 若缓冲区满或遇到 `flush/endl` 则系统调用写设备
	- 格式控制（后续展开）
		- 通过 `<iomanip>` 可进一步指定宽度、精度、进制、填充字符等；
		- `put/write`不受这些格式影响，原样输出
	- 异常状态
		- `ostream` 在写入失败时会设置状态位 `badbit/failbit`；
		- 可用 `if(cout)` 或  `cout.good()`检测；严重错误后需 `cout.clear()` 恢复
	- 性能提示
		- 批量输出先用 `ostringstream` 拼接，再一次性 `cout << oss.str();` 减少多次系统调用
		- 二进制大块数据用 `write` 避免逐字符格式转换
		- 交互式大量读写可 `cin.tie(nullptr)` 解除关联，防止每次 `>>` 都隐式刷新  `cout`
		- 标准规定：`cin >>` 会隐式`flush cout` 确保“先提示、再输入”的交互场景不会丢输出
	- 小结口诀
		-  `<<` 格式化，`put` 单字符，`write` 块数据，`flush` 立即刷；关联输入自动刷，`endl` 换行+刷
3. 格式化输出（`<iomanip>`与成员函数版）
	- 头文件
		- 任何**带参数**的流操纵符（`setw`、`setprecision`、`setbase`...）必须：`#include <iomanip>`
	- 整型基数控制
		- 操纵符（**持久生效**，直到被覆盖）
		- `oct / dec / hex` 切换 8/10/16 进制
		- `setbase(8/10/16)` 同效
		- `showbase` 显示前缀： `0`（八进制）、`0x`（十六进制）；`noshowbase`关闭
		- `uppercase` / `nouppercase` 控制十六进制字母大小写
		- 示例
			- `int n = 30;`
			- `cout << oct << n;             // 输出 36 （无前缀）`
			- `cout << showbase << hex << n; // 输出 0x1e`
			- `cout << dec;                  // 恢复十进制`
	- 浮点数精度
		- `cout.precision(n)` 成员函数，对所有后续浮点输出生效，直到下次设置
		- `setprecision(n)` 操纵符，效果同上，可链式：`cout << setprecision(3) << x;`
		- 精度含义配合 `fixed / scientific`
			- 默认模式：精度 = 有效数字总数
			- `fixed`：定点小数，精度 = 小数点后位数
			- `scientific`：科学计数，精度 = 小数点后位数（指数始终一位）
			- 示例
				- `double x = 123.456789;`
				- `cout << fixed << setprecision(3) << x;   // 123.457`
				- `cout << scientific << setprecision(3) << x; // 1.235e+02`
	- 域宽与对齐
		- `width(n)` 成员函数，仅影响下一次输出；之后自动回归 0（即“按需长度”）
		- `setw(n)` 操纵符，效果同 `width`，可链式
		- 对齐方式（持久）
			- `left` 左对齐，右边填填充符
			- `right` 右对齐（默认），左边填填充符
		- 填充字符
			- `setfill(ch)` 设置任意字符（含 '0'）；持久生效，直到被覆盖
		- 综合示例
			- `cout << setw(5) << setfill('0') << right << 12;   // 00012`
			- `cout << left << setw(5) << setfill('*') << 12;    // 12***`
	- 正号显示
		- `showpos` / `noshowpos` 控制正数前是否输出 '+'；常用于财务/科学表格
	- 跳过空白
		- 输入流默认 `skipws`；若想读入空格，用 `noskipws` 复位
		- `cin >> noskipws >> ch;   // 把下一个空白字符读入 ch`
	- 设置持久性速记
		- 持久（直到显式改）：`precision`、`fill`、`locale`、`flags`（对齐/进制/正负号/大小写）
		- 一次性：`setw`
	- 恢复默认
		- `cout.unsetf(ios_base::adjustfield | ios_base::basefield | ios_base::floatfield); // 清除对齐、进制、浮点标记`
		- `cout.setf(ios_base::dec, ios_base::basefield);`
	- 性能提示
		- 频繁设置/恢复格式会略微增加代码体积；
		- 对热路径可：一次性设置好所需标志；用 `ostringstream` 先拼格式串，再整体输出，减少多次刷新
	- 小结
		- 精度持久、宽度一次、进制/对齐/填充持久、科学/定点/显示基/正号持久；
		- 复位用 `unsetf` 或对应 `no-` 操纵符
4. `cin` 输入错误处理与字符级读取
	- 错误状态位（`iostate`）
		- 位于 `std::ios_base`，可用以下成员函数检测
		- `bool eof()`：遇到文件结束符（Ctrl-D / Ctrl-Z）
		- `bool fail()`：格式错误 或 未读到预期数据（如输入 `"abc"` 给 `int x`）
		- `bool bad()`：底层 I/O 失败（磁盘错误、管道破裂等）
		- `bool good()`：上述三者全为 false 时返回 true
		- 一旦流进入错误状态，后续所有提取操作自动成为空操作，必须手动清除
	- 典型检测模式
		- `if (cin.eof()) { cout << "正常到达文件尾\n"; }`
		- `else if (cin.fail()) { cout << "格式错误，需清理\n"; cin.clear(); // 清除错误标志，此时错误字符仍在缓冲区，需手动丢弃 }`
	- 清除与同步
		- `cin.clear()` 仅复位状态位，不丢弃导致失败的字符
		- 常用“丢到行尾”套路
			- `cin.clear();`
			- `cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');`
	- 字符级读取（不跳过空白）
		- `int cin.get()`
			- 作用：读 一个字符（含 `EOF`）
			- 终止条件：返回 int（`EOF` 为 -1）
			- 终止字符去向：—
		- `istream& cin.get(char& ch)`
			- 作用：读 一个字符（含 `EOF`），存到  ch
			- 终止条件：引用形参
			- 终止字符去向：—
		- `istream& cin.get(char* buf, streamsize n)`
			- 作用：读 n-1 字符 或 到  `'\n'`
			- 终止条件：缓冲区满或换行
			- 终止字符去向：留在缓冲区中
		- `istream& cin.get(char* buf, streamsize n, char delim)`
			- 作用：自定义分隔符 `delim`
			- 终止条件：读满或遇到 `delim`
			- 终止字符去向：留在缓冲区中
		- `istream& cin.getline(char* buf, streamsize n, char delim)`
			- 作用：自定义分隔符 `delim`
			- 终止条件：读满或遇到 `delim`
			- 终止字符去向：从缓冲区中移除
	- 读取示例
		- `char str[10], ch;`
		- `cin.get(str, 10, '#');   // 读到 '#' 为止，'#’ 仍留在流`
		- `cin.get(ch);             // 再读一次即可拿到 '#’`
	- 空白字符处理
		- 提取运算符 `>>` 自动跳过 前导空白（空格、换行、制表）
		- `get/getline` 不会跳过，可读取空格与换行
	- `EOF` 判别
		- `int c; while ((c = cin.get()) != EOF) { ... }`
		- 键盘输入 `EOF`：Unix `Ctrl+D`、Windows `Ctrl+Z` 后回车
	- 异常安全
		- 字符读取函数不抛异常；失败仅置位，由调用者检测
	- 最佳实践
		- 数值循环用 `while (cin >> x)`，失败即停
		- 格式错后 先 `clear()` 再 `ignore()`，否则死循环
		- 要读整行含空格：`std::getline(cin, string);`
		- 要读固定长度或到分隔符：`cin.get / cin.getline`
		- 混用 `>>` 与 `getline` 时，务必 `ignore()` 残留换行
