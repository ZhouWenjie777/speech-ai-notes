1. 函数使用
	- 原型：
		- 返回类型 函数名(形参表); 放在调用前或头文件，供编译器做类型检查
	- void函数：
		- 无返回值，调用独立成句，如 cheers(5); 执行完即返回
	- 有返回值：
		- 函数体用 return 表达式; 表达式类型须与声明一致；调用可嵌入表达式，如 volume = cube(side); 或 cheers(cube(2)); 先求值再传参
	- 参数传递：
		- 实参到形参按值复制，函数内修改不影响调用者；返回值通过临时对象传回
2. 值传递
	- 机制：函数调用时，实参值复制给形参，形参独立于实参，函数内修改形参不影响实参。
	- 多参数：用逗号分隔实参列表，如 area(3.7, 5.4) 或 area(r, h)，一一对应传递。
	- 表达式实参：允许，如 area(2 * r, 3.7 + 1.5)，先求值再传入形参。
	- 避免副作用：实参用表达式时需谨慎，如 area(++r, r) 可能引发未定义行为，因 r 的值在调用前被修改
3. 数组传递
	- 本质：传递数组首地址，形参指针与实参数组首地址指向同一块内存，修改形参数组会改变实参。
	- 参数：需传数组名（指针）和长度，如 `int sum_arr(int arr[], int n);` 调用 `sum_arr(cookies, ArSize);`
	- 指针表示：可用指针传数组，如 `int sum_arr(int *arr, int n);` 调用 `sum_arr(&cookies[0], ArSize);`
	- 灵活引用：指针可偏移，如 `int *p = cookies + 2; sum_arr(p, ArSize - 2);` 只计算部分元素。
	- 优点：节省时间和空间，无需复制数组内容。
	- 缺点：实参数组可能被修改，需用const限定符保护，如 `void func(const int arr[], int n);`
4. C风格字符串传参与返回
	- 传参：
		- 只传`const char* str`即可，函数从首地址顺序读到`'\0'`；例`c_in_str`统计字符出现次数，无需额外长度参数。
	- 返回：
		- 若需在函数外继续使用，必须返回指向**动态数组**或**静态数组**的指针；例`buildstr`用`new char[n+1]`创建串，调用者负责`delete []`。
	- 注意：返回局部数组名是未定义行为；动态返回虽合法，但易忘`delete`导致泄漏，优先改用`std::string`
