1. 定义及使用
	- 目的
		- 把“类型”作为参数，不同数据复用同一份类实现，达到“代码重用 + 类型安全”
	- 定义格式
		- `template <typename Item>` `class Queue { ... };`
		- 类名后加 `<Item>` 表示整个类体里可用 `Item` 当作具体类型
	- 成员函数定义（类外）
		- 每个成员都是函数模板，必须带模板头
		- `template <typename Item>` `返回类型 Queue<Item>::函数名(形参) { ... }`
		- 注意：类名用 `Queue<Item>`，不能裸写 `Queue`
	- 实例化
		- 编译期用“真实类型”替换 `Item`，生成一份专用类代码
		- 语法：`Queue<Customer> line(20);`，生成 `Queue<Customer>` 类并创建对象
	- ~~类型别名~~
		- 可在类内用 `typedef Customer Item;` 简化书写，但属于**旧式写法**；推荐直接用模板参数
	- 使用方式
		- 模板类对象与普适对象完全相同：`line.enqueue(cust);``Customer c; line.dequeue(c);`
		- 编译器会对 `Item = Customer` 生成对应版本的 `enqueue/dequeue`
	- 多文件注意
		- 类模板**声明与定义**通常放在同一头文件（或 `.hpp` 内联），否则其它翻译单元实例化时找不到定义会链接失败。
	- 常见实例
		- `Queue<int> intQ(100);` `Queue<double> doubleQ;`
		- 同一模板，两种类型，两份独立代码，零运行时开销
2. 栈类模板（非类型参数）
	- 设计目标
		- 基于数组实现后进先出（LIFO）容器，数组大小在**编译期确定**，避免动态分配，提高效率
	- 模板声明：`template <class Type, int MAX>`
		- `Type`：元素类型（类型参数）
		- `MAX`：数组大小（非类型参数），必须是编译期常量（整型、枚举、指针）
	- 数据成员
		- `Type items[MAX];`：定长数组
		- `int top;`：栈顶指针（指向下一空位）
	- 成员函数
		- 构造：`Stack() { top = 0; }`
		- `bool push(const Type &item)`：若 `top < MAX` 则 `items[top++] = item;` 返回 true，否则返回 false
		- `bool pop(Type &item)`：若 `top > 0` 则 `--top; item = items[top];` 返回 true，否则返回 false
		- `bool isempty()`：`top == 0` 
		- `bool isfull()`：`top == MAX`
	- 实例化示例
		- `Stack<std::string, 10> st;`
		- 编译器生成大小为 10 的 `std::string` 专用栈类；`MAX` 必须是字面量或 `constexpr` 变量 
	- 使用演示：循环读取命令
		- `A`：读入字符串 → `st.push(po)`
		- `P`：若栈非空 → `st.pop(po)` 并打印
		- `Q`：退出
		- 全程无动态内存，所有操作 O(1)
	- 与类型参数组合 
		- 可同时出现多个非类型参数：`template <class T, int ROW, int COL>`
		- 用于矩阵、缓冲区、静态表等编译期大小已知的容器
	- 非类型参数限制
		- 只能是整型、枚举、指针类型；浮点、类类型不允许
		- 类方法体内不能修改该参数值，也不能取地址（它是模板常量）
3. 安全数组模板 `ArrayTP`
	- 功能：编译期固定大小、运行时下标越界检查的数组容器，演示“非类型模板参数 + 运算符重载”。
	- 模板头：`template <class T, int n>`，第二参数`n`为**非类型模板参数**，必须是编译期整型常量，用于定义内部数组`T ar[n];`
	- 下标运算符重载（只能作为成员函数）
		- 非 const 版本
			- `T& ArrayTP<T, n>::operator[](int i)`
			- 内部断言 `if (i < 0 || i >= n)` 打印错误并 `exit(EXIT_FAILURE)`；否则返回 `ar[i]`，支持写操作
		- const 版本
			- `T ArrayTP<T, n>::operator[](int i) const`（或 `const T&`）
			- 提供只读访问，同样做越界检查；返回**值拷贝**可避免 const 引用悬空问题，也可返回 `const T&` 减少拷贝
	- 越界行为：触发时向 `std::cerr` 输出提示并立即终止程序；也可改为抛异常供外部捕获
	- 使用示例：`ArrayTP<int, 5> a;` 编译期生成 `int[5]` 若访问 `a[5]` 立即报错退出
	- 模板实例化：每对 `<T,n>` 都产生一份独立类代码；`ArrayTP<double,10>` 与`ArrayTP<int,5>` 完全是两个类型
	- 继承友好：把 `operator[]` 声明为 `virtual`，允许后续派生类改成抛异常、记录日志等策略，而保持接口一致
	- 限制：`n`必须是编译期常量表达式，不能运行时决定；若需动态大小应使用  `std::vector`或自行管理堆内存
4. 类模板的多功能性总结
	- 作为“类模板参数”
		- 把实例化的模板类当成类型给另外模板：`ArrayTP<node<double>, 100> s1;` 
		- 含义：元素类型是 `node<double>`，数组大小 100
		- 结果：编译期再生成一份`ArrayTP<node<double>, 100>`的专属代码，内嵌链表节点对象
	- 作为“类的成员”
		- 把实例当成数据成员（无名子对象）：`ArrayTP<ArrayTP<int, 5>, 7> a;`
		- 内层 `ArrayTP<int, 5>` 是一个“行”类型（5 个 int）
		- 外层 `ArrayTP< ... , 7>` 得到 7 行，即 7×5 二维矩阵
		- 访问：`a[i][j]` 先取第 i 行对象，再取行内第 j 列元素
	- 作为“基类”
		- 公有/保护继承，复用已有模板实现并追加功能
		- `template<class T, int low, int high>` `class ArraySUB : public ArrayTP<T, high - low + 1>`
	- 递归组合价值
		- 通过“模板套模板”即可在零运行时成本下得到
			- 任意高维数组
			- 任意下标区间数组
			- 元素本身又可以是模板实例（如节点、矩阵）
			- 一套代码，无限组合，全部类型安全且内联展开
5. 类模板的具体化（显式 / 部分 / 显式实例化）
	- 显式实例化（explicit instantiation）
		- 声明语法：
			- `template class 类名<实际参数>;` 
			- `template class ArrayTP<double, 5>; ArrayTP<double, 5> a`
		- 作用：
			- 编译器立即为整个类模板生成一份完整实例代码（包括所有成员函数），即使某些函数从未被调用
		- 用途：
			- 把模板定义放在 `.cpp` 而接口放 `.h` 时，用显式实例化避免链接期“找不到符号”错误
			- 控制二进制大小：只生成需要的版本，防止隐式实例化泛滥
	- 隐式实例化（implicit instantiation）
		- 仅当代码真正“使用”到某个模板实参组合时，编译器才按需生成对应类及被调用成员函数的代码
		- 示例：
			- `ArrayTP<int, 10> *p;` 只实例化“类”本身，不生成任何成员
			- `p = new ArrayTP<int, 10>;` 实例化构造函数、`operator[]`
			- `cout << p->foo();` 若 foo 存在错误，此处才报错
		- 优点：编译速度快、目标文件小
		- 缺点：错误延迟暴露（如上例foo中存在的问题，只有调用时才报错）
	- 显式具体化（full/explicit specialization）
		- 对所有模板参数给定具体类型，提供完全替代实现
		- 语法：`template <> class sortarray<const char*> { ... };`
		- 当实参完全匹配时优先选用，不再实例化通用模板
	- 部分具体化（partial specialization）
		- 对模板参数部分加以约束，提供专属实现
		- 语法：
			- 通用版本：`template<class T1, class T2, class T3> class sample { ... };`
			- 部分具体化版本：`template<class T1, class T2> class sample<T1, T2, int> { ... };` 把第三参数固定为 int
		- 匹配规则：编译器选择“最特化”版本
	- 使用建议
		- 库接口保持通用模板，性能关键或语义特殊时再提供具体化版本
		- 部分/显式具体化必须与主模板同名且参数列表可区分，否则属于重定义
		- 在头文件中写具体化时加`inline`或放`.cpp`并显式实例化，避免多重定义链接错误
6. 约束 vs. 非约束 模板友元
	- 约束模板友元（bound template friend）
		- 声明
			- `template <class T, int n>`
			- `class ArrayTP { friend ostream& operator<< <T, n> (ostream&, const ArrayTP<T, n>&); };`
		- 特点
			- 友元函数模板参数与类模板参数一一对应
			- 只有相同实例之间才是友元；不同`<T, n>`组合生成各自的友元副本
			- 可访问对应实例的所有私有成员，实现类型安全的输出/比较等操作
		- 示例
			- `ArrayTP<double, 5> a;`
			- `cout << a;` 与 `operator<< <double,5>` 对应匹配
	- 非约束模板友元（unbound template friend）
		- 声明
			- `template <class T, int n>`
			- `class ArrayTP { template <class T1, class T2> friend void show(const T1& t1, const T2& t2) };`
		- 特点
			- 友元模板参数与类模板参数无关；编译器为每一种 `<T1, T2>` 组合生成一个友元函数，该函数可访问所有`ArrayTP`实例（任意`<T, n>` ）的私有成员
			- 一次定义，全局通用；适合“跨类型”工具函数，例如同时打印两个不同实例的首元素
		- 示例
			- `ArrayTP<double, 5> a; ArrayTP<int, 10> b;`
			- `show(a, b);` 同一 `show<double, int>` 可访问 a、b 的私有 `ar[0]`
	- 非模板友元（最简形式）
		- `friend void f();`
		- `f`成为所有`ArrayTP`实例的友元，但本身不是模板，无法参数化
	- 权限与实例化
		- 友元函数模板只有在被调用时才会被实例化；未被使用则不生成代码
		- 约束友元更安全——只暴露给“同参”实例；非约束友元功能更强，但潜在暴露面更大，需权衡使用
	- 常见组合
		- 约束：输出、比较、算术运算符——需要类型一一对应
		- 非约束：跨实例复制、交换、统计——需要同时操作多种实例
		- 二者可并存，互不冲突；编译器根据调用实参选择最匹配的友元模板实例
