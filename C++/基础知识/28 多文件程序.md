1. 多文件程序
	- 划分原则：
		- 按功能或模块把函数分散到多个`.cpp`源文件
		- `main`单独一份，便于独立编译与链接
	- 源文件（`.cpp`）
		- 只放函数定义与模块私有实现；
		- 谁要用到里面的函数，就包含声明了该原型的对应头文件
		- 例：`struct.cpp` 实现 `rect_to_polar()、show_polar()`；`main.cpp` 实现 `int main()`；各自 `#include "struct.h"` 拿到原型与类型
	- 头文件（`.h`）
		- 只放“声明”：结构体、类、符号常量、内联函数、普通函数原型
		- 加保护符 `#ifndef A #define A … #endif` 防止重复展开
	- 编译流程
		- `g++ -c struct.cpp → struct.o`
		- `g++ -c main.cpp → main.o`
		- `g++ struct.o main.o -o app` 链接成可执行文件
	- 依赖管理：
		- 改 `.cpp` 只需重编该单元
		- 改 .h 所有包含它的 `.cpp` 都要重编
		- 因此头文件应“最小公开”，把只在源文件内部用的声明/定义放到 `.cpp` 里，减少耦合
2. 多文件编译链接
	- 单命令编译：
		- UNIX 下 `cc file1.cpp file2.cpp` 一次性完成全部步骤，适合小项目
	- 四步流程：
		- 预处理器：把 `#include` 头文件内容插入对应源文件，生成临时翻译单元 `temp1.cpp`、`temp2.cpp`
		- 编译器：分别将翻译单元编译成目标文件 `file1.o`、`file2.o`，包含机器码与未解析符号表
		- 链接器：合并所有目标文件、启动代码（runtime startup）和库代码（`libc++`、`libm` 等），解析外部符号，生成单一可执行文件 `a.out`
	- 优势：
		- 修改单个源文件只需重编对应 `.o` 再链接，显著缩短大型项目构建时间
		- 可配合 `Makefile` 实现增量编译
