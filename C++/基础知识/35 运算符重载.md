1. 运算符重载
	- 目的：
		- 让类对象像内置类型一样用运算符操作，如 `a3 = a1 + a2`
	- 实现方式：
		- 写特殊函数，名称为 operator@，@为待重载运算符；
		- 可作为成员函数或友元/普通函数
	- 成员函数版本：
		- 隐含 this 作为左操作数，形参比运算数少一个；
		- 例 `A operator+(const A& rhs) const {return A(/* *this + rhs */);}` 
		- 调用时 `a1 + a2` 被编译器解析为 `a1.operator+(a2)`。
	- 返回值：
		- 与运算结果类型一致；+、- 等算术运算通常返回新对象；
		- 比较运算符返回 bool
	- 限制：
		- 不能创建新运算符，不能改变原优先级、结合性与操作数个数
		- 不能重载 `.`  `.*`  `::`  `?:`  `sizeof`  `typeid` 等少数运算符
		- 至少一个操作数必须是用户自定义类型。
	- 使用示例：`A a, b, c;` 
		- `a = b + c;`         // 自动匹配 operator+ 
		- `if (b > c) …`     // 自动匹配 operator> 
		- 代码可读性高，符合直觉
2. 运算符重载实例——Time 类的加法
	- 设计目标：让两个 `Time` 对象能够像内置类型一样执行 `t3 = t1 + t2`。
	- 数据模型：`int hours; int minutes;`  内部保持分钟 < 60，溢出自动进位到小时
	- 核心重载：调用时 `t1 + t2` 被编译器解析成 `t1.operator+(t2)`，返回临时对象后赋给 `t3`
```cpp
Time Time::operator+(const Time & t) const
  {
      Time sum;
      sum.minutes = minutes + t.minutes;          // 先加分钟
      sum.hours   = hours + t.hours + sum.minutes / 60; // 进位
      sum.minutes %= 60;                          // 保留余数
      return sum;                                 // 按值返回新对象
  }
```
3. 运算符重载：成员 vs 非成员
	- 选择依据
		- 第一运算数必须是当前类对象 → 优先成员函数（少一个形参，隐含 this）
		- 第一运算数是其他类（如 `ostream/istream`）或需要对称转换 → 必须非成员（全局/友元）
		- C++ 强制规定：赋值 =、下标 []、调用 ()、成员指针 ->、递增递减前缀/后缀、new/delete 等**只能**重载为成员函数
	- 成员函数写法
		- `返回类型 Time::operator@(形参) const { ... }`
		- 形参个数 = 运算数 - 1；左操作数由 `*this` 提供
		- 例 `+=`：`Time& Time::operator+=(const Time& t)` 直接修改当前对象并返回 `*this`，支持链式 `t3 += t1 += t2;`
	- 非成员函数写法
		- `返回类型 operator@(左形参, 右形参) { ... }`
		- 若需访问私有成员，在类内加 `friend` 声明；否则保持普通全局函数
		- 例 `+` 对称加法：`friend Time operator+(const Time& l, const Time& r)` 可同时接受 `t1 + t2` 与 `2 + t1`（借助构造函数隐式转换）
	- 转换友好性
		- 非成员版本允许左、右操作数发生用户自定义转换
		- 成员版本仅右操作数可转换。因此数值与对象混写时`2 + t1`必须用非成员实现
	- 常见决策表：运算符：推荐实现方式
		- `=  []  ()  ->`：只能成员
		- `+=  -=  *=  /=`：成员（修改第一操作数）
		- `+  -  *  /`：友元/全局（保持对称）
		- `<< >>`：友元/全局（左操作数是 `ostream/istream`）
