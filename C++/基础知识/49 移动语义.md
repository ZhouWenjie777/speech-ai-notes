1. 移动语义（Move Semantics）——接管而非拷贝
	- 触发场景
		- 当源对象是**右值**（临时量、`std::move` 显式转换）且类提供了**移动构造/移动赋值**时，编译器优先选择移动版本，避免深拷贝，从而改善时间性能
	- 关键函数签名
		- ```cpp
			  // 移动构造
			  Useless(Useless&& f) noexcept : n(f.n), pc(f.pc) {
				  f.pc = nullptr;   // 置空源对象，确保“独占”
				  f.n  = 0;
			  }
			  // 移动赋值
			  Useless& operator=(Useless&& f) noexcept {
				  if (this != &f) {
					  delete[] pc;          // 释放自身旧资源
					  n  = f.n;
					  pc = f.pc;            // 窃取
					  f.pc = nullptr;
					  f.n  = 0;
				  }
				  return *this;
			  }
		  ```
	- 运行流程
		- `one` + `three`  产生临时对象 `temp`（右值）
		- `Useless four(one + three);` 匹配移动构造，`temp.pc` 直接给 `four.pc`， `temp` 被置空
		- 临时对象析构时因 `pc == nullptr` 不再 `delete[]`，无二次释放
	- `noexcept`承诺
		- 移动操作应标记 `noexcept`，否则容器（ vector ）在重分配时不敢使用移动，仍会回退到拷贝
	- 规则速记
		- 三五法则补充：若自定义析构/拷贝/赋值，一般也要手写移动构造和移动赋值，或 `=default` 显式生成
		- 移动后源对象处于可析构、可赋值状态，但不可假设仍拥有资源（被置空）
		- 注意：
			- 移动后的源对象不是右值本身，只是被当右值用过一次`std::move()`
			- 它依旧有名字、有存储、有生命周期，因此可当左值再赋值
	- 性能收益
		- 深拷贝类（ `vector`、`string`、动态数组）从 O(N) 复制 降为 O(1) 指针移交；对大型对象/容器重分配、返回值优化效果显著
	- 禁用移动
		- `Useless(const Useless&) = default; // 显式要求生成“默认的拷贝构造函数”`
		- `Useless(Useless&&) = delete; // 禁用移动构造`
		- `Useless& operator=(Useless&&) = delete; // 禁用移动赋值`
	- 标准库已全面支持
		- `std::vector`、`std::string`、`std::unique_ptr`等均有 `noexcept`移动成员
		- 算法 `std::move_iterator` 可把拷贝算法变移动算法
			- `std::vector<std::string> src = {"a", "b", "c"};`
			- `std::vector<std::string> dst(std::make_move_iterator(src.begin()), std::make_move_iterator(src.end())); // dst接管src内部缓冲区，src变空串` 
	- 与右值引用关系
		- `T&&` 是绑定到右值的引用类型；移动构造/移动赋值的参数即是它，从而区分“拷贝”与“移动”两个重载
	- 小结
		- 移动不是复制，而是**资源窃取+置空源对象**
