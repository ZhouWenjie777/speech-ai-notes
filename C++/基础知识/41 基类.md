1. 重定义基类函数（覆盖 / 隐藏）
	- 规则：
		- 派生类定义与基类**同名**函数（不管参数是否相同）即形成**隐藏**（hide），而非重载；通过派生类对象/指针调用时，只会找到派生版本，基类版本被“盖住”
		- 非 virtual → 静态绑定，基类对象/指针调用只会找到基类版本
		- 是 virtual → 动态绑定，基类对象/指针调用会根据实际对象类型去找派生类的覆盖版本
		- 一句话：非 virtual 看指针/引用类型；virtual 看实际对象类型
	- 覆盖（override）前提：
		- 函数签名完全一致 + 基类函数为 **virtual**；此时通过基类指针/引用调用，会依据实际对象类型走派生实现；否则仅是普通隐藏
	- 示例行为差异：
		- `Brass::Withdraw`：只允许余额内取款，超限拒绝。
		- `BrassPlus::Withdraw`：先检查余额，不足部分走透支；若透支后仍超限，拒绝并输出罚金信息
		- 对同一消息 `ba.Withdraw(amt)`，对象真实类型决定执行路径：
			- 若 `Brass` 对象 → 走基类版本
			- 若 `BrassPlus` 对象 → 走派生版本（即使通过基类指针调用，只要函数为 virtual 即动态绑定）
	- 派生类内“复用”基类逻辑
		- 使用**完全限定名** `Base::func()` 显式调用被隐藏的基类实现，避免代码重复；
		- 例 `Brass::Withdraw(amt);` 先完成余额扣款，再在派生类处理透支部分
	- 建议：
		- 若预期多态，把基类对应函数声明为 `virtual`；派生类函数写 `override`（C++11）显式检查签名一致性
		- 若仅为功能扩展而不需要多态，可用隐藏，但文档注明“派生类屏蔽基类版本”
	- 内存与对象切片：
		- 非 virtual 且按值传递派生对象给基类参数时，发生对象切片（slicing），只拷贝基类子对象，派生部分被切掉；应按引用/指针传递
2. 抽象类
	- 定义：
		- 含有**至少一个纯虚函数**的类；纯虚函数声明格式：`virtual 返回类型 函数名(参数表) = 0;` 无函数体，直接“=0”标识。
	- 规则约束：
		- 不能创建抽象类对象（`Shape s;` 编译错误）
		- 不能用作参数类型、返回值或显式转换目标；
		- **可声明指针/引用**，用来指向派生类对象，实现多态
	- 派生责任：
		- 若派生类**全部实现**了基类所有纯虚函数，则派生类变为**具体类**，可实例化；
		- 只要有一个纯虚函数未实现，派生类**仍是抽象类**，继续禁止实例化
	- 设计意义：
		- 强制接口统一：抽象基类定义“能做什么”，派生类决定“怎么做”
		- 运行期多态：通过基类指针遍历异质集合，调用同名虚函数却执行不同算法，例 `ptr->area()` 对矩形返回 `w*h`，对圆返回 `π*r²`。
	- 实例流程：
		- 抽象基类 `shape` 提供 `virtual void area() = 0;`
		- 派生 `rectangle` 与 `circle` 分别实现 `area()`
		- `shape *ptr;` 可依次指向`rectangle`对象、`circle`对象，统一调用`ptr->area()`得到各自面积，实现“一个接口，多种实现”
