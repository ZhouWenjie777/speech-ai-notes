1. 静态数据成员
	- 共享属性：
		- 整个类只有一份拷贝，被所有对象共享，不属于单个对象，因此 sizeof 不计入其大小
	- 声明：
		- 类内写 `static 类型 变量名;` 如 `static double m_rate;` 
		- 仅为声明，不是定义，不分配存储
	- 定义（分配存储）：
		- 必须且只能在类外（通常放 `.cpp`）写`类型 类名::变量名 = 初值;`
		- 例：`double SavingAccount::m_rate = 0.05;`
		- 不加 static，否则重定义；可缺省初值，默认 0
	- 访问两条路：
		- 通过类：`SavingAccount::m_rate`
		- 通过对象：`obj.m_rate`（语法糖，实际仍走类作用域）
	- 典型用途：保存类级别常量（利率、版本号）、对象计数器、共享缓存等
	- 特殊规则:
		- 静态数据成员可以是 incomplete 类型、自身类类型、数组等，而普通成员只能是 complete 类型
		- 静态数据成员不能是 mutable；可在 `constexpr` 下类内初始化（C++17 起允许 `inline static` 成员变量类内定义）
2. 静态成员函数
	- 声明：
		- 类内写 `static 返回类型 函数名(形参表);` 如 `static void SetRate(double);` 专用于处理静态数据成员
	- 定义：
		- 类外写 `返回类型 类名::函数名(形参表) { ... }` 不再加 static，否则重定义
	- 特点：
		- 没有隐式 this 指针，因此**不能访问非静态数据成员或非静态成员函数**；只能操作静态成员或调用其他静态成员函数
		- 调用不依赖对象：可通过类名 `类名::函数名()` 或对象 `对象.函数名()` 两种语法，效果相同
	- 典型用途：
		- 管理静态数据：如利率、版本号、全局计数器
		- 工厂/工具接口：提供与具体对象无关的创建、查询、配置功能
		- 跨对象统计：示例中 `obj_count` 记录历史创建总数，`obj_living` 记录当前存活数；构造函数 `++` 两者，析构函数 `--` 后者，`static void display()` 随时输出统计结果
	- 约束与注意
		- 不能被声明为 const（无 this，谈不上对象只读）、volatile 或 virtual
		- 静态成员函数指针类型为`返回类型 (*)(形参)`，与普通函数指针兼容，可用于回调或 C 接口
