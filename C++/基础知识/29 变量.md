1. 作用域与存储持续性
	- 块作用域：在复合语句或函数体内声明，从定义点到块结束；形参亦属此类，离开块即销毁。
	- 文件作用域：在函数外声明，从定义点到文件末尾；默认全局链接，其他文件可用 `extern` 引用。
	- 全局变量：文件作用域且外部链接，如 int global; 整个程序共享，首次定义处分配，程序结束释放。
	- 局部变量：块作用域，自动存储期，进入块时创建，出块时销毁；同名局部变量屏蔽外层同名标识符。
	- static 限文件：`static int fileVar;` 仍文件作用域，但内部链接，仅当前翻译单元可见，避免命名冲突。
2. 存储持续性
	- 自动存储持续性：
		- 块内声明的变量（含形参），进入块时创建，离开块时销毁
		- C++11起auto用于类型推导，不再表示存储类
	- 寄存器变量：
		- `register int x;` 建议编译器放寄存器以加速，现代编译器常忽略，仅供提示
	- 静态存储持续性：从分配点到程序结束一直存在，分三种链接性：
		- 内部链接：`static int fileVar;` 文件作用域，仅当前翻译单元可见
		- 外部链接：`int globalVar;` 文件作用域，全程序可见，其他文件用`extern`引用
		- 无链接：`static int blockVar;` 块作用域，首次执行时创建，以后保持值，仅所在块可见
3. 静态存储持续性
	- 外部链接：
		- 定义分配存储空间，只能出现一次；声明用 `extern` 仅告知类型，可多处出现
	- 外部变量：
		- 在函数外定义且无 static，具有外部链接，全程序可见；未显式初始化时编译器自动置 0
	- 内部链接：
		- 函数外用 static 修饰，如 `static int x`; 仅当前翻译单元可见，其他文件即使 `extern` 也无法访问，避免符号冲突
	- 无链接静态：
		- 函数或块内用 static 修饰，如 `static int count`; 在首次执行时创建，以后保持原值，函数退出也不销毁，实现“局部可见、全局寿命”
	- 单定义规则（`ODR`）：同一程序中每个外部变量只能有一处定义，但可有多处 `extern` 声明；违反将在链接阶段报错
