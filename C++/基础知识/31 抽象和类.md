1. 类
	- 概念：用户自定义类型，把数据（数据成员）与操作（成员函数）封装在一起，实现抽象与数据隐藏。
	- 访问控制：
		- public：类外部可见，用作接口；出现顺序任意，可多次出现
		- private：仅类内部成员函数可访问，实现数据隐藏；数据成员通常放此处。
		- 未指定时 class 默认 private，struct 默认 public，其余功能相同。
	- 声明与实现分离：
		- 头文件（.h）：
			- 只放类声明，包括数据成员、成员函数原型、内联函数定义；
			- 用宏保护符防止重复包含
		- 源文件（.cpp）：
			- 用`返回类型 类名::函数名(形参表){ ... }`实现成员函数；
			- 非内联成员函数只能有一份定义
	- 内联：成员函数直接在类体内定义时自动成为内联候选；显式 inline 也可放在类外，但需与声明同处头文件
	- 封装优势：外部代码无法直接篡改内部数据，所有访问必须通过公有接口，降低模块耦合，提高可维护性
2. 类的使用
	- 对象定义：
		- 声明类名后，可创建单个对象、对象数组或对象指针
		- 如 `Stock fluffy; Stock sArray[10]; Stock *sp;`
	- 成员访问：
		- 对象用句点运算符`.` ，指针用箭头 `->` 
		- 仅 public 成员对外可见；私有成员只能由同类成员函数或友元访问
	- 动态对象：
		- `Stock *ps = new Stock;` 通过指针调用 `ps->acquire(...);` 
		- 使用完毕必须 `delete ps;` 释放资源，防止内存泄漏
	- 对象数组：
		- `sArray[i].show();` 按下标访问，默认调用无参构造函数
		- 若需带参构造须提供初始化列表或逐个赋值
	- 封装效果：外部代码无法直接读写私有数据成员，所有操作必须通过公有成员函数，保证数据合法性与一致性
3. 构造函数
	- 定义：
		- 与类同名、无返回类型（连 void 也不写）的特殊成员函数，
		- 专门负责“对象诞生”时的初始化工作；可重载，可带默认实参
	- 调用时机：
		- 对象定义、new 动态创建、按值传参 / 返回时由编译器自动调用
		- 用户不可手工触发。
	- 默认构造：
		- 若程序员未提供任何构造函数，编译器自动生成无参空体版本；
		- 一旦自定义任何构造函数，默认版本消失，如需继续使用必须显式写出 `Stock() = default;` 或自行定义
	- 列表初始化（C++11）：
		- 支持花括号语法 `Stock st1{ "comp1", 20, 3.4 };` 与圆括号等价，均匹配对应构造函数；列表为空时调用默认构造函数
	- 显式调用：
		- 可用临时对象语法给已存在对象重新赋值，如 `st1 = Stock("comp2", 40, 5.4);` 右侧先构造临时对象，再移动 / 拷贝给左侧，随后临时对象被析构
	- 默认实参方案：
		- 在构造函数声明中为尾部参数提供默认值，如 `Stock(const std::string & co, long n = 0, double pr = 0);` 
		- 既可当默认构造用 `Stock st;` 也可当三参构造用 `Stock st("ACME", 100, 3.2);` 一个函数兼顾多种初始化需求
4. 析构函数
	- 原型：
		- `~类名();` 无参、无返回，不可重载；对象生命周期结束（离开作用域、delete、容器销毁）时由编译器自动调用，做善后清理。
	- 动态内存场景：
		- 若构造函数用 new 申请资源（如 `char* company = new char[len+1];`），必须手写析构函数释放，否则每次对象销毁都泄漏一块堆内存
	- 默认行为：
		- 未显式定义时编译器生成空析构函数；一旦自定义，编译器不再合成，所有清理逻辑由程序员负责。
	- 调用顺序：
		- 先执行析构函数体 → 再按成员声明逆序调用各成员对象的析构 → 最后释放对象本身内存
		- 继承时先派生后基类
	- 规则小结：有“裸”资源（`new/malloc`、文件句柄、锁）就要配套写析构；无资源需求可省略，让编译器生成空实现即可
5. 类作用域
	- 概念：类内声明的名字（成员变量/类型/枚举/成员函数）作用域贯穿整个类；不同类可同名互不影响。
	- 访问方式：
		- 类内：成员函数直接写名字即可，编译器自动视为 this->名字
		- 类外：需加类名限定 类名::名字，如 `Stock::show()、salary::income`
	- 共享常量（**所有对象共用**）：
		- static const 非整型
			- C++11前：类内声明 `static const double income;`，类外定义 `const double salary::income = 123.456;`
			- C++11 起允许类内直接给初值：static const double income = 123.456; 若后续取地址仍需类外定义一次（不带初值）。  
		- 整型静态常量：
			- 允许类内直接给初值 `static const int month = 12;` 并立即用于数组大小等编译期上下文，不取地址时可省略类外定义。
		- 枚举法：
			- `enum { month = 12 };` 生成整型符号常量，不占存储
			- 语法上属于类作用域，可替代 static const int
		- 注意：枚举成员与 static const int 均无法取地址；非整型静态常量必须类外定义后才能取地址或链接。
6. 类的类型转换
	- 内置类型 → 类类型（隐式）
		- 只需一个“单参构造函数”即可；编译器在需要时自动调用
		- 例：`Rational(int n = 0, int d = 1)` 可把 `5` 转成 `Rational(5,1)`，于是
		- `r1 = 5;`              // 隐式转换
		- `r2 = 4 + r1;`     // 4 先被转成 Rational(4,1) 再相加
	- 类类型 → 内置类型 / 其它类类型
		- 必须提供“类型转换运算符”（也叫转换函数）
		- 格式：`operator 目标类型名 () const { return 目标类型表达式; }`
		- 规则：
			- 必须是**成员函数**，无返回类型（名字即返回类型），无参数，通常加 const
			- 不能指定返回类型，也不能有参数；可隐式被调用，也可用显式强制转换
		- 例
			- `operator double() const { return double(num)/den; }`
			- `double x = r;` 隐式调用
			- `x = double(r);` 显式调用
			- 若 r 为 1/3，x 得到 0.333333
	- 隐式转换控制（C++11 起）
		- 在构造函数或转换运算符前加 `explicit` 可禁止**隐式**转换，只能显式强制转换，避免意外行为
	- 类 → 类转换
		- 同样用转换运算符：`operator AnotherClass() const { ... }`
		- 或给目标类写一个“单参构造函数”接收源类对象，两条路线编译器都会尝试
	- 注意事项
		- 同时提供“内置→类”和“类→内置”双向转换时，可能出现二义性（如 `r1 == 2` 编译器不知把 r1 转 double 还是把 2 转 Rational），应加`explicit`或避免过度重载
		- 类型转换运算符可返回类类型，实现“轻量级代理”或“视图”对象，但须确保生命周期安全
7. 包含对象成员的类（has-a 关系）
	- 语义
		- 学生“有”生日（date 对象）、“有”一组成绩（score 对象数组），形成 **has-a** 组合，而非 is-a 继承
	- 成员对象初始化
		- 语法：构造函数的**初始化列表**负责；编译器先调用成员对象对应构造函数，再进入派生类函数体
		- 次序：按类定义中成员声明顺序执行，与列表书写顺序无关。
		- 例：`student(const string &n, int y, int m, int d) : birthday(y, m, d) {name = n; coursenum = 0;}`若把 `birthday` 放列表外写 `birthday = date(y,m,d);` 属于赋值，对 const/引用/无默认构造的成员会编译失败
	- 访问限制
		- 宿主类（student）**不能直接读写**成员对象的私有数据，必须通过其公有接口：`data[i].setcourse(c); data[i].setvalue(val);`,这强制封装，避免内部表示泄露
	- 典型接口实现
		- `inputscore(const string &c, int val)` 把新课程名与分数封入下一个数组元素，并递增 `coursenum`
		- `avg() const` 遍历数组累加 `data[i].getvalue()`，返回平均分
		- 重载 `operator<<` 时连续输出 `birthday` 与所有成绩，利用成员对象自身已重载的`<<`，形成级联打印
	- 内存布局
		- 宿主对象 = “成员子对象”顺序排列 + 自身新增成员
		- `sizeof(student)` 包含完整 `birthday` 与固定大小 `score data[MAX]`
	- 生命周期
		- 成员对象随宿主对象创建而构造，随宿主对象析构而析构；
		- 若成员类未提供默认构造函数，宿主类**必须**在初始化列表显式传参构造
	- 组合 vs 继承决策
		- 当“一部分”关系强、生命周期严格同步，用**组合**
		- 当“一种”关系且需要运行时多态，用**公有继承**
8. 私有继承（“is-implemented-in-terms-of”）
	- 访问控制
		- 基类所有 `public`/`protected` 成员在派生类中**全部降为 `private`**；
		- 外部以及派生类的派生类都再无法直接访问它们，但派生类自己的成员函数仍可自由使用，达到“实现复用而非接口复用”
	- 与“成员对象”对比
		- 成员对象：具名对象，生命周期与宿主严格同步，接口需显式转发
		- 私有继承：无名基类子对象，同样“生于构造、死于析构”，但可用 `using` 把个别基类方法拉到派生 `public` 区域，或直接在内部调用而无需写转发函数
	- 构造函数责任链
		- 写法与公有继承相同，先调基类构造再初始化自身
		- `student(const string &n, int yy, int mm, int dd) : date(yy, mm, dd) { name = n; coursenum = 0; }`
		- 若基类无默认构造，必须在初始化列表显式传参
	- 内部使用基类接口
		- 派生类成员函数可直接调用基类公用/保护方法
		- `os << static_cast<const date&>(obj);`，把自身引用转换为基类引用，利用基类已重载的 `operator<<` 输出日期部分，无需写转发函数
	- 对外不可见，外部代码：
		- `student s("Li", 2000, 5, 20);`
		- `s.Name();` OK，属于 student 自身接口
		- `s.SetDate(1,1,1);` ERROR，基类 date 所有 public 方法在派生类中已变成 private，外部无法访问
	- 再导出接口（可选）
		- 若希望个别基类方法对外仍可用，可在派生类写：`using date::SetDate;` 把该方法拉回 student 的 public 区域
		- 或封装一层 inline 转发函数，保持“最小暴露”原则
	- 适用场景
		- 仅需“实现”而非“接口”复用，且派生类对象**不被当作**基类对象使用（即不会 `Base* pb = &derived;`）
		- 需要访问基类 protected 成员或重写基类虚函数（成员对象不行）
		- 想利用空基类优化（EBO）节省一字节（成员对象至少占一字节）
	- 与组合决策口诀
		- 需要多态/保护成员/重写虚函数 → 私有继承；仅需功能嵌入 → 成员对象
9. protected 成员
	- 访问权限：介于 public 与 private 之间，**仅允许派生类的成员函数和友元访问**，对象外部依旧不可见
	- 用途：把“实现细节”暴露给子类，避免频繁通过 public 接口绕行，提高派生类代码效率与可读性
	- 示例：基类把 `balance` 设为 protected，派生类 `BrassPlus::Withdraw` 可直接读写 `balance`，无需先 `Deposit(advance)` 再走 public 接口
	- 规则提醒：
		- 派生类成员函数只能访问**自己对象**的 protected 成员，不能访问**其他独立的基类或派生类对象**的 protected 成员（即只允许‘纵向’继承关系内访问，“横向”访问被禁止）
		- protected 继承（`class Derived : protected Base`）会进一步把基类 public/protected 全部降为 protected，对外彻底隐藏，仅派生类内部可见；这与“protected 成员”是两个不同概念，不要混淆。
10. 多重继承
	- 场景与问题
		- **多基类实例冲突**：
			- 派生类继承多个基类时，若基类有共同基类，会导致该共同基类实例被重复继承多次
			- `teacher_doctor` 继承 `teacher` 和 `doctor`，而二者都继承 `people`，结果 `teacher_doctor` 对象里出现两个`people`实例，访问时编译器报错
		- **同名函数冲突**：
			- 不同基类有同名函数，派生类对象调用时编译器无法确定用哪个
	- 虚基类解决方案
		- 格式：
			- 在基类继承前加 `virtual` 关键字，如 `class teacher: virtual public people`
		- 效果：
			- 使共同基类在最终派生类里**仅有一份实例**，由最底层派生类直接构造，避免重复继承
		- 构造规则：
			- 最终派生类构造函数必须显式初始化虚基类，即使中间基类已初始化过
			- `teacher_doctor(参数表) : people(参数表), teacher(参数表), doctor(参数表) { }`
	- 同名函数解决方法
		- **作用域运算符**
			- 通过 `对象.基类名::函数名()` 明确指定调用哪个基类版本
			- `x.A::f(); x.B::f();`
		- **封装重载**
			- 在派生类里重新定义函数，封装基类同名函数，提供统一接口
			- `void fA() { A::f(); } void fB() { B::f(); }`
	- 多重继承语义
		- 公有多重继承表示 **is-a** 关系，如 `teacher_doctor` 是 `teacher` 也是 `doctor`
		- 私有/保护多重继承表示 **has-a** 关系，如 `C` 包含 `A` 和 `B` 的功能，但外部不能直接当 `A` 或 `B` 使用
	- 注意事项
		- 虚基类构造顺序：按声明顺序初始化，与继承图无关
		- 虚基类访问成本：多一次间接跳转，可能影响性能
		- 避免复杂继承图：尽量减少多层虚继承，降低理解与维护难度
11. 嵌套类（内部类）
	- 定义与位置
		- 在另一个类体内写 `class Node { ... };` 或 `struct Node { ... };` 即成嵌套类；
		- 可出现在外部类的 `private / protected / public` 任意区域，决定可见范围。
	- 访问权限规则
		- 嵌套类是**独立类型**，与外部类无自动 friend 关系
		- 外部类成员**只能访问嵌套类的公有成员**；若想用私有成员，必须在嵌套类里写 `friend 外部类;`
		- 外部类**不拥有**嵌套类对象，需要时再显式定义或 new
	- 三种场合
		- **private**（默认）
			- 仅外部类自己可见，用于隐藏实现细节；例 Queue 的 Node 只供队列内部链接使用，外部代码完全看不到 Node 类型。 
		- **protected**
			- 允许外部类及其派生类使用 Node；适合“工具节点”需要被派生类扩展的场景
		- **public** 
			- 任何代码都能通过“外部类::嵌套类”访问；适用于“同接口不同实现”需求，如各家电视遥控器共用类名 Remote，但内部实现不同
	- 语法示例
		- `template <class Item> class Queue { private: class Node {...}; // 仅 Queue 可见 public: bool enqueue(const Item& i) { Node* tmp = new Node(i); // 外部类正常使用嵌套类 }; }`
	- 公有嵌套类调用
		- `class SonyTv { public: class Remote { /* Sony 专属实现 */ }; };`
		- `SonyTv::Remote rmt;` 任意代码都可使用
	- 与友元搭配
		- 若嵌套类需要反向访问外部类私有成员，同样要在外部类里写`friend class 嵌套类;` 实现双向突破封装
	- 生命周期
		- 嵌套类对象与外部类对象完全独立；可在外部类成员函数里`new Node`也可在全局`new Outer::Node`，各自管理自己的生存期
	- 设计价值
		- 限域：
			- 把仅属于“外部类领域”的工具类型收拢到内部，避免污染全局命名空间
		- 多实现：
			- 同名嵌套类在不同外部类里可以有完全不同的数据与实现，满足同接口多版本需求（`Remote`在 `SonyTv/LGTv` 内部各自实现）
		- 隐藏：
			- private 嵌套类完全对外不可见，库升级时可随意增删内部节点类型而保持 ABI 兼容
			- ABI 兼容 = “二进制接口兼容”（Application Binary Interface）：升级库文件（`.so / .dll / .dylib`）后，已经编译好的旧的可执行文件无需重新编译链接，直接替换动态库就能跑，不会崩溃、不会链接失败
