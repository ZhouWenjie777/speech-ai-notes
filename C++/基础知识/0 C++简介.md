# 1. C++语言
1. 定位：系统级、高性能通用语言，覆盖科学计算、网络、游戏、图像处理等场景
2. 时间线
	- 1979：贝尔实验室给 C “加类”，初名 “C with Classes”
	- 1983：正式更名 C++
	- 1998：ANSI 发布首版标准（C++98），进入标准化时代
3. 编程范式：一门语言，三种写法
	- 过程化（Procedural）
	- 面向对象（OOP）
	- 泛型（Generic）
4. 过程化程序设计
	- 核心视角：数据 + 算法。明确机器每一步如何加工数据，再用语言把流程“翻译”成代码
	- 规模痛点 ：代码量一上来，函数/变量随意耦合 → 结构混乱、调试困难、正确性难保证
	- 结构化改良
		- 自顶向下：先画总体框架，再逐级细化
		- 控制结构“三件套”：顺序、分支、循环，其余一律禁用
	- C 语言：支持结构化程序设计
		- 定位：结构化程序设计原生支持的语言，兼具“接近汇编”的效率与“接近高级”的可移植性
		- 时间线
			- 1970s 早期：贝尔实验室开发 UNIX 时顺手发明
			- 1989：ANSI 发布首个完整标准（C89），语言正式定型
		- 双重特点
			- 低级语言的效率、硬件访问能力：直接操作地址、位、寄存器，写驱动、写 OS 毫无压力
			- 高级语言的通用性、可移植性：控制流简洁、类型系统简单，一份源码跨平台编译
5. 面向对象程序设计
	- 核心思路：让“数据类型”直接映射问题域实体，而非强迫问题去适配机器流程
	- 两步走
		- 设计类：把现实世界概念（直线、圆、图片……）封装成属性 + 操作
		- 使用类：在主逻辑里组合、消息传递，程序 ≈ 对象交互脚本
	- 收益
		- 开发体验：问题与代码同构，思考负担 ↓
		- 代码重用：类即模板，继承/组合即可复用、扩展
6. 泛型程序设计
	- 核心思想：写一次代码，适用于任意类型，把“类型”当成参数传进来
	- 用途
		- 提供通用工具：排序、查找、容器、算法……一份实现，覆盖所有数据类型
		- 组成：泛型函数 + 泛型类，编译期自动实例化为具体版本。
	- 结果
		- 零运行时开销：模板在编译期展开，生成与手写专用代码同效率的机器码
		- 极高复用度：标准库 ≈ 泛型大仓库，业务代码专注于领域逻辑
7. 程序创建流程
	- 阶段一：写源码
		- 用任意文本编辑器（vi、emacs）或 IDE（VS2010 等）录入代码，保存为 `*.cpp` 源文件
	- 阶段二：编译
		- 编译器把 `*.cpp` 翻译成机器码，生成 `*.o` / `*.obj` 目标文件
		- 示例（Windows + GCC）：`gcc -c prog.cpp`  # 生成 prog.o
	- 阶段三：链接
		- 链接器把目标文件与运行时库、第三方库目标文件捆在一起，输出可执行文件
		- 同一示例：`gcc prog.o -o prog.exe`  # 得到 prog.exe
	- 注意：具体命令、扩展名、调试信息开关随 OS 与编译器而定，但“编译→链接”两步本质不变。
8. C++程序示例
	- 注释：
		- `//`单行或`/* */`多行，编译器忽略，给程序员看
	- 预处理指令：
		- 以`#`开头，例`#include <iostream>`在编译前把`iostream`内容插入
	- 头文件：
		- 老版带`.h`，新版无后缀；C库头加`c`前缀，如`<cstdio>`
	- 名字空间：
		- 防同名冲突，标准库都在`std`，可用`std::cout`或`using namespace std;`一次性打开
	- main函数：
		- 操作系统入口，原型`int main()`，返回0表示正常结束
	- 基本流程：
		- 预处理→编译→链接，生成可执行文件后运行
9. 声明语句和变量
	- 命名规则：
		- 字母、数字、下划线，首字符非数字，区分大小写，避开关键字，长度平台有限制
	- 定义格式：
		- `类型名 变量列表;` 如`int carrots;` 立即分配存储空间，必须先定义后使用
	- 赋值语句：
		- 右结合`=`，如`carrots = 25;` 把右值存进对应内存；`carrots = carrots - 1;` 读-改-写原子操作
	- cout输出：
		- 自动把变量值转成字符流，可拼接字符串与变量，如`cout << "I have " << carrots << " carrots.";`
	- cin输入：
		- `cin >> 变量;` 从键盘读，支持链式`cin >> x >> y;` 按空白分隔
	- 示例要点：
		- 输入 carrots → 加 2 → 输出新值，展示变量定义、赋值、输入/输出全流程
10. 函数入门
	- 定义格式：
		- 返回值类型 函数名(参数表) { 函数体; } 
		- 无返回写void，无参写void或空括号
	- 原型声明：
		- 返回值类型 函数名(参数类型); 
		- 告诉编译器接口，可提前调用；通常放头文件或main前
	- 有返回值：
		- 用return表达式; 表达式类型须与返回值匹配，main返回0表示正常结束
	- 无返回值：
		- void函数体可省略return，执行完自动回退；用于“只做动作”场景
	- 调用方式：
		- 函数名(实参列表); 有返回值可参与表达式或赋值，无返回值单独成语句
