1. 拷贝构造与赋值运算符重载（深拷贝）
	- 场景：类内含“指向动态内存的指针”时，默认逐位拷贝仅复制指针值，导致
		- 多次析构同一块堆内存 →  double free 崩溃；
		- 修改一个对象会殃及另一个 →  逻辑错误
	- 拷贝构造函数
		- 格式：`String(const String& other)`
		- 实现要点：
			- 计算源字符串长度 `len = other.len;`
			- 分配新空间 `data = new char[len + 1];`
			- 逐字节复制 `strcpy(data, other.data);` （或手写循环）
			- 确保新对象与源对象完全独立
	- 赋值运算符重载
		- 格式：`String& operator=(const String& other)`
		- 经典“自赋值保护 + 先释放再申请”套路
			- `if (this == &other) return *this;`   // ① 自赋值检测
			- `delete [] data;`                                  // ② 释放旧资源
			- `len = other.len; data = new char[len + 1];`   // ③ 申请新空间
			- `strcpy(data, other.data);`                 // ④ 深拷贝
			- `return *this;`               // ⑤ 返回引用，支持链式 `s1 = s2 = s3;`
	- 运算符 + 与 == 实现（非成员，友元）
		- `operator+`：创建临时对象 `tmp`，长度 `s1.len + s2.len`，先拷贝 `s1` 再追加 `s2`，返回 `tmp`（依赖拷贝构造优化）
		- `operator==`：先比较长度，再逐字符比对，相等返回 true
	- 输出运算符 `<<`必须为非成员：
		- `ostream& operator<<(ostream& os, const String& s)`
		- 直接`os << s.data;`  并返回`os`以支持链式输出
	- 常见错误回放
		- 使用编译器默认拷贝/赋值 → 浅拷贝 → 程序异常终止
		- 赋值函数未返回引用 → 无法链式赋值
		- 忘记自赋值保护 → 先 delete 后读取，造成野指针
	- 规则总结
		- 一旦类里出现“资源管理型”成员（`new、fopen、mutex`等），必须同时提供
			- 析构函数（释放）
			- 拷贝构造函数（深拷贝）
			- 赋值运算符重载（深拷贝 + 自赋值保护）简称“三/五法则”（C++11 起加入移动版本）
