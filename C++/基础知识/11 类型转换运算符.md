1. 类型转换运算符
	- C++ 提供的 4 种类型转换运算符
		- **const_cast**：
			- 删除对象的 const 或 volatile 属性，但修改 const 对象是未定义行为
			- `const int pop2 = 2000;`
			- `int* pc = const_cast<int*>(&pop2); // 去掉 const`
			- `pc += 100;              // 不安全，修改 const 对象`
		- **static_cast**
			- 用于隐式转换（如 int → double）或派生类指针/引用向上转型
			- `double x = 1.998;`
			- `int a = static_cast<int>(x); // 显式截断`
		- **reinterpret_cast**
			- 强制转换指针或引用类型，但不改变数据位模式，仅重新解释类型
			- `struct data { int a; int b; }; data at = {5, 7};`
			- `double* dp = reinterpret_cast<double*>(&at); // 危险，类型不匹配`
		- **dynamic_cast**
			- 仅用于多态基类指针/引用向下转型，运行时检查安全性
			- `Base* pb = new Derived;`
			- `Derived* pd = dynamic_cast<Derived*>(pb); // 安全向下转型`
	- C 风格强制转换
		- `(类型名)表达式`：不安全，会跳过类型检查，可能引发未定义行为
		- `int a = 5; double* dp = (double*)&a; // 危险，类型不匹配`
		- C 风格一句话干4件事，隐患大；C++ 强制拆分4 种类型，每一步意图清晰，编译器可逐层校验
	- 转换运算符对比
		- const_cast：仅修改类型修饰符，不改变数据
		- static_cast：安全的隐式转换或向上转型，不涉及运行时检查
		- reinterpret_cast：危险，仅改变类型解释，不改变数据位模式
		- dynamic_cast：安全的向下转型，涉及运行时检查，可能返回 nullptr 或抛异常
	- 使用建议
		- 尽量避免类型转换，优先使用多态或模板。
		- 必须转换时，优先用 static_cast 或 dynamic_cast，避免 C 风格和 reinterpret_cast
		- 使用 const_cast 时，确保不修改原 const 对象，否则行为未定义
		- 使用 dynamic_cast 时，确保基类有虚函数，且开启 RTTI
