1. 函数模板
	- 定义：
		- `template<typename T> 返回类型 函数名(形参表) { ... }`
		- 用T代替具体类型，编译器根据实参推导出真正函数并实例化
	- 实例化：
		- 调用`swapT(a,b)`时，若a,b为int则自动生成`swapT(int&,int&)`；为double则生成`swapT(double&,double&)`，避免重复编码
	- 显式实例化：
		- 在调用点把模板参数写在尖括号里，明确告诉编译器具体版本
		- 示例：若有模板 `template<class T1,class T2,class T3> T3 calc(T1 x,T2 y){return x+y;}`
		- 则 `calc<int,char,int>(5,'a')` 强制把`T3`绑定成int，返回`int(102)`
		- 若写`calc(5,'a')`编译器会推成`calc<int,char,int>`，结果相同
		- 但显式写法可消除歧义或控制返回类型
	- 多模板参数/尾置返回：
		- `template<class T1,class T2> auto calc(T1 x,T2 y)->decltype(x+y)` 
		- 让返回类型随表达式自动推导，支持混合类型运算
2. 函数模板重载
	- 规则：同名模板之间必须参数列表不同（个数、类型或顺序），与具体化程度无关；编译器按“最特化优先”原则挑选。
	- 示例：提供两个版本
		- `template<typename T> void Swap(T& a, T& b);`          // 交换单值
		- `template<typename T> void Swap(T* a, T* b, int n);`   // 交换数组
		- 调用 `Swap(i, j)` 走版本1；`Swap(d1, d2, Lim)` 走版本2，自动匹配指针+长度形式
	- 注意：非模板函数与模板共存时，完全匹配的非模板优先；否则才实例化更合适的模板
3. 显式具体化
	- 场景：
		- 默认模板对某类型行为不合需求（如只交换结构部分成员），可提供**显式具体化**版本
	- 语法：
		- 先保留通用模板，再写 `template<> 返回类型 函数名<具体类型>(形参表) { ... }`  
		- 例：`template<> void Swap<job>(job &j1, job &j2)` 只交换 salary 与 floor，忽略 name。
	- 匹配优先级：
		- 普通函数 > 显式具体化 > 普通模板；
		- 当实参类型与具体化完全吻合时优先调用该版本
	- 注意：具体化必须与通用模板**同名且参数列表一致**，否则视为重载而非特化
