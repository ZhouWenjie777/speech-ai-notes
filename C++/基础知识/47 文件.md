1. 简单文件 I/O：文本文件读写全流程
	- 头文件与流类
		- `#include <fstream>   // 提供 ofstream / ifstream / fstream`
		- `std::ofstream`：
			- 方向—输出（只写）、默认模式—`ios::out`|`ios::trunc`、常用场景—日志、导出、生成新文件
		- `std::ifstream`
			- 方向—输入（只读）、默认模式—`ios::in`、常用场景—配置读取、数据加载
		- `std::fstream`
			- 方向—双向、默认打开模式—`ios::in`|`ios::out`、常用场景—随机读写、原地修改、数据库式文件
	- 关联文件两种方式
		- 构造函数直接打开
			- `std::ofstream fout(filename);  // filename 可为 string 或 const char*`
		- 先定义对象，再 `open`
			- `std::ofstream fout;`
			- `fout.open(filename, std::ios::out | std::ios::app); // 追加写`
	- 检测打开成功与否
		- `if (!fout) { /* 错误处理 */ }   // 运算符重载 operator bool`
		- `if (fout.fail()) { /* ... */ }`
	- 写入操作
		- 与 `cout` 语法完全一致
		- `fout << "For your eyes only!\n";`
		- `fout << "Secret number is " << secret << std::endl;`
	- 读取操作
		- 与 `cin` 语法一致，可按行、按字符、按格式化提取
		- `std::ifstream fin(filename); char ch; while (fin.get(ch))  std::cout << ch; // 逐字符，含空白`
		- `std::string line; while (std::getline(fin, line)) std::cout << line << '\n';    // 按行`
	- 关闭与自动清理
		- `fout.close();   // 显式关闭，刷新缓冲区`
		- 离开作用域时析构函数自动调用 `close()`，但仍推荐手动关闭以立即检测错误（`failbit`）
	- 文件模式位（ `ios_base` 标志）
		- `app` 追加写，每次写前定位到尾
		- `ate` 打开后立刻定位到尾（可读写）
		- `binary` 二进制模式，不作换行转换
		- `in` 读
		- `out` 写
		- `trunc` 若文件存在先清空（默认与 `out` 配合）
		- 组合示例：`std::fstream fs("db.dat", std::ios::in | std::ios::out | std::ios::binary);`
	- 随机访问（简单提及）
		- `fin.seekg(0, std::ios::beg);    // 读指针移到开头`
		- `fout.seekp(-10, std::ios::end); // 写指针移到尾前 10 字节`
	- 异常安全
		- 析构时自动释放文件句柄
		- 抛异常亦能保证关闭
	- 与控制台 I/O 对比
		- 文件流需显式定义对象并关联文件名
		- 语法与 `cin/cout` 完全一致，可无缝切换
		- 关闭后仍可重新 `open` 另一文件，复用同一个流对象
	- 最小完整模板
		```cpp
	  #include <fstream>
	  #include <iostream>
	  int main() {
		  std::ofstream fo("out.txt");
		  if (!fo) return 1;
		  fo << "Hello file\n";
		  fo.close();
		  
		  std::ifstream fi("out.txt");
		  if (!fi) return 1;
		  char ch;
		  while (fi.get(ch)) std::cout << ch;
		  fi.close();
		  return 0;
	  }
	  ```
2. 文件打开模式 & 文本 vs 二进制
	- 打开模式一览（`ios_base` 标志可位或组合） 
		- `in`：读；`ifstream` 默认已含
		- `out`：写；`ofstream` 默认已含
		- `app`：每次写前先定位到文件尾（append）；与 `trunc` 互斥
		- `ate`：打开后立刻定位到文件尾；可读写，不影响后续 seek
		- `trunc`：若文件存在先清空；`ofstream` 默认隐含
		- `binary`：以二进制模式打开，不做字符转换；可与其他位或
	- 组合示例
		- `ofstream fout("log.txt", ios_base::app);` 文本追加写
		- `ofstream fout("data.bin", ios_base::app | ios_base::binary);` 二进制追加写
		- `fstream fs("db.dat", ios_base::in | ios_base::out | ios_base::ate);` 读写且不清空，定位到尾
		- `ifstream fin("image.png", ios_base::binary);` 只读二进制
	- 成功性检查
		- `ofstream fout("file1", ios_base::app);`
		- `if (!fout) 运算符隐式转换`
		- `if (fout.fail()) { /* 错误 */ }`
		- `if (fout.is_open()) { /* 已打开 */ }`
	- 文本文件 vs 二进制文件（以-1为例）
		- |模式|写入字节|文件内容|大小|可读性|
		  |---|---|---|---|---|
		  |文本|`fprintf "%d"`| 字符 `-` 和 `1` | 2 B | 人可读 | 
		  |二进制|`fwrite(&x, 4, 1, fp)`| 补码 `0xFFFFFFFF` | 4 B | 不可直接读|
		- ```cpp
		  int x = -1;
		  ofstream ftxt("a.txt");
		  ftxt << x;               // 写入 "-1"
		  
		  ofstream fbin("a.bin", ios_base::binary);
		  fbin.write(reinterpret_cast<const char*>(&x), sizeof(x));
		  ```
	- 字符转换差异
		- 文本模式：`\n`→系统换行序列→Windows `\r\n`，Unix `\n`；读取时逆转换
		- 二进制模式：按字节原样读写，不转换换行符；适合图片、音频、结构化记录
	- 定位与大小（二进制常用）
		- `fs.seekg(0, ios_base::end);   // 读指针到尾`
		- `auto len = fs.tellg();        // 字节数`
		- `fs.seekg(0, ios_base::beg);   // 回到头`
	- 异常安全再强调
		- 打开失败不会抛异常，只置位；写入失败可能抛异常（底层 I/O 错误）
		- `fout.exceptions(ofstream::failbit | ofstream::badbit);` 开启异常机制
	- 最佳实践
		- 只要文件不是给人直接读，一律加 `| ios_base::binary`，避免换行转换与编码意外
		- 追加写用 `app`；随机读写用 `fstream` + `ate` 定位，别用 `app`
		- 同时读写：打开 `in|out`，否则第二次操作会失败
		- 写完立即 `close()` 并检查 `failbit`，确保数据落盘
3. 二进制文件访问（read / write 版）
	- 读写原型（成员函数）
		- `ostream& write(const char* addr, streamsize size);`
		- `istream& read (char* addr, streamsize size);`
		- 二者均返回流对象本身，可链式；
		- 参数 `addr` 需强转为 `char*`：`reinterpret_cast<char*>(&x)`
	- 结构体整块存取示例
		- `struct Planet { char name[20]; long population; double g; };`
		- `Planet pl{};`
		- `fout.write(reinterpret_cast<char*>(&pl), sizeof(pl));  // 写`
		- `fin.read (reinterpret_cast<char*>(&pl), sizeof(pl));  // 读`
		- 特点：文件字节数 = `sizeof(Planet)`，与值无关，随机访问时可直接  `seekg(pos * sizeof(Planet))`
	- 文件结束判断
		- `read` 后应检查流状态 `while (fin.read(reinterpret_cast<char*>(&pl), sizeof(pl))) { // 处理 pl }`
		- `if (fin.eof()) cout << "到达文件尾\n";` 
		- `if (fin.fail()) cout << "格式或尺寸错误\n";`
		- 仅用 `eof()` 判断会多读一次；应以 `read` 返回值为主
	- 字节大小与平台
		- `sizeof(int)` 、`sizeof(long)`随平台变化；
		- 长期存储或跨网络应使用固定宽度类型（`int32_t`、`uint64_t`等）或手动序列化
	- 随机访问（seek/tell）
		- `fin.seekg(2 * sizeof(Planet)); // 跳至第 3 条记录`
		- `fin.read(...);`
		- `auto pos = fin.tellg();        // 当前字节偏移`
	- 异常模式（可选）
		- ```cpp
		  fin.exceptions(std::ifstream::failbit | std::ifstream::badbit);
		  try {
			  fin.read(reinterpret_cast<char*>(&pl), sizeof(pl));
		  } catch (std::ios_base::failure& e) {
			  std::cerr << "I/O error: " << e.what() << '\n';
		  }
		  ```
	- 性能提示
		- 二进制读写无格式转换，速度比文本高一个数量级
		- 大块数据一次性 `read/write` 优于循环单字节
		- 打开时加 `std::ios::binary` 禁止换行转换，否则 Windows 下 `\n` ↔ `\r\n` 破坏定长记录
	- 常见误区
		- 用 `eof()` 作为 `while` 条件 → 最后一条多读一次
		- 忘记 `ios::binary` → 文件变大/变短，记录错位
		- 直接 `write` 含指针的成员 → 下次运行地址失效，应序列化深层数据
	- 完整最小模板
		- ```cpp
		  #include <fstream>
		  #include <iostream>
		  int main() {
			  std::ofstream fout("data.bin", std::ios::binary);
			  int x = 5;
			  fout.write(reinterpret_cast<char*>(&x), sizeof(x));
			  fout.close();
			  
			  std::ifstream fin("data.bin", std::ios::binary);
			  int y;
			  fin.read(reinterpret_cast<char*>(&y), sizeof(y));
			  std::cout << y;          // 5
			  fin.close();
		  }
		  ```
	- 与文本对比
		- |维度|文本|二进制|
		  |:---:|---|---|
		  |可读性|可直接阅读|不可读|
		  |大小|数字变大（-1→“-1”占 2 B）|固定 sizeof|
		  |跨平台|需处理本地化/换行|需处理字节序与大小|
		  |速度|慢（格式转换）|快（原样读写）|
		  |随机访问|需解析行|直接 seek 到记录|
4. 随机访问（`seekg` / `seekp`）
	- 打开方式
		- 必须同时支持 **读+写** 且 **二进制** 模式，防止换行符被篡改导致记录错位
		- `std::fstream fs("planets.dat", std::ios_base::in | std::ios_base::out | std::ios_base::binary);`
		- `if (!fs.is_open()) { /* 错误处理 */ }`
	- 定位函数
		- |函数|作用|偏移原点|原型|
		  |---|---|---|---|
		  |`seekg(offset, origin)`|设置 **读** 指针|`beg` / `cur` / `end`|`istream&`|
		  |`seekp(offset, origin)`|设置 **写** 指针|同上|`ostream&`|
		- 原点缺省值为 `ios_base::beg`（文件开头）
	- 记录定长策略
		- 结构体大小固定：`sizeof(Planet)`
		- 第 `rec` 条记录的字节偏移 = `rec * sizeof(Planet)`
		- 因此可随机跳转到任意记录读写，而无需顺序扫描
	- 显示文件内容（只读遍历）
		- ```cpp
		  fs.seekg(0); 
		  Planet pl; 
		  size_t ct = 0;
		  while (fs.read(reinterpret_cast<char*>(&pl), sizeof(pl))) {
			  std::cout << ct++ << ": " << pl.name << " ...\n";
		  }
		  if (fs.eof()) std::cout << "到达文件尾\n";
		  else if (fs.fail()) std::cerr << "读取出错\n";
		  ```
		- `fs.read(...)` 是 `std::ifstream` 的成员函数，内部做了两件事
			- 把当前文件位置开始的 `sizeof(Planet)` 字节拷进 `pl`
			- 把文件位置指针向后移动读过的字节数 `sizeof(Planet)`
	- 修改指定记录（读-改-写）
		- ```cpp
		    // 计算偏移
		    std::size_t rec = 用户输入序号;
		    std::streamoff offset = 
			    static_cast<std::streamoff>(rec) * sizeof(Planet);
			// 定位 → 读出
			fs.seekg(offset);
			fs.read(reinterpret_cast<char*>(&pl), sizeof(pl));
			// 修改内存中结构体
			std::cout << "Enter new population: ";
			std::cin >> pl.population;
			// 定位 → 写回（同一偏移）
			fs.seekp(offset);   // 写指针移到该记录开头
			fs.write(reinterpret_cast<const char*>(&pl), sizeof(pl));
			// 错误检测
			if (fs.fail()) {
			    std::cerr << "写入失败\n";
			    exit(EXIT_FAILURE);
			}
		  ```
	- 同时读写指针
		- `fstream` 内部共用同一文件位置，但对 `seekg` / `seekp` 分开维护；读写切换后必须手动 `seek`，否则结果未定义
		- 例：读完立刻写，需 `seekp`(当前位置)  或 `seekg`(当前位置)  明确同步
	- 获取当前偏移
		- `std::streampos hereg = fs.tellg();   // 读指针位置`
		- `std::streampos herep = fs.tellp();   // 写指针位置`
	- 文件长度计算
		- `fs.seekg(0, ios_base::end);`
		- `std::streampos fileSize = fs.tellg();`
		- `std::size_t recordCount = fileSize / sizeof(Planet);`
	- 异常模式（可选）
		- `fs.exceptions(fstream::failbit | fstream::badbit);`
		- `try { fs.write(...); }`
		- `catch (const ios_base::failure& e) { /* 处理 */ }`
	- 最佳实践 checklist
		- 定长记录 → 用结构体，不含指针/可变成员
		- 始终 `binary` 模式，防止 `\n` 被扩张/收缩
		- 修改前先读后写，写完立即检查 `failbit`
		- 多线程场景：对同一文件用 `seek+read/write` 需外部同步，或改用记录锁
