1. const 与类
	- const 对象：
		- 定义格式 `const 类名 对象(构造实参);` 
		- 一旦初始化完成，任何成员变量均不允许再被修改
	- 语法限制：
		- 编译器只允许对 const 对象调用 **const 成员函数**，否则报错；从而保证对象状态不变。
	- const 成员函数：
		- 在参数列表后加 const 关键字，如`返回类型 函数名(形参表) const { ... }`
		- 内部若尝试修改数据成员或调用非 const 成员函数，编译期报错
	- 重载特性：
		- 同一函数名可同时提供 const 与非 const 版本，编译器根据对象是否为 const 自动选择
		- 常见如 `T& operator[](size_t)` 与 `const T& operator[](size_t) const`
	-  mutable 例外：
		- 被 mutable 修饰的数据成员即使在 const 对象中也可被修改，用于缓存、引用计数等逻辑上“不影响对象值”的场景
2. mutable 成员
	- 作用：被 mutable 修饰的数据成员**即使在 const 对象中也可被修改**，逻辑上“不影响对象外部可见值”。
	- 使用场景：缓存、引用计数、调试标记、惰性求值等“内部 housekeeping 数据”。
	- 语法：在类定义里直接写 `mutable 类型 成员名;` 即可，与其他成员一样访问。
	- 规则：只能在非静态数据成员上使用；const 成员函数里可多次修改，对象整体仍被视为 const。
3. “非静态数据成员”就是属于对象、而不属于整个类的普通变量成员
	- 非静态 → 每个对象各有一份，随对象生而生，随对象死而死
	- 静态 → 全类只有一份，跟对象无关，用 `static` 关键字声明
	- `class Foo { int  a;// 非静态数据成员（每个 Foo 对象都有自己的 a）static int b;// 静态数据成员（整个 Foo 类共享一份 b）};`
	-  `mutable` 只能修饰像 `a` 这种对象级的成员，不能修饰像 `b` 这种的静态成员
