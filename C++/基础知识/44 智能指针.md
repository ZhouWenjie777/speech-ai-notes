1. 基础概念
	- 裸指针的痛点
		- 必须手动 `delete`，一旦忘记、提前return、抛异常，就**内存泄漏**
		- 多次 `delete` 同一地址 → **double-free** 未定义行为
		- 无法告诉“别人”自己是否还拥有这块内存，**所有权语义不明确**
	- 智能指针本质：
		- “用对象管理资源”（RAII）思想的类模板；
		- 构造函数里 `new` / 接管指针；
		- 析构函数里 `delete`（或 `delete[]`）并置空；
		- 生命周期结束（离开作用域、delete、容器擦除）时**自动释放**，无需手写`delete`
	- 所有权模型（三种策略）
		- **独占**：同一时刻**只有一个**智能指针拥有对象；拷贝即转移
		- **共享**：多个智能指针**共同拥有**对象；内部引用计数为 0 时才真正释放
		- **弱引用**：只观察对象，**不计入引用计数**；用于打破循环，可能返回空
	- 智能指针目的
		- **异常安全**：函数中途抛异常，栈展开时析构函数仍被调用，资源不会泄漏
		- **自动释放**：无论多少条 return 路径，离开作用域即清理
		- **语义清晰**：代码里看到 `unique_ptr` 就知道“独占”，看到 `shared_ptr` 就知道“共享”，可读性高
		- **与 STL/算法无缝协作**：可复制、可移动、可放入容器，无需手写删除器
	- 通用语法
		- `#include <memory>`
		- `std::unique_ptr<Foo> up{new Foo(args)};   // 独占`
		- `std::shared_ptr<Foo> sp = std::make_shared<Foo>(args); // 共享`
		- 推荐`std::make_unique / std::make_shared`一次分配，减少内存碎片与异常风险
	- 头文件与命名空间
		- 全部位于 `<memory>`，属于命名空间`std`；
		- 模板参数为指向的对象类型，不是指针写法；
	- 简单对比裸指针
		- 裸指针：手动 new/delete；异常时易泄漏；所有权模糊；可随意拷贝
		- 智能指针：自动构造/析构；异常安全；语义明确（独占/共享）；按策略禁止或共享拷贝
		- 智能指针 = “资源管理对象”，把内存生命周期绑定到对象生命周期，用析构函数自动释放，从根本上消灭忘记`delete`的问题
2. 智能指针对比（`auto_ptr vs unique_ptr vs shared_ptr`）
	- 不再使用`auto_ptr`；用`unique_ptr`管理独占资源，用`shared_ptr`管理共享资源，循环引用处加`weak_ptr`

| 特性     | auto_ptr（C++98 已弃用）           | unique_ptr（C++11）                            | shared_ptr（C++11）                       |
| ------ | ----------------------------- | -------------------------------------------- | --------------------------------------- |
| 所有权模型  | 独占，但**可复制**；复制时**转移**所有权，右值变空 | 独占，**不可拷贝**；仅可 `move` 转移                     | 共享，**引用计数**；最后一个 `shared_ptr` 析构时才释放对象  |
| 拷贝语义   | `p1 = p2;` 编译通过，`p2` 成空，极易误用  | `p1 = p2;` **编译错误**；必须 `p1 = std::move(p2);` | `p1 = p2;` 合法，引用计数 +1                   |
| 数组支持   | 不支持数组 `new[]`                 | 支持 `unique_ptr<T[]>`，自动调用 `delete[]`         | 支持 `shared_ptr<T[]>`（C++17 起）           |
| 自定义删除器 | 不支持                           | 支持，模板参数或 `std::unique_ptr<T, Deleter>`       | 支持，`shared_ptr<T> p(new T, deleter);`   |
| 与容器/算法 | 可复制导致坑，被禁用                    | 可`move`进容器，安全                                | 可直接放入容器，共享语义                            |
| 线程安全   | 无，转移非原子                       | 无，独占                                         | 引用计数原子，对象本身仍需手动同步                       |
| 循环引用   | 不存在                           | 不存在                                          | 需 `std::weak_ptr<>` 打破循环，否则内存泄漏         |
| 异常安全   | 转移可能抛，已弃用                     | `unique_ptr` 构造/析构 noexcept，最安全              | 引用计数原子，但构造 `shared_ptr` 可能抛 `bad_alloc` |
```cpp
// unique_ptr：独占，不可拷贝
std::unique_ptr<Report> p1(new Report("uni"));
// std::unique_ptr<Report> p2 = p1;     // ❌ 编译错误
std::unique_ptr<Report> p2 = std::move(p1); // ✅ 转移

// shared_ptr：共享，可拷贝
std::shared_ptr<Report> s1(new Report("share"));
std::shared_ptr<Report> s2 = s1;   // ✅ 引用计数+1

// 数组形式（unique_ptr 示例）
std::unique_ptr<std::string[]> films(new std::string[5]);
films[0] = "Gone with the Wind";
// 自动 delete[] 释放
```
