1. 异常机制基础
	- 抛出（throw）
		- 语法：`throw 表达式;`  可抛出**任何类型**（int、字符串、对象、指针等）
		- 例：`throw bad_hmean(a,b);`  构造临时对象并拷贝到异常对象
	- 检测点（try）
		- 把可能出错的代码块包在 `try { ... }` 中；一旦 `throw` 发生，控制流立即跳出当前函数，沿调用栈寻找匹配 `catch`
	- 捕获（catch）
		- 语法：`catch (异常类型 形参) { ... }`
		- 按**类型严格匹配**（允许派生→基类、const、引用转换），**无隐式数值转换**
		- 多个 `catch` 分支按书写顺序**自上而下**匹配，只进入第一个匹配的处理器
		- `catch (...)` 可捕获**任何异常**，通常放最后做兜底
	- 自定义异常类
		- 推荐从 `std::exception` 派生，重写 `const char* what() const noexcept`
			- `std::exception`本身只有一个虚函数`virtual const char* what() const noexcept;`
			- 在派生类中重写 `what()`，把错误信息带出去`const char* what() const noexcept override {←关键接口 return buf;}`
		- 也可像示例那样手写小型类，携带专有数据成员 `v1/v2` 和 `mesg()` 方法，供 `catch` 块按需读取
	- 栈展开（stack unwinding）
		- 从 `throw` 点开始，逐层析构已构造的自动对象，直到找到匹配 `catch`；若最终无 `catch`，调用 `std::terminate`（默认 `abort`）
	- 异常规范（C++11 前已废弃，了解即可）
		- `double gmean(double a, double b) throw(const char*);`
		- 声明函数可能抛出的类型，违反时在运行时调用 `unexpected()`；现代 C++ 改用 `noexcept`
	- 与错误码对比
		- 异常：强制处理、自动传播、无法忽略；适合**罕见、严重、无法就地恢复**的错误
		- 返回值/输出参数：调用者易遗漏检查；适合**频繁、可预期、可就地处理**的状态
	- 最佳实践
		- **按引用捕获**自定义对象，避免切片与拷贝开销：`catch (const bad_hmean& e)`
		- 不抛析构函数异常；若析构函数可能抛，用 `try/catch` 内部吞掉并记录
		- 不在异常类型里放大型数据，用智能指针或错误码字段传递资源/详情
		- 入口 `main` 用 `catch (...)` 兜底，记录日志后 `return EXIT_FAILURE`，防止程序直接崩溃
2. exception类与标准异常体系
	- 标准基类
		- `#include <exception>`
		- 核心接口：`virtual const char* what() const noexcept;` 
		- 返回描述性字符串，派生类可重写以提供具体错误信息
	- 典型派生异常（均定义于 `<stdexcept>`）
		- `std::runtime_error`   运行时错误（越界、溢出等）
		- `std::logic_error`      逻辑错误（前置条件不满足）
		- `std::bad_alloc`         内存分配失败（`new` 抛出）
		- `std::bad_cast`           动态转型失败
		- `std::bad_exception`   异常规范违例（已废弃）
		- `std::out_of_range`    容器/数组下标越界
		- `std::invalid_argument` 非法实参
		- `std::length_error`     长度超限
	- 使用模式
		- `try { pb = new Big[10000]; }` 可能抛出 `std::bad_alloc`
		- `catch (std::bad_alloc& e) { std::cerr << e.what() << '\n'; exit(EXIT_FAILURE); }` 按引用捕获，避免切片
		- 按引用捕获基类即可包揽整个派生体系：`catch (const std::exception& e)` 一条分支就能处理所有标准异常
	- 自定义异常
		- 推荐继承体系：
			- `class bad_hmean : public std::runtime_error { public:`
			- `bad_hmean(double a, double b) : std::runtime_error("invalid arguments to hmean"), v1(a), v2(b) {}`
			- `double v1, v2; };`
		- 重写`what()`可选，基类已提供字符串构造器；新增数据成员供`catch`块读取
	- 异常安全保证
		- 基本：异常发生后对象不泄漏资源、不变式保持
		- 强：操作要么成功，要么回滚到调用前状态
		- 不抛：承诺绝不抛出异常（`noexcept`）遵循“获取资源即初始化”（RAII），用智能指针、容器自动释放内存，减少手写`try/catch`
	- 内存失败策略
		- `new`失败默认抛`std::bad_alloc`；可用`nothrow`版本返回`nullptr`兼容旧代码：`int* p = new (std::nothrow) int[big];`
		- `noexcept`析构：析构函数绝不允许抛异常，否则程序直接`terminate()`
	- 最佳实践速记
		- 抛对象，捕引用；自定义继承`std::exception`
		- 资源管理靠 RAII，别让`catch`里再`delete`
		- 一层`catch(const std::exception&)`兜底，顶层`catch(...)`防猝死
