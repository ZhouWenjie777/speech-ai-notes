1. 左值（`lvalue`）
	- 有名字、有地址、生命周期超出当前表达式
	- 代表“**对象本身**”，可以放在赋值号左边
	- 例：变量、解引用后的 `*ptr`、返回引用的函数调用结果
2. 右值（`rvalue`）
	- 没名字、没持久地址、生命周期只在当前表达式
	- 代表“**纯数值/临时结果**”，只能读，不能写
	- 例：字面量 `42`、临时对象 `Foo()`、算术表达式 `a+b`
3. 示例
	- `int  a = 1;     // a 是左值`
	- `int& r = a;     // OK，左值绑定到左值引用`
	- `int& r2 = 2; // ERROR，字面量 2 是右值`
	- `int&& rv = 2;   // OK，右值引用专门接右值`
	- `cout << &a;     // OK，能取地址`
	- `cout << &(a+b); // ERROR，a+b 是右值，没持久地址`
4. 总结
	- 能取地址`&`的是左值，不能的是右值
	- 要把 `*ptr` 当成“变量本体”而不是“里面的数字”
	- 不要把引用 `r` 或 `rv` 想成“地址”，它们都是别名（alias），不是指针、不是地址，对它们取地址`&r` / `&rv`才得到地址
	- `&` 出现在**类型**里是**引用**（别名），出现在**表达式**里是取**地址**；引用本身不是指针，只是同一块内存的另一个名字
	- | 声明 | 念法 | 绑定对象 | 生命周期 | 典型用途 |
	  |---|---|---|---|---|
	  | `int& r=obj;` | 左值引用 | 必须是**左值**(有持久地址) | 与`obj`相同 | 传参/返回值避免拷贝 |
	  | `int&& rv=42;` | 右值引用 | 必须是**右值**(临时、将亡) | 延长临时对象寿命 | 实现移动语义、完美转发 |
